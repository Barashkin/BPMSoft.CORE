## Назначение

Этот документ — **конституция** справочной базы по дереву `docs/` и по коду приложения
конфигурации (см. размещение **`Autogenerated`** относительно корня workspace в
[configuration-workspace-layout.md](architecture/configuration-workspace-layout.md)):

- задаёт неизбыточное **замыкание**: каждая страница включена в навигируемый граф без
  «повисших» файлов по отношению к корневым узлам;
- фиксирует единственные входы по доменам (primary entry), чтобы исключить
  конкурирующее описание одной темы;
- задаёт процедуры **изменений** и **верификации**, применимые к всей дереву
  документации.

Операционное оглавление и обучение по-прежнему в **[INDEX.md](INDEX.md)**; быстрая карта для агента — **[AGENT_INDEX.md](AGENT_INDEX.md)**. Энциклопедия не заменяет их, а **страхует согласованность**.

## Корневые узлы графа (хабы)

Следующие файлы задают начальный обход («замыкание без сирот»). Из них должны быть
при достижении по внутренним ссылкам все остальные `*.md` в `docs/`:

| Корень | Роль |
| ------ | ------ |
| [README.md](README.md) | Вход со стороны репозитория, переход на INDEX и разделы |
| [INDEX.md](INDEX.md) | Человекочитаемое оглавление, learning path, задача → файл |
| [AGENT_INDEX.md](AGENT_INDEX.md) | Маршрутизация задач по символам и тегам |
| [DOCUMENTATION_FINAL_AUDIT.md](DOCUMENTATION_FINAL_AUDIT.md) | Primary entry policy, критерии расширения, baseline проверки |
| [extended/INDEX.md](extended/INDEX.md) | Расширенное руководство (examples) |

Страница **ENCYCLOPEDIA.md** — формальный «паспорт» системы документов и обязательна в обходе (см. [VERIFY.md](VERIFY.md)).

## Аксиомы замыкания

Формально документация считается **замкнутой** одновременно по четырём условиям:

1. **Граф связности.** Каждый файл `docs/**/*.md` входит в орфанируемость «нет» —
   см. успешное выполнение [Verify-Docs.ps1](tools/Verify-Docs.ps1).

2. **Целость ссылок.** Нет внутренних ссылок на отсутствующие `.md` — проверка
   тем же [Verify-Docs.ps1](tools/Verify-Docs.ps1).

3. **Единый primary entry по теме.** Для разработки используется таблица primary
   entry в [DOCUMENTATION_FINAL_AUDIT.md](DOCUMENTATION_FINAL_AUDIT.md); старые
   «legacy» файлы указаны там как вторичный справочный слой.

4. **Согласованность с кодом.** Тексты привязаны к `Autogenerated/Src` и паттернам
   имён модулей; для контуров **вне** исходного кода — только boundary-обзоры
   (например [architecture/infrastructure-deployment-boundaries-overview.md](architecture/infrastructure-deployment-boundaries-overview.md)).

При несогласованности с любой аксиомой новую версию набора документов **не считают** целевой выпуск encyclopedia-билда.

| Артефакт | Человекочитаемый слой | Машинный / обновление |

## Изменение системы документов

1. Текст и ссылки — в одной или нескольких целевых страницах; при новой крупной
   подсистеме — отдельный dive по критериям из [DOCUMENTATION_FINAL_AUDIT.md](DOCUMENTATION_FINAL_AUDIT.md).

2. Каждая новая страница обязана стать **достижимой** по ссылкам от хабов (через
   обновление **INDEX** и/или **AGENT_INDEX** и по крайней мере одной цепочки от
   существующей страницы).

3. Перед фиксацией — **[VERIFY.md](VERIFY.md)** (ссылки + сироты + при необходимости
   markdownlint).

4. Навигация для агента в репозитории кода: при новом primary entry обновляйте
   **`.cursor/rules/bpmsoft-docs.mdc`**.
