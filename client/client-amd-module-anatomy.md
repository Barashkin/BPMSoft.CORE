# Client AMD Module Anatomy

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: AMD, define, require, entitySchemaName, mixins, methods -->

> Структура клиентского AMD-модуля: `define`, зависимости, schema object и границы ответственности.

## define

Классический клиентский модуль объявляется через `define`.

```javascript
define("WebhookV2Page", [
    "WebhookV2PageResources",
    "ServiceHelper",
    "ModalBox",
    "WebhookMetaItemViewModelMixin"
], function(resources, ServiceHelper, ModalBox) {
    return {
        entitySchemaName: "VwWebhookV2",
        mixins: {},
        attributes: {},
        methods: {},
        diff: []
    };
});
```

Порядок аргументов factory function должен соответствовать порядку зависимостей.

## require внутри логики

`require` используется для динамической загрузки, когда зависимость нужна не всегда.

```javascript
require(["UisCtiProviderResources"], function(module) {
    if (module) {
        scope.localizableStrings = module.localizableStrings;
    }
});
```

Применяйте динамический `require` точечно: для provider, optional module, lazy loading или тяжёлой библиотеки.

## entitySchemaName

`entitySchemaName` связывает модуль с серверной entity schema.

```javascript
return {
    entitySchemaName: "Metric"
};
```

Для page/detail/section это влияет на:

- загрузку колонок;
- ESQ для данных;
- binding полей;
- права на операции;
- lookup и detail filters.

## mixins

Миксины добавляют поведение без наследования.

```javascript
mixins: {
    webhookMetaItemViewModelMixin: "BPMSoft.WebhookMetaItemViewModelMixin",
    sortableGrid: "BPMSoft.SortableGridViewModelMixin"
}
```

Если метод приходит из миксина, сохраняйте `callParent`/`this.mixins.*`-цепочку по существующему паттерну модуля.

## attributes

Attributes описывают client view model state.

```javascript
attributes: {
    "NeedReloadData": {
        dataValueType: BPMSoft.DataValueType.BOOLEAN,
        type: BPMSoft.ViewModelColumnType.VIRTUAL_COLUMN,
        value: false
    }
}
```

Используйте virtual attributes для UI state, collection, flags и transient data.

## methods

Методы содержат lifecycle hooks, handlers и helper-логику.

```javascript
methods: {
    init: function() {
        this.callParent(arguments);
        this.sandbox.subscribe("UpdateArchivalButtonMenu", this.updateSection, this);
    }
}
```

В overridden methods почти всегда сначала вызывается `this.callParent(arguments)`, если локальный код не должен полностью заменить базовое поведение.

## SCHEMA markers

В autogenerated modules часто встречаются маркеры:

```javascript
details: /**SCHEMA_DETAILS*/{}/**SCHEMA_DETAILS*/,
diff: /**SCHEMA_DIFF*/[]/**SCHEMA_DIFF*/
```

Они помогают платформе и дизайнеру отделять generated content от ручных изменений.
