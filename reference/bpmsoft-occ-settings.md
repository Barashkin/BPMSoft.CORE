# Настройки OCC и Sender

<!-- Версия: 1.0 | Обновлено: 2026-04-23 | Платформа: BPMSoft 1.9 -->
<!-- Теги: OCC, Sender, SysSettings, feature flags, configuration, WhatsApp -->

> Справочник ключевых системных настроек и feature flags, которые реально используются в OCC-контуре по коду `Autogenerated/Src`.

## Обзор

Настройки OCC используются в трёх основных сценариях:

- конфигурация connector и каналов;
- поведение UI, мастера настройки и long-polling;
- интеграции WhatsApp и рассылок Sender.

Ниже перечислены настройки, найденные в коде OCC/Sender. Это не глобальный каталог всех SysSettings платформы, а практический список для работы с OCC.

## 1. Настройки connector и базовой OCC-конфигурации

| Настройка | Где используется | Назначение |
| ----- | ----- | ----- |
| `BPMSoftOCCOperatorAppId` | `BPMSoftOCCSetupService`, `BPMSoftOCCAddChannelService`, клиентские channel view | Идентификатор приложения/клиента OCC для connector и webhook URL |
| `BPMSoftOCCOperatorToken` | `BPMSoftOCCSetupService` | Токен интеграции OCC |
| `BPMSoftOCCTokenService` | `BPMSoftOCCSetupService` | Сервис/endpoint валидации токена на этапе настройки |
| `BPMSoftOCCServices` | `BPMSoftOCCAddChannelService`, `BPMSoftOCCAddMfmsJsonService` | Шаблон URL внешнего connector service |
| `BPMSoftOCCOperatoHost` | `BPMSoftOCCAddChannelService`, `BPMSoftOCCAddMfmsJsonService` | Базовый host BPMSoft для построения webhook URL |
| `BPMSoftOCCFileServer` | `BPMSoftOCCRouting` | Файловый сервер OCC для работы с файловыми вложениями |

## 2. Настройки мастера настройки OCC

| Настройка | Где используется | Назначение |
| ----- | ----- | ----- |
| `CurrentSettingStep` | `BPMSoftOCCSetupService`, `BPMSoftOCCMockupModule` | Текущий шаг wizard/мастера настройки OCC |
| `AreConfiguredChats` | `BPMSoftOCCSetupService`, `BPMSoftOCCMockupModule`, `BPMSoftOCCChat1Section`, `BPMSoftOCCChatConfiguration1Section` | Признак завершённой конфигурации OCC |

Особенности:

- `SetupNextStage()` увеличивает `CurrentSettingStep`;
- по достижении финального шага выставляется `AreConfiguredChats = true`;
- клиентский mockup/wizard читает эти значения через `BPMSoft.SysSettings.querySysSettingsItem(...)`.

## 3. Feature flags и поведение чата

| Настройка | Где используется | Назначение |
| ----- | ----- | ----- |
| `BPMSoftOCCMeesageLongPooling` | `BPMSoftOCCChatService`, `BPMSoftOCCChatSettings` | Включение long-polling доставки сообщений в OCC UI |
| `BPMSoftOCCClient` | `BPMSoftOCCChatSettings` | Служебная настройка/признак OCC-клиента |
| `BPMSoftOCCKnowledgeBaseLength` | `BPMSoftOCCChatSchema.js` | Ограничение/настройка UI-поиска по knowledge base |
| `MaxFileSize` | OCC client pages и user task property pages | Ограничение размера файлов в send file/image сценариях |

Примечание: `BPMSoftOCCChatSettings` - это отдельная OCC-сущность настроек, но часть поведения также продублирована через SysSettings и UI query.

## 4. Настройки каналов и внешних интеграций

| Настройка | Где используется | Назначение |
| ----- | ----- | ----- |
| `BPMSoftOCCFBLService` | `BPMSoftOCCFBChannelView`, `BPMSoftOCCFBMChannelView` | URL/endpoint интеграции Facebook |
| `BPMSoftOCCInstagramService` | `BPMSoftOCCInstagramChannelView` | URL/endpoint интеграции Instagram |

Во многих channel view дополнительно читается `BPMSoftOCCOperatorAppId`, который участвует в построении webhook URL или payload для connector.

Для channel-specific полей, endpoint и маппинга в `BPMSoftOCCChannel` см. отдельную матрицу [occ-channel-integrations.md](../server/occ-channel-integrations.md).

## 5. WhatsApp MFMS / Edna

| Настройка | Где используется | Назначение |
| ----- | ----- | ----- |
| `BPMSoftOCCUseObsoleteWhatsAppApi` | `BPMSoftOCCWhatsAppChannelView.BPMSoftOCCWAMfmsJson.js` | Переключение между `wa/edna` и legacy `wa/mfms/json` |
| `BPMSoftOCCOperatorAppId` | `BPMSoftOCCWhatsAppChannelView.BPMSoftOCCWAMfmsJson.js` | Используется при построении webhook URL |
| `BPMSoftOCCOperatoHost` | `BPMSoftOCCAddMfmsJsonService` | Базовый host BPMSoft для webhook |
| `BPMSoftOCCServices` | `BPMSoftOCCAddMfmsJsonService` | Connector service template |

Если `BPMSoftOCCUseObsoleteWhatsAppApi = true`, UI:

- скрывает `CascadeId`;
- показывает `Subject`;
- вызывает `AddObsoleteWhatsAppMfmsJsonChannel(...)`.

Если флаг выключен, используется современный сценарий `wa/edna`.

## 6. Sender и связанные константы

Для Sender часть логики сидит не в SysSettings, а в lookup-константах и GUID, доступных в JS-константах и C#:

| Константа/справочник | Где используется | Назначение |
| ----- | ----- | ----- |
| `BSConstants.Delivery.StartType` | `BPMSoftOCCDelivery1Page.BPMSoftSender.js` | Старт по команде / по расписанию |
| `BSConstants.Delivery.DeliveryType` | `BPMSoftOCCDelivery1Page.BPMSoftSender.js` | ChatBot / TextMessage |
| `BSConstants.Delivery.DeliveryContentType` | `BPMSoftOCCDelivery1Page.BPMSoftSender.js` | Text / Picture / File / комбинированные типы |
| `BSConstants.Delivery.Status` | `BPMSoftOCCDelivery1Page.BPMSoftSender.js` | `InWaiting`, `Running`, `Finished`, `ReadyForStart` |

См. файл: `Autogenerated/Src/BSConstants.BPMSoftSender.js`.

## 7. Что проверять в первую очередь

| Симптом | Что проверить |
| ----- | ----- |
| Канал не создаётся | `BPMSoftOCCServices`, `BPMSoftOCCOperatoHost`, `BPMSoftOCCOperatorAppId` |
| Wizard OCC не доходит до конца | `CurrentSettingStep`, `AreConfiguredChats` |
| Сообщения не приходят в UI | `BPMSoftOCCMeesageLongPooling` |
| WhatsApp webhook генерируется не тем способом | `BPMSoftOCCUseObsoleteWhatsAppApi` |
| Не работают вложения | `BPMSoftOCCFileServer`, `MaxFileSize` |

## 8. Практические замечания

- Для OCC важно различать SysSettings и lookup/константы сущностей: часть поведения задаётся настройками, часть - GUID справочников.
- В `BPMSoftOCCWAMfmsJson` режим legacy/new определяется только настройкой, поэтому её состояние критично для поддержки.
- В Sender-контурах статусы доставки часто живут как GUID-константы в коде и JS, а не как “настраиваемые” SysSettings.
