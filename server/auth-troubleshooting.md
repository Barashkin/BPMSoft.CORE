# Auth Troubleshooting

<!-- Версия: 1.1 | Обновлено: 2026-04-28 | Платформа: BPMSoft 1.9 -->
<!-- Теги: auth, LDAP, OAuth, SSO, troubleshooting -->

> Диагностика auth-проблем зависит от контура: LDAP sync, OAuth app/token,
> SSP registration/password recovery, SAML mapping или runtime login pipeline.

## LDAP не подключается

Проверьте:

1. `LDAPServer`, порт и `LDAPUseSecureConnection`.
2. `LDAPAuthType`.
3. `LDAPServerLogin`, `LDAPServerPassword`, `LDAPKeyDistributionCenter`.
4. Feature `LDAPUseSecureConnection`.
5. Лог `LDAP`.
6. Ошибки `LdapConnector.Authorize`: invalid credentials или connection error.

Если используется Active Directory, учитывайте retry/rebind поведение
`LdapConnector`: часть временных сетевых ошибок может быть повторена только для
поисковых запросов.

## LDAP sync не создает пользователей

Проверьте:

- `LDAPUsersEntry`, `LDAPUsersFilter`;
- `LDAPUserLoginAttribute`, `LDAPUserIdentityAttribute`;
- `LDAPEntryModifiedOnAttribute`;
- staging table `SysLDAPSynchUser`;
- `LdapSyncId`;
- `SyncWithLDAPProcessHelper`;
- блокировки `SemaphoreSlim`, если параллельно идет несколько sync;
- наличие лицензий, если включена раздача лицензий при LDAP sync.

## LDAP группы не попадают в роли

Проверьте:

- `LDAPGroupsEntry`, `LDAPGroupsFilter`, `LDAPUsersInGroupFilter`;
- `LDAPGroupNameAttribute`, `LDAPGroupIdentityAttribute`;
- `LDAPElement`;
- `LDAPUserInLDAPGroup`;
- операции insert/delete в `SysUserInRole`;
- feature `DeactivateExcludedLdapUsers`, если группы деактивируются.

## LDAP UI недоступен

Проверьте:

- operation `CanManageAdministration`;
- `LDAPServerSettings.LDAP.js`;
- `LDAPSysSettingsService`;
- маршруты сервиса и текущий `UserConnection`;
- System Designer tile `SystemDesigner.LDAP.js`.

## OAuth client не создается

Подробная диагностика OAuth20Integration вынесена в
[auth-oauth20-troubleshooting.md](auth-oauth20-troubleshooting.md).

Проверьте:

1. `GlobalAppSettings.FeatureEnableOAuth20Integration`.
2. Operation `CanManageSolution`.
3. Наличие ровно одного default `OAuthResource`.
4. Валидность `SystemUserId` (`SysAdminUnitTypeValue = 4`).
5. Ошибки `IdentityServiceWrapper.AddClient`.
6. `OAuthClientAppListener.OnInserting`.

## OAuth client обновился в базе, но не в identity service

Проверьте:

- `OAuthClientAppListener.OnUpdating`;
- `BaseOAuthApiCaller`;
- ошибки `ApiServerException` / `ApiServerConnectivityException`;
- доступность identity service;
- локализованные сообщения listener.

## OAuth tokens не сохраняются или устарели

Проверьте:

- `OAuthTokenStorage` по `SysUser` и `OAuthApp`;
- `AccessToken`, `RefreshToken`, `ExpiresOn`;
- cascade при удалении `OAuthApp`;
- client module `OAuthAuthenticationModule.NUI.js`, если сценарий связан с
  social account OAuth.

## SSP регистрация не работает

Проверьте:

- `GlobalAppSettings.ShowPortalSelfRegistrationLink`;
- `LoginReservationPeriod`;
- `SysAdminUnitNameReserve`;
- существующий `Contact` по email;
- существующий `SysAdminUnit` по `Contact`;
- `PortalRegistrationEmailTemplate`;
- mailbox `SSPRegistrationMailbox` или `SupportServiceEmail`;
- features `SSPRegistrationMailboxEnabled`, `UseEmailSenderForSelfReg`.

## Письмо восстановления пароля не приходит

Проверьте:

- лог `Authentication`;
- `TotpSecret` для пользователя;
- email в `SysAdminUnit`;
- `ITotpSetupTokenGenerator`;
- URL `/Login/TotpRecoverPassword.aspx`;
- feature `MultilanguageRegistrationEmail`;
- recovery email template;
- отправку через `RegistrationHelper`.

Наружу сервис может вернуть success даже при отсутствии email. Это ожидаемое
поведение против user enumeration.

## SAML маппинг не заполняет Contact

Проверьте:

- записи `SAMLFieldNameConverter`;
- `SAMLFieldName`;
- `ContactFieldName`;
- `ColumnDefaultValue`;
- наличие нужной колонки на `Contact`;
- `SamlResponse` по связанному `Contact`.

Если проблема в redirect/assertion/signature, это не слой
`SAMLFieldNameConverter`; смотрите runtime SSO configuration.

## Сессию нельзя завершить

Проверьте:

- запись `SysUserSession`;
- `SessionId`;
- operation `CanManageAdministration`;
- `AdministrationService.TerminateSession`;
- `IUserSessionManager.Expire`;
- корректность `recordId` или `userId`.

## Пароль не проходит валидацию

Проверьте:

- `AdministrationService.ValidatePassword`;
- `password.ValidatePassword(UserConnection, userName)`;
- возвращаемый текст `SecurityException`;
- политики паролей платформы;
- `ForceChangePassword` и `PasswordExpireDate` на `SysAdminUnit`.

## Не найден login page код

В `Autogenerated/Src` почти нет полноценной NUI login page. Если проблема в
cookie, auth ticket, login redirect или низкоуровневом pipeline, смотрите host,
web config и runtime платформы.
