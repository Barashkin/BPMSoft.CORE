# Process Designer DCM Boundary

<!-- Версия: 1.1 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->

> DCM designer находится рядом с BPM process designer и наследует общий
> process-designer base. Это полезный источник паттернов, но не прямой
> replacement для BPM property pages.
> Полный DCM/Dynamic Case dive см. в
> [dcm-dynamic-case-overview.md](dcm-dynamic-case-overview.md).

## DcmSchemaDesignerViewModel

Файл: `Autogenerated/Src/DcmSchemaDesignerViewModel.DcmDesigner.js`.

Класс:

- `extend: "BPMSoft.BaseProcessSchemaDesignerViewModel"`;
- alternate class `BPMSoft.Designers.DcmSchemaDesignerViewModel`;
- использует `BPMSoft.DcmSchemaManager`;
- задаёт storage keys для DCM schema metadata/resources;
- имеет `contextHelpCode = "DcmSchemaDesigner"`;
- управляет selection, stages и property pages.

## Что видно через DCM

DCM показывает общие designer-паттерны:

- загрузка property page при клике по пустой области;
- `loadElementProperties`;
- selection state;
- проверка удаления элементов;
- показ designer message box через `ProcessSchemaDesignerUtilities`;
- отметка элементов как validated на save.

## Чем DCM отличается от BPM

DCM работает со stages и DCM elements. BPM designer работает с process flow
elements, gateways, events и user tasks.

Не переносите DCM-specific storage keys и stage logic в BPM:

- `DcmSchemaLocationHref`;
- `DcmSchemaMetaData`;
- `DcmSchemaResources`;
- `DcmSchemaLoadedPropertiesStageColumnUId`;
- stage color actualization.

## DcmElementSchemaManager

Файл: `Autogenerated/Src/DcmElementSchemaManager.DcmDesigner.js`.

Полезные паттерны:

- наследование от `BaseProcessFlowElementSchemaManager`;
- feature flag для not implemented elements;
- отдельный `userTaskSchemaManagerName`;
- фильтрация доступных элементов.

## Validation removal

`DcmSubProcessPropertiesPage.DcmDesigner.js` использует
`ProcessSchemaDesignerUtilities.validateElementRemoval`.

Это пример общей designer utility, которую можно искать при ошибках удаления
элементов.

## Документировать как границу

DCM стоит использовать в документации как:

- доказательство, что base designer вынесен за пределы конкретного BPM UI;
- пример наследования от `BaseProcessSchemaDesignerViewModel`;
- источник общих паттернов property loading и validation.

Не стоит использовать DCM как:

- источник BPM element semantics;
- пример BPM user task parameters;
- пример process runtime behavior.
