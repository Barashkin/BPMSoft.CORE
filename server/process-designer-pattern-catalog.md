> Каталог практических UI-паттернов Process Designer из `Autogenerated/Src`.

## Открытие дизайнера через utility

Источник: `ProcessModuleUtilities.NUI.js`.

Паттерн:

- вызывающий код не строит URL сам;
- передаёт `schemaUId`;
- utility выбирает `process` или `processOld`;
- открывает `SchemaDesigner`.

Использовать для Process Library actions и переходов из связанных UI.

## Feature switch нового canvas

Источник: `ProcessModuleUtilities.NUI.js`.

`UseProcessDiagramComponent` выбирает:

- `process` vs `processOld`;
- `processLog` vs `processLogOld`.

Это позволяет держать backward-compatible вход без дублирования вызывающего UI.

## Property page как ConfigurationModule

Источник: `ProcessSchemaElementPropertiesEdit.ProcessDesigner.js`.

Паттерн:

- `extend: BPMSoft.ConfigurationModule`;
- `useHistoryState: false`;
- custom container suffix;
- PTP rerender message;
- mask на время рендера.

## Базовая property page + leaf pages

Источник: `BaseProcessSchemaElementPropertiesPage.ProcessDesigner.js`.

Паттерн:

- общие attributes и validation в base;
- element-specific поля в leaf pages;
- mixins для editable/observable/mapping поведения;
- validation results сохраняются в process element.

## Flow element messages для parameters

Источник: `ProcessFlowElementPropertiesPage.ProcessDesigner.js`.

Паттерн:

- одна page владеет `Parameters`;
- вложенные editors общаются через PTP messages;
- save/discard разделены;
- parameter edit получает данные через `GetParameterEditInfo`.

## SourceCodeEditPage как вложенный editor

Источники:

- `ProcessSchemaPropertiesPage.ProcessDesigner.js`;
- `RootUserTaskPropertiesPage.ProcessDesigner.js`.

Паттерн:

- source editor подключается как module;
- `useHistoryState: false`;
- `Tag` указывает редактируемое поле;
- данные идут через `GetSourceCodeData` / `SourceCodeChanged`.

## User task schema manager

Источник: `UserTaskPropertiesPage.ProcessDesigner.js`.

Паттерн:

- `schemaManagerName = "ProcessUserTaskSchemaManager"`;
- schema загружается по `schemaUId`;
- список схем фильтруется через `SysProcessUserTask`;
- открыть schema designer можно из property page.

## Mapping module specialization

Источник: `ProcessParametersMappingModule.ProcessDesigner.js`.

Паттерн:

- конкретный module наследует base edit module;
- задаёт только `schemaName`;
- state и lifecycle остаются в base.

## Modal save intent

Источник: `SaveSchemaVersionMessageBox.ProcessDesigner.js`.

Паттерн:

- modal не выполняет сохранение сам;
- публикует sandbox message родителю;
- родитель решает, как сохранить version.

## DCM как расширение base designer

Источник: `DcmSchemaDesignerViewModel.DcmDesigner.js`.

Паттерн:

- concrete designer наследует `BaseProcessSchemaDesignerViewModel`;
- задаёт schema manager и storage keys;
- переопределяет selection/property behavior.

Использовать как пример общей архитектуры designer, но не как BPM semantic
reference.

## Validation into element

Источник: `BaseProcessSchemaElementPropertiesPage.ProcessDesigner.js`.

Паттерн:

- base page собирает validation из element, columns и custom validators;
- складывает всё в `processElement.validationResults`;
- canvas может показать invalid state на элементе.
