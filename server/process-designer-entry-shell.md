# Process Designer Entry Shell

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: ProcessModuleUtilities, ProcessLibrary, SchemaDesigner, UseProcessDiagramComponent -->

> Entry shell описывает, как пользователь из библиотеки процессов попадает в
> `SchemaDesigner` и какие feature flags влияют на тип дизайнера.

## ProcessModuleUtilities

Файл: `Autogenerated/Src/ProcessModuleUtilities.NUI.js`.

Ключевые service constants:

| Constant | Service |
| -------- | ------- |
| `PROCESS_ENGINE_SERVICE_NAME` | `ServiceModel/ProcessEngineService.svc` |
| `PROCESS_SCHEMA_MANAGER_SERVICE_NAME` | `ServiceModel/ProcessSchemaManagerService.svc` |
| `SCHEMA_MANAGER_SERVICE_NAME` | `ServiceModel/SchemaManagerService.svc` |
| `WORKSPACE_EXPLORER_SERVICE_NAME` | `ServiceModel/WorkspaceExplorerService.svc` |
| `APP_INSTALLER_SERVICE_NAME` | `ServiceModel/AppInstallerService.svc` |

`REQUEST_TIMEOUT` очень большой, потому что операции дизайнера/публикации могут
быть долгими.

## Открытие дизайнера процесса

Метод `showProcessSchemaDesigner(schemaUId)` выбирает тип по feature:

```text
UseProcessDiagramComponent = true  -> type = process
UseProcessDiagramComponent = false -> type = processOld
```

Затем вызывает:

```text
showSchemaDesigner({
  designer: "SchemaDesigner",
  type: type,
  schemaUId: schemaUId
})
```

Для логов процессов есть похожие методы:

- `showProcessDiagram` -> `processLog` / `processLogOld`;
- `showProcessHeatMap` -> `processLog` / `processLogOld` + `heatMap=true`;
- `showDcmDiagram` -> `dcmLog`.

## URL и window

`showSchemaDesignerWindow` получает URL через `getSchemaDesignerUrl` и открывает
его через `window.open`.

Если popup заблокирован браузером, используется timeout-проверка окна.

## Process Library section

Файл: `Autogenerated/Src/VwProcessLibSection.ProcessLibrary.js`.

Раздел библиотеки процессов:

- entity schema `VwProcessLib`;
- operation right `CanManageProcessDesign`;
- умеет запускать выбранные процессы;
- открывает процесс в designer;
- копирует процесс и после копирования открывает designer;
- проверяет foreign lock перед редактированием;
- публикует событие изменения активной версии.

Ключевой вызов после копирования:

```text
BPMSoft.ProcessModuleUtilities.showProcessSchemaDesigner(schemaUId)
```

## Process properties card

`openProcessProperties` в Process Library открывает карточку свойств процесса,
а не canvas designer. Это разные UI-сценарии:

- card/page для записи библиотеки процесса;
- `SchemaDesigner` для графа процесса.

## Demo mode

`ProcessModuleUtilities` читает SysSetting `ShowDemoLinks` и выставляет
`isDemoBuild`. Некоторые операции, например publish, могут быть заблокированы
в demo mode.

## Практические правила

- Для открытия BPM designer используйте `showProcessSchemaDesigner`, а не
  вручную собранный URL.
- При проблемах с новым/старым canvas первым делом проверяйте
  `UseProcessDiagramComponent`.
- При недоступности действий в библиотеке проверяйте `CanManageProcessDesign`.
- При ошибках редактирования версии проверяйте locks и active version state.
