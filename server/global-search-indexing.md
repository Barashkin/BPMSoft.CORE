# Global Search Indexing

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: GlobalSearch, indexing, ConfigurationBuildWatcher, Quartz -->

> Индексация Global Search отделена от чтения результатов. Конфигурации
> индексируемых сущностей отправляются через `IndexingConfigService`, а
> изменения записей передаются через indexer/event listener слой.

## App start и Quartz

Файл: `Autogenerated/Src/GlobalSearchEventListener.GlobalSearch.cs`.

На старте приложения `GlobalSearchEventListener`:

1. Инициализирует DI для query builder и search provider.
2. Проверяет feature `GlobalSearch`.
3. Удаляет старые jobs группы `GlobalSearchGroup`.
4. Запускает `LogstashConfigsJob` через `AppScheduler.TriggerJob`.

`LogstashConfigsJob` связан с генерацией и отправкой конфигураций для
индексации. Для поведения real-time indexing используется feature
`GlobalSearch_V2`.

## IndexingConfigService

Файл: `Autogenerated/Src/IndexingConfigService.GlobalSearch.cs`.

Сервис содержит две операции:

| Метод | Назначение |
| ----- | ---------- |
| `SendIndexationConfigs` | отправляет конфиги индексации |
| `GetIndexationConfigs` | возвращает список `IndexingEntity` |

Сервис работает от `AppConnection.SystemUserConnection`, потому что конфиги
индексации должны собираться системно, а не от прав текущего пользователя.

## Build watcher

Файл: `Autogenerated/Src/ConfigurationBuildWatcher.GlobalSearch.js`.

Клиентский watcher подписан на `BPMSoft.ServerChannel` и вызывает
`IndexingConfigService.SendIndexationConfigs`, когда приходит:

- `ConfigurationBuildCompleted`;
- `SysSettingsChanged` для `GlobalSearchLookupFilterEnabled`;
- `SysSettingsChanged` для `GlobalSearchIndexedDataConfig`.

Этот механизм позволяет переслать конфигурацию индексации после сборки
конфигурации или изменения важных настроек поиска.

## Состав индексируемых сущностей

За сбор состава отвечают:

- `IndexingEntityListManager.GlobalSearch.cs`;
- `IndexingEntityListBuilder.GlobalSearch.cs`;
- `IndexingDetailListBuilder.GlobalSearch.cs`;
- `ConfigurableIndexingEntities.GlobalSearch.cs`;
- `IndexedDataConfig.GlobalSearch.cs`;
- `IgnoredEntityList.GlobalSearch.cs`.

`IndexedDataConfig` связан с настройкой `GlobalSearchIndexedDataConfig` и
позволяет описывать секции, сущности и игнорируемые колонки.

## SysModule.GlobalSearchAvailable

Файлы:

- `Autogenerated/Src/SysModuleSchema.Base.cs`;
- `Autogenerated/Src/GlobalSearchSysModuleListener.GlobalSearch.cs`.

Колонка `SysModule.GlobalSearchAvailable` управляет доступностью секции для
глобального поиска. `GlobalSearchSysModuleListener` слушает вставку,
изменение и удаление `SysModule`; если изменена колонка
`GlobalSearchAvailable`, очищается кэш `AvailableIndexingEntities`.

## Real-time indexing

Файл: `Autogenerated/Src/BaseIndexer.GlobalSearch.cs`.

`BaseIndexer.IndexEntity` проверяет feature `GlobalSearch_V2`. Если feature
выключена, entity не отправляется на индексацию. Если включена, indexer:

1. Собирает payload через `IndexingRequestDataBuilder`.
2. Передает его в `IndexingEntitySender`.
3. Указывает operation type: create/update/delete.

Этот слой обычно вызывается из entity listeners Global Search.
