# Security Licensing Operations Matrix

<!-- Версия: 1.0 | Обновлено: 2026-04-28 | Платформа: BPMSoft 1.9 -->
<!-- Теги: security, rights, licenses, operations, matrix, SysAdminOperation -->

> Прикладная матрица безопасности: operation rights, schema/record rights,
> лицензии, portal access и клиентские проверки в одном месте.

## Матрица проверок

| Слой | Где хранится | API/проверка | Что защищает |
| ---- | ------------ | ------------ | ------------ |
| Operation right | `SysAdminOperation`, `SysAdminOperationGrantee` | `DBSecurityEngine.CheckCanExecuteOperation` | действия и админские операции |
| Schema operation right | `SysEntitySchemaOperationRight` | `RightsHelper`, `EntitySchemaManager`, NUI services | CRUD на уровне схемы |
| Record right | record rights tables/runtime ACL | `GetSchemaRecordRightLevel`, entity rights checks | конкретные записи |
| Column right | column rights metadata | schema/column access checks | отдельные колонки |
| License | `SysLic`, `SysLicPackage`, runtime license APIs | `LicHelper`, `LicManager` | право использовать функциональность |
| Portal access | SSP allowed schemas/columns | SSP services and portal ACL helpers | внешний portal user surface |
| Client UX | no authoritative storage | `RightUtilities` | видимость кнопок/секций |

Клиентские проверки нужны для UX, но не являются границей безопасности.

## Operation rights

Типовой серверный паттерн:

```text
UserConnection.DBSecurityEngine.CheckCanExecuteOperation("CanManageAdministration")
```

Для ветвления без исключения используется `GetCanExecuteOperation`. Несколько
операций могут формировать один сценарий доступа, например read через
`CanViewConfiguration` или `CanManageSysSettings`.

Source files:

- `SysAdminOperationSchema.Base.cs`;
- `SysAdminOperationGranteeSchema.Base.cs`;
- `SysAdminOperationPageV2.UIv2.js`;
- `SysAdminOperationSectionV2.UIv2.js`.

## Schema/record rights and licenses

`RightsService.NUI.cs` комбинирует права и лицензии:

- schema operation rights;
- record operation rights;
- `LicHelper.GetSchemaLicRights`;
- разные сообщения для "нет права" и "нет лицензии".

Это важно для UX: одинаковый запрет может быть вызван разными причинами.

## License boundary

`LicHelper` и `LicManager` используются как runtime API платформы. Их реализации
не лежат в `Autogenerated/Src`, поэтому в этом workspace обычно видны только
вызовы и entity metadata `SysLic*`.

Документируйте это явно, чтобы не искать реализацию лицензирования среди
сгенерированных схем.

## Новая админ-операция

Минимальный чеклист:

1. Завести `SysAdminOperation` с понятным code.
2. Выдать доступ через `SysAdminOperationGrantee` или UI раздела операций.
3. На сервере вызвать `CheckCanExecuteOperation` до изменения данных.
4. На клиенте использовать `RightUtilities` только для видимости/доступности UI.
5. Если функциональность лицензируется, проверить license API отдельно.
6. Добавить troubleshooting: чем отличается отсутствие operation right от
   отсутствия лицензии.

## Troubleshooting matrix

| Симптом | Вероятный слой |
| ------- | -------------- |
| Кнопка скрыта, но прямой service call проходит | client UX check differs from server check |
| Service возвращает security exception | operation/schema/record right |
| Права выданы, но функциональность недоступна | license missing |
| В портале не видна схема/колонка | SSP ACL, not regular section right |
| Excel/export/grid action недоступна | operation right plus NUI-specific feature/right |
