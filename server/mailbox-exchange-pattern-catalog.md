# Mailbox Exchange Pattern Catalog

<!-- Версия: 1.0 | Обновлено: 2026-04-28 | Платформа: BPMSoft 1.9 -->
<!-- Теги: MailboxSyncSettings, Exchange, patterns, Quartz, listener -->

> Каталог рабочих паттернов mailbox sync, Exchange listener и calendar
> integration из `Autogenerated/Src`.

## Layered schema extension

**Файлы:**

- `MailboxSyncSettingsSchema.Base.cs`;
- `MailboxSyncSettingsSchema.Exchange.cs`;
- `MailboxSyncSettingsSchema.ExchangeListener.cs`.

**Паттерн:** базовая entity хранит общие mailbox поля, Exchange layer добавляет
intervals и auto-sync flags, listener layer добавляет event/listener-specific
поведение.

## Secure mailbox credentials

**Файл:** `MailboxSyncSettingsSchema.Base.cs`.

`UserPassword` хранится как `SecureText`, а OAuth flow хранит ссылку на
`OAuthTokenStorage`. Не переносите credentials в client-side code, logs или
документацию.

## Domain service as job orchestrator

**Файл:** `MailboxSettingsService.IntegrationV2.cs`.

Service принимает UI request, проверяет owner mailbox и решает, какой scheduler
использовать:

```text
Exchange mailbox -> ISyncJobScheduler
IMAP mailbox -> IImapSyncJobScheduler
```

## Self-rescheduling recovery job

**Файл:** `MailSyncJob.MailSync.cs`.

Job сравнивает `MailReSynchronizationFrequency` с initial frequency из
parameters и при изменении перерегистрирует себя.

## Listener endpoint delegates to domain processor

**Файл:** `ExchangeListenerService.IntegrationV2.cs`.

WCF endpoint не содержит бизнес-логику, а делегирует в processor из
`ClassFactory`:

- `IExchangeEventsProcessor`;
- `IEmailEventsProcessor`;
- `IMailboxStateEventsProcessor`;
- `IMailboxEventsProcessor`.

## Listener manager facade

**Файл:** `ExchangeEventsService.IntegrationV2.cs`.

Start/stop/recreate operations получают manager через
`IListenerManagerFactory.GetExchangeListenerManager(UserConnection)`.

## Office365 post-auth provisioning

**Файл:** `BaseOffice365OAuthAuthenticator.Exchange.cs`.

После успешной OAuth-аутентификации транзакционно создаются mailbox settings,
contact communication, contact sync settings и activity sync settings.

## Client mixin service flow

**Файл:** `MeetingInvitationsMixin.IntegrationV2.js`.

Mixin скрывает сложный service flow за методами:

- `canUserChangeMeeting`;
- `initMeetingInvitationInfo`;
- `sendInvitations`;
- `confirmInvitationsSending`.

## Чего не делать

- Не смешивать mailbox sync с Email Mining: mining начинается после создания
  email `Activity`.
- Не редактировать Quartz tables напрямую, если есть `ISyncJobScheduler`.
- Не считать `IntegrationV2` частью Integration Tools.
- Не логировать passwords, access tokens и full email payload без необходимости.
- Не разбирать generated process byte arrays: документируйте process parameters
  и entry points.
