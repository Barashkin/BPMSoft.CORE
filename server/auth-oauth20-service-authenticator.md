# Auth OAuth20 Service Authenticator

<!-- Версия: 1.0 | Обновлено: 2026-04-28 | Платформа: BPMSoft 1.9 -->
<!-- Теги: OAuth20Integration, ServiceOAuthAuthenticatorEndpoint, OAuthApplications -->

> Runtime/UI flow авторизации пользователя во внешнем OAuth provider для
> `OAuthApplications`.

## Назначение

`ServiceOAuthAuthenticatorEndpoint` обслуживает OAuth authorization code flow
для integration apps. В Autogenerated/Src явно присутствует client helper
`ServiceOAuthAuthenticatorEndpointHelper.OAuth20Integration.js`, который
показывает публичный контракт endpoint.

## Helper endpoints

`ServiceOAuthAuthenticatorEndpointHelper` строит URL:

| Helper method | Endpoint |
| ------------- | -------- |
| `getOAuthServiceClientSecretKey` | `/ServiceModel/ServiceOAuthAuthenticatorEndpoint.svc/GetOAuthClientSecret/{OAuthAppId}` |
| `getAuthorizationGrantUrl` | `/ServiceModel/ServiceOAuthAuthenticatorEndpoint.svc/GetAuthorizationGrantUrl/{appId}` |
| `removeOAuthUser` | `/ServiceModel/ServiceOAuthAuthenticatorEndpoint.svc/RemoveOAuthUser` |
| `getOAuthServiceAuthorizationCodeRedirectUrl` | `/ServiceModel/ServiceOAuthAuthenticatorEndpoint.svc/AuthorizationCodeRedirectHandler` |

`RemoveOAuthUser` получает JSON:

```json
{
  "SysUserId": "user-id",
  "OAuthAppId": "app-id"
}
```

## Application validation

`validateApplication` проверяет `OAuthApplications` через ESQ:

- `UseSharedUser`;
- `SharedUser`;
- length of `ClientId`;
- length of `Name`;
- length of `AuthorizeUrl`;
- length of `TokenUrl`;
- наличие secret через `getOAuthServiceClientSecretKey`.

Для shared mode требуется заполненный `SharedUser`.

## Authorization grant URL flow

```text
OAuth20AppPage._loginUser
  -> helper.getAuthorizationGrantUrl(appId)
  -> GET ServiceOAuthAuthenticatorEndpoint/GetAuthorizationGrantUrl/{appId}
  -> window.open(url)
  -> provider redirects to AuthorizationCodeRedirectHandler
  -> endpoint processes authorization code
  -> stores/updates OAuthTokenStorage
  -> sends ServerChannel message
```

Client слушает `BPMSoft.ServerChannel` и принимает только сообщения, у которых:

```text
message.Header.Sender === "ServiceOAuthAuthenticatorEndpoint"
```

## Callback result handling

`onLoggedIn` разбирает JSON `message.Body`:

| Result | UI behavior |
| ------ | ----------- |
| `success = false` | показывает сообщение `UnableToProcessAuthorizationCode` |
| `success = true`, `message` не пустой | показывает предупреждение `RefreshTokenNotFound` |
| `success = true` | вызывает `onSuccess` |

Для `OAuth20AppPage` успешный callback назначает shared user и обновляет
details.

## Связь с token storage

Endpoint сохраняет результат авторизации в `OAuthTokenStorage`, а page
показывает пользователей через `VwOAuthAppUser`. UI не должен напрямую писать
tokens.
