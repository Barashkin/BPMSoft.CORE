# Administration Configuration Pattern Catalog

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: administration, configuration, patterns, SysAdminUnit, SysSettings -->

> Каталог рабочих паттернов Administration / Configuration с точками входа в
> `Autogenerated/Src`.

## Admin WCF service

**Когда использовать:** нужна операция администрирования из UI.

**Паттерн:**

- `ServiceContract`;
- `WebInvoke POST JSON`;
- `UserConnection` из session;
- проверка `CanManageAdministration`;
- делегирование в manager/helper.

**Пример:** `AdministrationService.UIv2.cs`.

## Partial service by domain

**Когда использовать:** service разрастается по нескольким зонам.

**Паттерн:** общий `AdministrationService`, отдельные partial-файлы для users,
licenses, roles, granted rights.

**Примеры:**

- `AdministrationServiceUsers.UIv2.cs`;
- `AdministrationServiceLicenses.UIv2.cs`;
- `AdministrationServiceSysAdminUnitRoles.UIv2.cs`.

## Org structure hierarchy

**Когда использовать:** нужно получить дочерние роли/подразделения.

**Паттерн:** `Select` по `SysAdminUnit` + `HierarchicalSelectOptions`.

**Пример:** `AdministrationService.GetChildAdminUnits`.

## SysSettings increment

**Когда использовать:** нужен номер документа/заявки по настройке.

**Паттерн:** `CoreSysSettings.GetDefValue` → increment →
`CoreSysSettings.SetDefValue`; при `UseDBSequence` использовать `SequenceMap`.

**Пример:** `SysSettingsService.NUI.cs`.

## Runtime settings script

**Когда использовать:** настройка должна влиять на клиент с момента логина.

**Паттерн:** `ConfigurationSectionHelper` читает `CoreSysSettings` и добавляет
значение в стартовый JS.

**Примеры:** request cache, min search chars, colors, logo hash.

## Lookup metadata

**Когда использовать:** нужно описать справочник как системный объект.

**Паттерн:** `SysLookup` хранит entity schema, edit page, grid page, folder,
`IsSystem`, `IsSimple`.

**Пример:** `SysLookupSchema.Base.cs`.

## Workplace visibility

**Когда использовать:** секция не видна пользователю.

**Паттерн:** проверить `SysWorkplace`, `SysModuleInWorkplace`,
`SysAdminUnitInWorkplace`, позиции и client type.

**Пример:** `SysWorkplaceSchema.NUI.cs`.

## Section descriptor loading

**Когда использовать:** нужно понять, почему секция не открывается.

**Паттерн:** `ConfigurationSectionHelper` выбирает `SysModule`, связывает
`SysModuleEdit`, `SysModuleEntity`, LCZ-схемы и workspace links.

**Пример:** `ConfigurationSectionHelper.NUI.cs`.

## Package metadata

**Когда использовать:** нужно анализировать пакет или перенос конфигурации.

**Паттерн:** `SysPackage` + hierarchy/dependency views +
`SysPackageInstallType`.

**Пример:** `SysPackageSchema.Base.cs`.

## Feature state

**Когда использовать:** включить функцию пользователю или группе.

**Паттерн:** `FeatureService` + `AdminUnitFeatureState` + actual state для
current user/group.

**Пример:** `FeatureService.Base.cs`.
