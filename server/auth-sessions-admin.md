# Auth Sessions And Admin Operations

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: auth, sessions, SysUserSession, AdministrationService -->

> Сессии и административные auth-операции в Base описаны через
> `SysUserSession`, `AdministrationService`, password validation и
> `UserConnection` из HTTP session.

## SysUserSession

Файл: `Autogenerated/Src/SysUserSessionSchema.Base.cs`.

Ключевые колонки:

| Колонка | Назначение |
| ------- | ---------- |
| `SysUser` | пользователь |
| `SessionId` | идентификатор runtime-сессии |
| `SessionStartDate` | начало сессии |
| `SessionEndDate` | завершение сессии |
| `SessionEndMethod` | способ завершения |
| `ClientIP` | IP клиента |
| `Agent` | user agent |

Индексы:

- `ISysUserSessEndDateId`;
- `ISysUserSessionUserId`.

Они оптимизируют поиск активных/завершенных сессий по пользователю и end date.

## AdministrationService session endpoints

Файл: `Autogenerated/Src/AdministrationService.UIv2.cs`.

`AdministrationService` доступен через:

- `[DefaultServiceRoute]`;
- `[SspServiceRoute]`.

`UserConnection` берется из:

```text
HttpContext.Current.Session["UserConnection"]
```

Ключевые методы:

| Метод | Назначение | Проверка |
| ----- | ---------- | -------- |
| `TerminateSession` | завершить сессию по record id | `CanManageAdministration` |
| `TerminateCurrentUserSessions` | завершить все сессии текущего пользователя | текущий пользователь |
| `TerminateUserSessions` | завершить все сессии указанного пользователя | `CanManageAdministration` |
| `ValidatePassword` | проверить пароль по правилам платформы | `password.ValidatePassword` |

Завершение выполняется через `IUserSessionManager.Expire`.

## Password validation

`ValidatePassword` вызывает:

```text
password.ValidatePassword(UserConnection, userName)
```

При `SecurityException` метод возвращает текст ошибки валидации, а не падает
500. Это позволяет UI показать причину пользователю.

## UserConnection boundary

Для admin endpoints используется текущий `UserConnection` из HTTP session.
Для self-registration и recovery password используется `SystemUserConnection`,
потому что пользователь еще не авторизован или не имеет полноценной сессии.

Практическое разделение:

| Контекст | Когда использовать |
| -------- | ------------------ |
| `UserConnection` | авторизованный пользователь, проверка прав текущей сессии |
| `SystemUserConnection` | self-service операции до логина, системные lookup/update |
| `UseAdminRights = false` | когда нужно явно учитывать права текущего пользователя |

## ExternalAccess sessions

Есть расширение:

- `SysUserSessionSchema.ExternalAccess.cs`.

Его стоит смотреть при диагностике внешнего доступа/API-сценариев, где сессии
могут отличаться от обычного UI login flow.

## Что вне этого документа

Низкоуровневые cookie-флаги, FormsAuthentication и выдача auth ticket не видны
в `Autogenerated/Src`. Для них нужно смотреть host/runtime конфигурацию.
