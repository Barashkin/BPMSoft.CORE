# ESQ Columns and Paths

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: ESQ, AddColumn, path, lookup, alias, GetTypedColumnValue -->

> Документ о колонках `EntitySchemaQuery`: обычные колонки, lookup path, bracket path, alias и чтение значений через `GetTypedColumnValue`.

## Базовый `AddColumn`

`AddColumn(...)` добавляет колонку в выборку и возвращает `EntitySchemaQueryColumn`.
Возвращённый объект лучше сохранять, если значение потом читается из `Entity`.

```csharp
var esq = new EntitySchemaQuery(userConnection.EntitySchemaManager, "AccountCommunication");
EntitySchemaQueryColumn accountIdColumn = esq.AddColumn("=Account.Id");
var entities = esq.GetEntityCollection(userConnection);
Guid accountId = entities[0].GetTypedColumnValue<Guid>(accountIdColumn.Name);
```

Такой подход используется в `AccountSearcher.EmailMining.cs`.

## Почему важно хранить alias

Для простых колонок часто работает чтение по имени:

```csharp
string name = entity.GetTypedColumnValue<string>("Name");
```

Но для path, функций и агрегатов итоговое имя колонки может отличаться от исходного path.
Надёжнее читать по `column.Name`:

```csharp
var nickName = esq.AddColumn("[BPMSoftOCCOperatorUnit:SysAdminUnit:Id].Nickname");
string value = entity.GetTypedColumnValue<string>(nickName.Name);
```

## Lookup path

Lookup path проходит от текущей схемы к связанной.

Примеры:

- `Account.Name`
- `Contact.Id`
- `Operator.Id`
- `Channel.Type.Id`
- `Delivery.StartType.Id`

В OCC встречается типовой паттерн:

```csharp
esq.AddColumn("Operator.Id");
esq.Filters.Add(esq.CreateFilterWithParameters(
    FilterComparisonType.Equal, "Operator.Id", operatorId));
```

## Primary query column

`PrimaryQueryColumn` управляет выборкой primary key.

Варианты:

```csharp
esq.PrimaryQueryColumn.IsVisible = false;
esq.PrimaryQueryColumn.IsAlwaysSelect = true;
```

Практический смысл:

- `IsVisible = false` убирает primary column из обычного результата, если нужна компактная проекция;
- `IsAlwaysSelect = true` гарантирует наличие primary key, даже когда выбираются специфические колонки.

Примеры:

- `AccountSearcher.EmailMining.cs`
- `BPMSoftOCCApi.BPMSoftOCC.cs`
- `BaseEntityWithPositionSchema.Base.cs`

## Bracket path

Bracket path описывает обратную или нестандартную связь:

```text
[RelatedSchema:RelatedColumn:CurrentColumn].TargetColumn
```

Примеры:

- `[BPMSoftOCCOperatorUnit:SysAdminUnit:Id].Nickname`
- `[SysAdminUnit:Id:User].Contact.Id`
- `[BPMSoftOCCChatMessage:Chat:Id].CreatedOn`

Используйте bracket path, когда обычного `Lookup.Column` недостаточно.

## `=` и `>` в path

В автогенерированном коде встречаются специальные path-варианты:

- `=Account.Id`
- `>AdditionalParam.ColumnCaption`
- `=MainParam.[SysModuleEntity:SysEntitySchemaUId:SubjectSchemaUId].TypeColumnUId`

Их стоит рассматривать как расширенную path-нотацию платформы. Документируйте такие выражения рядом с кодом: без контекста они плохо читаются.

## Агрегатные колонки

ESQ может добавлять колонки-функции:

```csharp
var maxColumn = esq.AddColumn(
    esq.CreateAggregationFunction(AggregationTypeStrict.Max, "Position"));
var countColumn = esq.AddColumn(
    esq.CreateAggregationFunction(AggregationTypeStrict.Count, entity.Schema.GetDBPrimaryColumnName()));
```

Чтение обязательно делать по alias:

```csharp
int count = entities[0].GetTypedColumnValue<int>(countColumn.Name);
int max = entities[0].GetTypedColumnValue<int>(maxColumn.Name);
```

Reference: `BaseEntityWithPositionSchema.Base.cs`.

## Forced aliases

В GlobalSearch используется продвинутый паттерн:

```csharp
EntitySchemaQueryColumn esqColumn = esq.AddColumn(columnName);
esqColumn.SetForcedQueryColumnValueAlias(alias);
```

Это нужно, когда результат ESQ дальше превращается в SQL text или внешний индекс, где имена колонок должны быть стабильными.

Reference: `IndexingEntityListBuilder.GlobalSearch.cs`.

## Сортировка через колонку

Сортировка навешивается на колонку:

```csharp
var primaryDisplayColumn = esq.AddColumn(primaryDisplayColumnName).OrderByAsc();
primaryDisplayColumn.OrderPosition = 0;
```

Для стабильного результата задавайте `OrderPosition`, если сортировок несколько.

## `AddAllSchemaColumns`

`AddAllSchemaColumns()` добавляет все колонки схемы.
Это удобно для generic UI/diagram loading, но в продуктивном коде часто дорого.

Допустимые случаи:

- универсальный loader, который должен восстановить всю сущность;
- generic diagram/config механизм;
- legacy-код, где неизвестен набор нужных колонок.

Обычный сервисный код должен предпочитать явные `AddColumn(...)`.
