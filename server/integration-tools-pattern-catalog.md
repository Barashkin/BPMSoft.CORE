> Каталог рабочих паттернов Webhook V2 и Web Service V2 из `Autogenerated/Src`.

## Designer shell with hash rewrite

**Когда использовать:** нужен отдельный дизайнер, который открывает schema page
через `CardModuleV2`.

**Файлы:**

- `WebhookDesigner.Webhook.js`;
- `WebServicesDesigner.ServiceDesigner.js`.

**Паттерн:**

```text
BaseViewModule
  -> GetHistoryState
  -> parse domain-specific hash
  -> ReplaceHistoryState(CardModuleV2/...)
```

## View schema over workspace schema

**Когда использовать:** metadata schema должна отображаться как раздел/карточка.

**Файлы:**

- `VwWebhookV2Schema.Webhook.cs`;
- `VwWebServiceV2Schema.ServiceDesigner.cs`.

**Паттерн:** наследование от `VwSysSchemaInWorkspaceSchema`, `IsDBView = true`,
folder/tag helper schemas, embedded events process.

## Meta item view model mixin

**Когда использовать:** карточка редактирует manager item/schema metadata, а не
обычную прикладную entity.

**Файлы:**

- `WebhookMetaItemViewModelMixin.Webhook.js`;
- `ServiceMetaItemViewModelMixin.ServiceDesigner.js`;
- `WebhookV2Page.Webhook.js`;
- `WebServiceV2Page.ServiceDesigner.js`.

## Auth settings as nested module

**Когда использовать:** auth block должен переиспользоваться внутри schema page.

**Файлы:**

- `WebhookAuthInfoSettingsPage.Webhook.js`;
- `ServiceAuthInfoSettingsPage.ServiceDesigner.js`.

**Паттерн:** parent page передаёт schema UId в nested `ConfigurationModuleV2`,
а nested page получает schema через manager и редактирует `authInfo`.

## Feature-gated auth types

**Когда использовать:** тип авторизации должен быть доступен только при feature
flag.

**Файл:** `ServiceAuthInfoSettingsPage.ServiceDesigner.js`.

Digest показывается при `WebServiceDigestAuth`, OAuth20 - при
`WebServiceOAuth20Auth`.

## Builder/converter pipeline

**Когда использовать:** metadata request/response нужно собрать из разных
представлений: URI, curl, raw, JSON.

**Файлы:**

- `JsonRequestBuilder.ServiceDesigner.js`;
- `RawRequestBuilder.ServiceDesigner.js`;
- `CurlRequestBuilder.ServiceDesigner.js`;
- `UriJsonConverter.ServiceDesigner.js`;
- `JsonWebhookRequestBuilder.Webhook.js`;
- `JsonWebhookResponseBuilder.Webhook.js`.

## Service schema client facade

**Когда использовать:** настроенную Web Service V2 schema нужно выполнить из UI
или server code.

**Файлы:**

- `CallServiceSchemaService.ServiceDesigner.cs`;
- `ServiceSchemaClient.ServiceDesigner.cs`;
- `ServiceSchemaParameterBuilder.ServiceDesigner.cs`.

**Паттерн:**

```text
DTO parameters
  -> IServiceSchemaParameterBuilder
  -> IServiceSchemaClient
  -> ServiceSchemaManager
  -> serviceClient.Execute
```

## JWT lifecycle notification

**Когда использовать:** metadata содержит JWT auth и нужен контроль срока
действия token.

**Файл:** `JWTTokenExpirationNotification.Webhook.cs`.

Паттерн: process script перебирает `WebhookSchemaManager.GetItems()`, находит
`JsonWebTokenAuthInfo`, проверяет `ValidBefore` и создаёт `Reminding` для
пользователей с `CanManageSolution`.

## Чего не делать

- Не смешивать OCC channel webhooks и Webhook V2 designer.
- Не хранить secrets в client-side code или документации.
- Не обходить `ServiceSchemaManager` при выполнении Web Service V2 metadata.
- Не открывать edit/save без проверки `CanManageSolution`.
- Не считать `IntegrationV2` частью Webhook V2: в текущем коде это Exchange и
  meeting services.
