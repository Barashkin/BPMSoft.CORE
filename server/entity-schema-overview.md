## Что описывает схема

`EntitySchema` описывает метаданные сущности:

- имя схемы и GUID-идентичность;
- родительскую схему и режим расширения;
- колонки, primary/display columns и default values;
- lookup-связи, cascade и weak references;
- индексы;
- связь с generated entity class;
- generated `EventsProcess`.

В автогенерированном коде схема обычно находится в файле вида:

```text
{SchemaName}Schema.{Package}.cs
```

## Базовая иерархия

```text
EntitySchema
  └── BaseEntitySchema
        ├── BaseLookupSchema
        ├── BaseFolderSchema
        ├── BaseItemInFolderSchema
        ├── BaseEntityInTagSchema
        ├── FileSchema
        └── прикладные схемы
```

`BaseEntitySchema` добавляет системные колонки и базовый `EventsProcess`. `BaseLookupSchema` добавляет `Name` и `Description`, а специализированные базовые схемы описывают папки, теги, файлы и связи.

## Как читать schema-файл

Порядок чтения:

1. `InitializeProperties` — понять имя, родителя, view/table и режим расширения.
2. `InitializePrimaryDisplayColumn` — найти display column.
3. `InitializeColumns` — увидеть собственные колонки схемы.
4. `Create*Column` — понять типы, lookup target, обязательность, cascade.
5. `InitializeIndexes` — найти уникальность и performance hints.
6. `CreateEntity` — понять generated entity class.
7. `CreateEventsProcess` — перейти к generated events pipeline.

## Граница с ESQ

`Columns.Add(...)` в схеме создаёт метаданные колонки. `esq.AddColumn(...)` в запросе выбирает колонку для чтения. Эти API похожи по названию, но относятся к разным слоям.

Для запросов см. [ESQ Overview](esq-overview.md).
