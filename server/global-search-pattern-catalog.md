> Каталог рабочих Search / Global Search паттернов из `Autogenerated/Src`.

## Service proxy

Источник: `GlobalSearchService.GlobalSearch.cs`.

Паттерн:

1. WCF service принимает клиентские параметры.
2. `GlobalSearchHelper` строит search request.
3. `RestSharp` отправляет запрос к `GlobalSearchUrl`.
4. Ответ приводится к `BpmSearchResponse`.
5. Ошибки логируются в logger `GlobalSearch`.

Использовать как пример proxy service к внешнему поисковому endpoint.

## Фильтрация типов по правам

Источник: `GlobalSearchService.GlobalSearch.cs`.

Перед запросом типы из параметра `type` фильтруются через
`DBSecurityEngine.GetIsEntitySchemaReadingAllowed`. Это защищает от ситуации,
когда клиент передает в фильтр сущность, недоступную текущему пользователю.

## Проверка record rights после ES

Источник: `GlobalSearchHelper.GlobalSearch.cs`.

ES может вернуть запись, которую пользователь не должен видеть. Helper
дополнительно проверяет доступные идентификаторы и только после этого
формирует `BpmSearchResponseEntity`.

Для file entities учитывается master record rights, а не только права на
саму файловую запись.

## Localizable result

Источник: `GlobalSearchHelper.GlobalSearch.cs`.

Если включен `UseLocalizableGlobalSearchResult`, значения колонок
дочитываются из БД через `EntitySchemaQuery`. Это дороже, но позволяет
вернуть локализованное отображение.

## Provider selection на старте

Источник: `GlobalSearchEventListener.GlobalSearch.cs`.

На `OnAppStart` выполняется тестовый HTTP-вызов к GS endpoint. Успешный
`NoContent` переключает DI на `GsSearchProvider`, иначе остается
`ESSearchProvider`.

## Query builder через Nest

Источник: `ESQueryBuilder.GlobalSearch.cs`.

`SearchQueryRule` конвертируется в `QueryContainer`: фильтры внутри rule
соединяются через `AND`, rules и значения фильтра — через `OR`.

## Config refresh после build

Источник: `ConfigurationBuildWatcher.GlobalSearch.js`.

Клиентский watcher слушает `ServerChannel` и вызывает
`IndexingConfigService.SendIndexationConfigs` после сборки конфигурации или
изменения важных search settings.

## System indexing context

Источник: `IndexingConfigService.GlobalSearch.cs`.

Сбор и отправка конфигов индексации выполняются через
`AppConnection.SystemUserConnection`, чтобы конфигурация не зависела от прав
пользователя, инициировавшего service call.

## Feature-gated real-time indexing

Источник: `BaseIndexer.GlobalSearch.cs`.

Перед отправкой entity на индексацию проверяется feature `GlobalSearch_V2`.
Это позволяет включать новый контур индексации без изменения listeners.

## Cache invalidation по SysModule

Источник: `GlobalSearchSysModuleListener.GlobalSearch.cs`.

При изменении `SysModule.GlobalSearchAvailable` очищается кэш
`AvailableIndexingEntities`. Так изменение доступности секции влияет на
состав индексируемых сущностей.

## Row schema специализация

Источники:

- `BaseSearchRowSchema.GlobalSearch.js`;
- `ContactSearchRowSchema.GlobalSearch.js`;
- `AccountSearchRowSchema.GlobalSearch.js`;
- `FileSearchRowSchema.GlobalSearch.js`.

Базовая строка строит общий UI, primary URL и found columns, а схемы сущностей
добавляют специализацию отображения.

## SSP allowed columns

Источник: `GlobalSearchSSPHelper.SSP.cs`.

Для SSP helper ограничивает entities и колонки через portal ACL и кэш
`AllowedPortalColumns`. Это отдельный слой поверх обычных schema/record/column
rights.
