# ESQ Joins and Subqueries

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: ESQ, joins, subqueries, CreateExistsFilter, bracket path -->

> Документ по сложным связям в ESQ: lookup path, bracket notation, reverse links, `CreateExistsFilter` и граница между ESQ и `Select`.

## Lookup path как join

Обычный lookup path:

```csharp
esq.AddColumn("Contact.Name");
esq.AddColumn("Contact.Photo.Id");
esq.Filters.Add(esq.CreateFilterWithParameters(
    FilterComparisonType.Equal, "Contact.Id", contactId));
```

Такой path фактически описывает переход от текущей схемы к reference schema.

## Bracket notation

Bracket notation используется для обратных или нестандартных связей:

```text
[RelatedSchema:RelatedColumn:CurrentColumn].TargetColumn
```

Пример:

```csharp
var nickName = esq.AddColumn("[BPMSoftOCCOperatorUnit:SysAdminUnit:Id].Nickname");
```

Смысл:

- текущая схема: `SysAdminUnit`;
- связанная схема: `BPMSoftOCCOperatorUnit`;
- колонка связанной схемы: `SysAdminUnit`;
- колонка текущей схемы: `Id`;
- целевая колонка: `Nickname`.

## Mixed dot + bracket path

Встречаются смешанные варианты:

```csharp
esq.Filters.Add(esq.CreateFilterWithParameters(
    FilterComparisonType.Equal,
    "[SysAdminUnit:Id:User].Contact.Id",
    contactId));
```

Это читается как:

1. найти `SysAdminUnit`, где `SysAdminUnit.Id = SocialAccount.User`;
1. перейти к `SysAdminUnit.Contact`;
1. сравнить `Contact.Id`.

Reference: `BaseGoogleSynchronizer.OldGoogleIntegration.cs`.

## `CreateExistsFilter`

`CreateExistsFilter(...)` создаёт exists-подобный фильтр.

Пример из `ESNFeedModuleService.ESN.cs`:

```csharp
EntitySchemaQueryFilter rightsFilter = esq.CreateExistsFilter("[SysAdminUnit:Contact].Id");
EntitySchemaQuery rightsSubQuery = rightsFilter.RightExpressions[0].SubQuery;
rightsSubQuery.Filters.Add(rightsSubQuery.CreateFilterWithParameters(
    FilterComparisonType.Equal, "Active", true));
esq.Filters.Add(rightsFilter);
```

Этот паттерн полезен, когда нужно проверить наличие связанной записи, но не тянуть её как обычную колонку.

## Настройка subquery

После `CreateExistsFilter` можно получить subquery:

```csharp
EntitySchemaQuery rightsSubQuery = rightsFilter.RightExpressions[0].SubQuery;
```

Дальше в subquery добавляются фильтры:

- активность пользователя;
- connection type;
- другие ограничения связанной сущности.

## Reverse links в колонках

В OCC request pipeline встречается получение последнего сообщения через reverse path:

```csharp
esq.AddColumn("[BPMSoftOCCChatMessage:Chat:Id].CreatedOn").OrderByDesc();
```

Практический смысл:

- от чата перейти к связанным сообщениям;
- использовать дату сообщения для сортировки;
- не писать отдельный `Select`, пока сценарий остаётся читаемым.

## Когда ESQ становится слишком сложным

Признаки:

- path содержит несколько bracket-сегментов;
- один метод строит много динамических path;
- нужно `GroupBy`, агрегаты по нескольким таблицам или сложные `Join`;
- результат удобнее читать через `IDataReader`;
- запрос надо анализировать как SQL.

В таких случаях в кодовой базе часто переходят на `Select`.
Пример: `BPMSoftOCCRouting.BPMSoftOCC.cs` использует `Select` для сложного routing-алгоритма и ESQ для более простого подсчёта открытых чатов.

## ESQ + `GetSelectQuery`

Промежуточный вариант:

```csharp
Select select = esq.GetSelectQuery(userConnection);
```

Он полезен, когда:

- ESQ удобно строит path и права;
- итог нужно выполнить/посмотреть как SQL;
- надо сформировать SQL text для индексации или диагностики.

Reference: `IndexingEntityListBuilder.GlobalSearch.cs`.

## Практические правила

1. Обычный lookup path используйте для прямых связей.
1. Bracket path используйте только там, где обычный path не выражает связь.
1. Для bracket path всегда храните alias из `AddColumn(...)`.
1. Для exists-сценариев не пытайтесь имитировать `EXISTS` через выборку лишних колонок.
1. Если path перестал быть читаемым, вынесите построение в helper или переходите на `Select`.
