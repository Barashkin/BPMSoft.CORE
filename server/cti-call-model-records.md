# CTI Call Model And Records

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: CTI, Call, SysActiveCall, call records -->

> Звонок хранится в сущности `Call`, текущий активный контекст — в
> `SysActiveCall`, а записи разговоров получают отдельный WebRTC service
> helper.

## Call

Файл: `Autogenerated/Src/CallSchema.Base.cs`.

Базовая схема `Call` содержит ключевые поля телефонии:

| Колонка | Назначение |
| ------- | ---------- |
| `IntegrationId` | внешний идентификатор звонка |
| `ParentCallId` | связь консультационного/переведенного звонка |
| `CallerId` | номер вызывающего |
| `CalledId` | номер вызываемого |
| `RedirectingId` | номер, с которого перенаправляли |
| `RedirectionId` | номер, на который перенаправляли |
| `StartDate`, `EndDate` | временные границы |
| `Duration` | длительность |
| `BeforeConnectionTime` | время до соединения |
| `TalkTime` | время разговора |
| `HoldTime` | время удержания |
| `Direction` | направление звонка |
| `Contact`, `Account` | идентифицированный абонент |
| `Result` | результат звонка |
| `Activity` | связь с активностью |
| `GloballyUniqueCallLinkageId` | внешний linkage id |

Для истории есть индекс `ICallCreatedOn` по `CreatedOn DESC`.

## WebRTC extension

Файл: `Autogenerated/Src/CallSchema.WebRTCCore.cs`.

WebRTC-пакет расширяет `Call`, в частности для связи с записью разговора и
дополнительными индексами. Этот слой важен, если проблема касается call record
links или provider-specific данных.

## SysActiveCall

Файл: `Autogenerated/Src/SysActiveCallSchema.Base.cs`.

`SysActiveCall` хранит текущий серверный контекст активной линии:

- `IntegrationId`;
- `Call`;
- `ParentCall`;
- `CallerId`, `CalledId`;
- `RedirectingId`, `RedirectionId`;
- `StartDate`, `EndDate`;
- `CurrentHoldStartTime`;
- `CurrentTalkStartTime`;
- `ConnectionStartTime`;
- `Duration`, `BeforeConnectionTime`, `TalkTime`, `HoldTime`;
- `State`;
- `CallContext`, `CallContextType`;
- `Direction`.

Связь `Call` имеет cascade behavior, поэтому удаление call record может
затрагивать активный контекст.

## Справочники и вспомогательные схемы

| Схема | Назначение |
| ----- | ---------- |
| `CallDirection` | направление звонка |
| `CallTopic` | тема звонка |
| `CallFolder`, `CallInFolder` | папки звонков |
| `CallTag`, `CallInTag` | теги звонков |
| `CallFile` | файлы звонка |
| `VwRecentCall` | view для последней истории |

`VwRecentCall` используется CTI-панелью для истории коммуникаций.

## Записи разговоров

Файлы:

- `Autogenerated/Src/TelephonyCallRecordServiceHelper.WebRTCCore.cs`;
- `Autogenerated/Src/TelephonyCallRecordServiceConstants.WebRTCCore.cs`;
- `Autogenerated/Src/UisCallRecordService.WebRTCCore.cs`;
- `Autogenerated/Src/UisCallRecordServiceHandler.WebRTCCore.cs`.

`TelephonyCallRecordServiceHelper` и `TelephonyHandler` инкапсулируют HTTP
интеграцию с внешней системой записей разговоров:

- читают `TelephonyIntegration`;
- учитывают `AuthType`;
- используют timezone сервера телефонии;
- формируют ссылку на запись по `CallRecordLinkUrlTemplate`;
- логируют ошибки отдельным logger через `LogManager`.

UIS-specific service и handler показывают provider-specific реализацию
получения записей.

## Webhook входящих событий

Файл: `Autogenerated/Src/UisWebHookService.WebRTCCore.cs`.

`UisWebHookService` — пример WCF/BaseService для входящих событий от внешней
телефонии. Он полезен как reference для обработки callback/webhook от АТС.
