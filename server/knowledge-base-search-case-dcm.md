# Knowledge Base Search, Case Search Row And DCM Cases

<!-- Версия: 1.1 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: KnowledgeBase, GlobalSearch, Case, DCM -->

> В Base-пакете есть search/UI-обвязка для Knowledge Base и Case, но нет
> полноценной ITSM-сущности `Case`. `CaseSearchRowSchema` ожидает поля из
> внешнего пакета с обращениями, а `SectionWizardCasesSettings` относится к
> DCM case settings, а не к service case lifecycle.

## KnowledgeBase Global Search row

Файл: `Autogenerated/Src/KnowledgeBaseSearchRowSchema.GlobalSearch.js`.

Строка результата:

- удаляет `EntitySchemaCaption`;
- растягивает `PrimaryColumnContainer`;
- переносит `FoundColumnsContainerList`;
- добавляет `Type`;
- добавляет `CreatedBy`.

Схема предназначена только для отображения результата. Сам поиск и права
описаны в [global-search-overview.md](global-search-overview.md).

## Case Global Search row

Файл: `Autogenerated/Src/CaseSearchRowSchema.GlobalSearch.js`.

В Base есть только строка результата для сущности `Case`. Она ожидает поля:

- `Subject`;
- `ServiceItem`;
- `Client`;
- `RegisteredOn`;
- `Status`;
- `Owner`.

`Client` описан как lookup attribute с `multiLookupColumns`:

```text
Contact, Account
```

Это означает, что UI готов показывать клиента как contact/account, но сама
сущность `Case`, lifecycle, маршрутизация и SLA должны находиться в другом
пакете.

## Что не покрывает Base

В `Base/Autogenerated/Src` не найдены:

- `CaseSchema`;
- `CasePageV2`;
- `CaseSectionV2`;
- `ServiceItemSchema`;
- `ServicePactSchema`;
- `ServiceInServicePactSchema`;
- SLA policy schemas;
- routing/escalation schemas.

Если задача касается этих объектов, нужно искать в пакете, где установлена
функциональность обращений/ITSM.

## DCM case settings

Файл: `Autogenerated/Src/SectionWizardCasesSettings.DcmLibrary.js`.

`SectionWizardCasesSettings` относится к настройке case schemas в DCM wizard.
Это не клиентское обращение и не сервисный кейс.

Полный DCM-контур описан в
[dcm-dynamic-case-overview.md](dcm-dynamic-case-overview.md).

Ключевые messages:

| Message | Направление | Назначение |
| ------- | ----------- | ---------- |
| `SectionDcmSettingsInitialized` | subscribe | DCM settings готовы |
| `SectionDcmLibraryInitialized` | subscribe | DCM library готова |
| `ValidateSectionDcmSettings` | publish | валидация настроек |
| `SaveSectionDcmSettings` | publish | сохранение настроек |
| `OnSectionDcmSettingsSaved` | subscribe | результат сохранения |
| `AddCase` | subscribe | добавить DCM case |
| `EditCase` | subscribe | редактировать DCM case |
| `ReloadCaseSettings` | broadcast publish | перезагрузить case settings |
| `ReloadDcmLibGridData` | broadcast publish | обновить DCM library grid |

## ServerChannel reload

`SectionWizardCasesSettings` слушает `BPMSoft.ServerChannel` и реагирует на
событие:

```text
ReloadSectionWizardCaseSettings
```

Если `dcmSettingsId` совпадает с текущим settings item, модуль публикует
`ReloadCaseSettings` и `ReloadDcmLibGridData`.

## BusinessRuleCaseDesigner

Файлы:

- `BusinessRuleCaseDesignerContainer.Wizards.js`;
- `BusinessRuleCaseDesignerViewModel.Wizards.js`;
- `BusinessRuleCaseDesignerCollectionContainer.Wizards.js`.

Эти файлы используют слово `Case` в контексте дизайнера условий/правил. Их
нужно отличать от service case lifecycle.
