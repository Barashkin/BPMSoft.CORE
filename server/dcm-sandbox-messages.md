# DCM Sandbox Messages

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: DCM, sandbox, SectionWizardCasesSettings, SectionDcmSettings -->

> Основной DCM sandbox-контур в Base виден в связке section wizard,
> section settings и DCM library. Он объединяет настройку stage column,
> список DCM-схем и сохранение wizard.

## SectionDcmSettings messages

Файл: `Autogenerated/Src/SectionDcmSettings.DcmLibrary.js`.

| Message | Mode | Direction | Роль |
| ------- | ---- | --------- | ---- |
| `SectionDcmSettingsInitialized` | PTP | publish | настройки секции готовы |
| `ValidateSectionDcmSettings` | PTP | subscribe | запустить validation настроек |
| `SaveSectionDcmSettings` | PTP | subscribe | сохранить настройки |
| `OnSectionDcmSettingsSaved` | PTP | publish | настройки сохранены |
| `ReloadCaseSettings` | broadcast | subscribe | перезагрузить case settings |
| `ReloadDcmLibGridData` | broadcast | publish | обновить DCM library grid |
| `EnabledDcmSchemas` | PTP | subscribe | получить включённые схемы |

## VwDcmLibSection messages

Файл: `Autogenerated/Src/VwDcmLibSection.DcmLibrary.js`.

| Message | Mode | Direction | Роль |
| ------- | ---- | --------- | ---- |
| `SectionDcmLibraryInitialized` | PTP | publish | библиотека DCM готова |
| `AddCase` | PTP | publish | добавить DCM case |
| `EditCase` | PTP | publish | редактировать DCM case |
| `ReloadDcmLibGridData` | broadcast | subscribe | обновить grid |
| `EnabledDcmSchemas` | PTP | publish | передать включённые схемы |
| `HideBodyMask` | PTP | publish | скрыть mask wizard |

## SectionWizardCasesSettings messages

Файл: `Autogenerated/Src/SectionWizardCasesSettings.DcmLibrary.js`.

Wizard объединяет оба модуля и зеркалит их сообщения:

- подписан на `SectionDcmSettingsInitialized`;
- подписан на `SectionDcmLibraryInitialized`;
- публикует `ValidateSectionDcmSettings`;
- публикует `SaveSectionDcmSettings`;
- подписан на `OnSectionDcmSettingsSaved`;
- подписан на `AddCase` и `EditCase`;
- публикует `ReloadCaseSettings`;
- публикует `ReloadDcmLibGridData`;
- подписан на `HideBodyMask`.

## Initialization pattern

`SectionWizardCasesSettings.Initialized` становится `true`, только когда:

- settings module отправил `SectionDcmSettingsInitialized`;
- library module отправил `SectionDcmLibraryInitialized`.

После этого wizard может скрыть body mask.

## ServerChannel reload

Wizard слушает `BPMSoft.ServerChannel` и проверяет:

```text
message.event === "ReloadSectionWizardCaseSettings"
```

Если `message.dcmSettingsId` совпадает с текущим item, вызывается reload:

- `ReloadCaseSettings`;
- `ReloadDcmLibGridData`.

## PTP vs broadcast

PTP используется для конкретного wizard/settings instance:

- validation;
- save;
- initialized;
- enabled schemas.

Broadcast используется для широкого reload:

- `ReloadCaseSettings`;
- `ReloadDcmLibGridData`;
- wizard navigation messages.

## Практические правила

- Если wizard завис на mask, проверяйте оба initialized message.
- Если save не запускается, проверьте PTP module id.
- Если после сохранения дизайнера wizard не обновился, проверяйте ServerChannel
  event `ReloadSectionWizardCaseSettings`.
- Если grid не обновляется, проверяйте broadcast `ReloadDcmLibGridData`.
