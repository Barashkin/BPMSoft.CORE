# ESN Feed Services And Mentions

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: ESN, EsnCenter, mentions, SocialMentionSearchRule -->

> Серверный слой ESN разделен на фасад `EsnCenter` и WCF-сервис для mentions.
> Основная чтение/запись ленты выполняется через `EsnService`, а автогенерат
> показывает DTO, права, фабрики и mention contracts.

## EsnCenter

Файл: `Autogenerated/Src/EsnCenter.ESN.cs`.

`EsnCenter` зарегистрирован как `IEsnCenter` через `[DefaultBinding]`.

Зависимости:

| Зависимость | Назначение |
| ----------- | ---------- |
| `IEsnLikeRepository` | добавить/удалить лайк, получить лайки |
| `IEsnMessageReader` | читать сообщения и комментарии |
| `IEsnMessageRedactor` | удалить пост или комментарий |
| `IEsnSecurityEngine` | проверить права на чтение/удаление |
| `EsnLogContext` | включить диагностическое логирование |

Ключевые методы:

- `GetMessageLikesForUser`;
- `GetWhoLikedMessage`;
- `LikeMessage`;
- `UnLikeMessage`;
- `ReadComments`;
- `ReadMessage`;
- `ReadEntityMessage`;
- `ReadFeedMessage`;
- `DeletePost`.

При чтении и удалении `EsnCenter` проверяет `IEsnSecurityEngine`. При
недостатке прав выбрасывается `SecurityException` с сообщением по `SocialMessage`.

## EsnCenterFactory

Файл: `Autogenerated/Src/EsnCenterFactory.ESN.cs`.

Фабрика создает `EsnCenter` через `ClassFactory` и передает `EsnLogContext`.
Это позволяет включать логирование чтения фида без изменения публичного
контракта фасада.

## Mention service

Файл: `Autogenerated/Src/ESNFeedModuleService.ESN.cs`.

`ESNFeedModuleService`:

- помечен `[ServiceContract]`;
- доступен через `[DefaultServiceRoute, SspServiceRoute]`;
- наследуется от `BaseService`;
- возвращает `ContactForMention`.

`ContactForMention` сериализуется для клиента:

| DataMember | Источник |
| ---------- | -------- |
| `value` | `Contact.Id` |
| `primaryImage` | `Contact.PhotoId` |
| `displayValue` | primary display column |
| `secondaryInfo` | `Email` |

Размер страницы mentions: `MentionContactsPageSize = 5`.

## Права mentions

Сервис добавляет права чтения через `DBSecurityEngine.GetRecordsByRightCondition`
для `Contact`, если схема администрируется по записям или пользователь SSP.

Дополнительная SSP-логика включается условиями:

- текущий пользователь `UserType.SSP`;
- feature `IsSSPContactSocialMentions`;
- есть `entitySchemaUId`;
- общий список еще не достиг `MentionContactsPageSize`.

После этого сервис добавляет контакты текущей страницы и контакты по правилам
mentions.

## SocialMentionSearchRule

Файл: `Autogenerated/Src/SocialMentionSearchRuleSchema.ESN.cs`.

Настраиваемое правило поиска пользователей при `@`:

| Колонка | Назначение |
| ------- | ---------- |
| `EntitySchema` | имя схемы |
| `FilterByColumn` | колонка фильтрации текущей записи |
| `UserColumn` | колонка пользователя |

UI настройки:

- `SocialMentionSearchRuleSection.ESN.js`;
- `SocialMentionSearchRulePage.ESN.js`.

SSP-расширение:

- `SocialMentionSchema.SSP.cs`.

## Клиент mentions

Файл: `Autogenerated/Src/SocialMentionUtilities.ESN.js`.

Миксин формирует клиентский сценарий `@`:

- определяет SSP-пользователя;
- вызывает mention endpoint;
- подставляет contact links в HTML сообщения;
- работает вместе с feed view models.
