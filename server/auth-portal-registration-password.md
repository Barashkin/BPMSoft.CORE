# Auth Portal Registration And Password

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: auth, SSP, registration, password, TOTP -->

> SSP-аутентификация в Base сильнее всего представлена self-registration,
> восстановлением пароля, email templates и TOTP recovery link.

## UserManagementService

Файл: `Autogenerated/Src/UserManagementService.SSP.cs`.

`UserManagementService` — WCF service для SSP registration/recovery. Он
использует `SystemUserConnection`:

```text
AppConnection.SystemUserConnection
```

Ключевые DTO:

- `UserManagementServiceToken` — имя, фамилия, email, пароль, workspace URL;
- `RecoveryPasswordToken` — email/login для восстановления.

Метод `Register`:

1. Проверяет `GlobalAppSettings.ShowPortalSelfRegistrationLink`.
2. Проверяет резервирование логина через `LoginReservationPeriod` и
   `SysAdminUnitNameReserve`.
3. Ищет существующий `Contact` по email.
4. Проверяет, есть ли уже `SysAdminUnit` для контакта.
5. Создает `Contact`, если нужно.
6. Запускает регистрацию/подтверждение через helper слой.

Если включен `CheckTermsAndConditions`, сервис добавляет virtual column
`IsAgreed` на `Contact`.

## RegistrationHelper

Файл: `Autogenerated/Src/RegistrationHelper.SSP.cs`.

`RegistrationHelper` отвечает за письма регистрации и восстановления:

- выбирает `EmailTemplate`;
- подставляет `#RegistrationUrl#`;
- заменяет macros через `EmailTemplateUtility`;
- выбирает mailbox: `SSPRegistrationMailbox` или `SupportServiceEmail`;
- отправляет через `HtmlEmailMessageSender` или legacy `IEmailSender`.

Важные настройки/features:

| Настройка / feature | Назначение |
| ------------------- | ---------- |
| `PortalRegistrationEmailTemplate` | шаблон письма регистрации |
| `SSPRegistrationMailboxEnabled` | использовать mailbox SSP |
| `SSPRegistrationMailbox` | mailbox регистрации |
| `SupportServiceEmail` | fallback sender email |
| `OldEmailIntegration` | legacy email path |
| `UseEmailSenderForSelfReg` | использовать новый sender для self-reg |

## TOTP recovery password

Файл: `Autogenerated/Src/TotpSendResetPasswordLinkService.SSP.cs`.

`TotpSendResetPasswordLinkService`:

- логирует в `Authentication`;
- генерирует token через `ITotpSetupTokenGenerator`;
- строит URL `/Login/TotpRecoverPassword.aspx?totpSetupToken=...`;
- ищет email пользователя через `TotpSecret` и `SysAdminUnit`;
- сбрасывает `ForceChangePassword`;
- выставляет `PasswordExpireDate = UtcNow + 1 day`;
- отправляет письмо через `RegistrationHelper`.

Метод `SendRecoveryPasswordLink` намеренно возвращает success даже если email
не найден. Это снижает риск user enumeration.

## TOTP storage

Связанные схемы:

- `TotpSetupTokenSchema.Base.cs`;
- `TotpSecretSchema.Base.cs`.

`TotpSecret` связывает пользователя с TOTP secret, а setup token используется
для recovery/setup сценариев.

## SSP service wrappers

Связанные файлы:

- `SspUserManagementService.SSP.cs`;
- `SspUserManagementServiceHelper.SSP.cs`;
- `UserManagementModule.SSP.js`;
- `RegistrationRes.SSP.js`.

Они формируют client/service слой SSP self-service поверх registration helpers.
