# Integration Tools Runtime

<!-- Версия: 1.0 | Обновлено: 2026-04-28 | Платформа: BPMSoft 1.9 -->
<!-- Теги: IntegrationTools, runtime, WebhookSchemaManager, ServiceSchemaManager, auth -->

> Runtime-контур Integration Tools: schema managers, права, auth helpers,
> генерация JWT, тестовый вызов Web Service V2 и граница с бизнес-процессами.

## Schema managers

Integration Tools работает не с обычными entity records как с источником
runtime-метаданных. UI сохраняет schema metadata, а runtime использует managers:

| Контур | Manager |
| ------ | ------- |
| Webhook V2 | `WebhookSchemaManager` |
| Web Service V2 | `ServiceSchemaManager` |

View-схемы `VwWebhookV2` и `VwWebServiceV2` нужны для отображения в UI,
workspace navigation, folders/tags и карточек.

## Webhook runtime

В Base-слое видны интеграционные точки с core namespace `BPMSoft.Webhooks`:

- `WebhookSchemaDesigner`;
- `WebhookSchemaManager`;
- `WebhookSchema`;
- `WebhookAuthInfo`;
- `JsonWebTokenAuthInfo`;
- `WebhookParameterValue`.

`JWTTokenExpirationNotification` показывает runtime-паттерн: получить manager,
перебрать schema manager items, прочитать webhook auth info и создать
платформенные reminders.

## Webhook auth service calls

Client-side auth page вызывает `WebhookService`:

```text
ServiceHelper.callService("WebhookService", "UseAnonymousAuth", ...)
ServiceHelper.callService("WebhookService", "IsSecretKeyValid", ...)
```

Реализация `WebhookService` в `Autogenerated/Src` не найдена, вероятно она
поставляется ядром/другим пакетом. В документации этого dive фиксируется только
видимый contract, используемый UI.

## Web Service runtime call

`CallServiceSchemaService` - runtime facade для тестового/управляемого вызова
настроенной service schema.

```text
Execute(serviceName, methodName, parameters)
  -> CanManageSolution
  -> IServiceSchemaParameterBuilder.Build
  -> IServiceSchemaClient.Execute
```

`ServiceSchemaClient`:

```text
ServiceSchemaManager.GetInstanceByName(serviceName)
  -> CreateServiceClient(userConnection)
  -> CreateRequest(methodName)
  -> RequestParameters = parameters
  -> Execute(request)
```

## Parameter builder

`ServiceSchemaParameterBuilder` превращает плоский/вложенный список DTO в
словарь:

```text
ServiceSchemaParameter.Code -> key
ServiceSchemaParameter.Value -> scalar value
NestedParameters -> list of nested dictionaries
```

Это важно для object/array request structures.

## Rights

Ключевые операции:

| Operation | Где используется |
| --------- | ---------------- |
| `CanManageSolution` | редактирование webhook/service schemas, вызов `CallServiceSchemaService.Execute` |
| `CanViewConfiguration` | загрузка `VwWebServiceV2` metadata |

Если пользователь не имеет права, designer может открыться частично или не
сохранить/не выполнить тестовый вызов.

## Process boundary

Webhook action rules могут запускать бизнес-процесс (`START_BP`), но общий
runtime процесса остаётся в Process Dive.

Web Service V2 может вызываться из process/custom logic через service schema
client, но сам designer не заменяет Process engine и не хранит process state.
