# Global Search Settings And SSP

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: GlobalSearch, SysSettings, feature, SSP, portal -->

> Поведение Global Search управляется системными настройками, feature flags и
> SSP override. Для портальных пользователей результат дополнительно
> ограничивается доступными схемами и колонками.

## Основные SysSettings

| Настройка | Где используется | Назначение |
| --------- | ---------------- | ---------- |
| `GlobalSearchUrl` | `GlobalSearchService`, `GlobalSearchEventListener`, `GsClient` | endpoint ES/GS сервиса |
| `GlobalSearchConfigServiceUrl` | `IndexingConfigSender` | endpoint сервиса конфигураций индексации |
| `DataServiceQueryTimeout` | `GlobalSearchService`, provider init | timeout HTTP-запросов |
| `GlobalSearchLookupFilterEnabled` | `ConfigurationBuildWatcher` | триггер пересылки indexing config |
| `GlobalSearchIndexedDataConfig` | `ConfigurationBuildWatcher`, indexing config | JSON конфигурация индексируемых данных |
| `GlobalSearchShouldMatchPercent` | `GlobalSearchHelper` | процент совпадения |
| `UseInexactGlobalSearch` | `GlobalSearchHelper` | неточный поиск |
| `UseLocalizableGlobalSearchResult` | `GlobalSearchHelper` | локализованные значения результата |
| `GlobalSearchDefaultPrimaryColumnWeight` | `GlobalSearchHelper` | вес primary display column |
| `GlobalSearchDefaultEntityWeight` | `GlobalSearchHelper` | вес сущности |

## Feature flags

| Feature | Назначение |
| ------- | ---------- |
| `GlobalSearch` | включает запуск Global Search config job на старте |
| `GlobalSearch_V2` | включает real-time indexing через `BaseIndexer` |

Если `GlobalSearch_V2` выключен, `BaseIndexer.IndexEntity` завершится без
отправки entity в indexing service.

## SearchSetting

Файл: `Autogenerated/Src/SearchSettingSchema.GlobalSearch.cs`.

`SearchSetting` хранит настройки поиска по сущности:

- `Name` — primary display column;
- `Settings` — JSON/text настройки;
- `SearchEntity` — lookup на `VwSysSchemaInfo`.

Эта сущность нужна для объектных search settings и связана с entity schema,
для которой настраивается поиск.

## IndexedDataConfig

Файл: `Autogenerated/Src/IndexedDataConfig.GlobalSearch.cs`.

`IndexedDataConfig` представлен как словарь:

```text
entity or section key -> IndexedSectionConfig
```

`IndexedSectionConfig` содержит `IgnoredColumns`. Эта структура используется
для настройки состава индексируемых данных и связана с
`GlobalSearchIndexedDataConfig`.

## SSP override

Файл: `Autogenerated/Src/GlobalSearchSSPHelper.SSP.cs`.

`GlobalSearchSSPHelper` помечен `[Override]` и расширяет
`GlobalSearchHelper`. Для пользователя с `ConnectionType = UserType.SSP` он:

- ограничивает группы результатов схемами, доступными на портале;
- фильтрует aggregation groups;
- учитывает `GlobalAppSettings.UsePortalSchemaAllowedColumns`;
- кэширует allowed columns в `SSPSecurityEngine.AllowedPortalColumnsCacheKey`;
- проверяет доступность entity через `DBSecurityEngine.GetIsAvailableOnSsp`;
- не применяет default score settings для SSP.

## Доступные SSP схемы

Доступные portal schemas определяются через связь рабочих мест, секций и
портального пользователя:

```text
SysAdminUnitInRole
  -> SysAdminUnitInWorkplace
  -> SysWorkplace
  -> SysModuleInWorkplace
  -> SysModule
  -> SysModuleEntity
```

Дополнительно schema должна быть лицензирована/доступна на SSP через
`GetIsAvailableOnSsp`.

## Доступные SSP колонки

Когда включена логика allowed columns, helper читает:

- `PortalSchemaAccessList`;
- `PortalColumnAccessList`.

В allowed columns всегда добавляется `Id`, а остальные колонки попадают в
ответ только если они разрешены для portal schema и проходят обычную проверку
`CanReadColumn`.

Подробнее о portal ACL см. [ssp-portal-access-rights.md](ssp-portal-access-rights.md).
