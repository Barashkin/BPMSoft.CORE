# Global Search Query Providers

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: GlobalSearch, ESSearchProvider, GsSearchProvider, ESQueryBuilder -->

> В Global Search есть два серверных режима: прямой provider к
> Elasticsearch-репозиторию и provider к новому Global Search service.

## Выбор provider

Файл: `Autogenerated/Src/GlobalSearchEventListener.GlobalSearch.cs`.

На старте приложения listener вызывает тестовый endpoint:

```text
{GlobalSearchUrl}/test/_getcollection
```

Если ответ имеет статус `NoContent`, регистрируется новый provider:

```text
ISearchProvider -> GsSearchProvider
```

Иначе используется старый provider:

```text
ISearchProvider -> ESSearchProvider
```

Выбор зависит от `SysSettings.GlobalSearchUrl`, доступности endpoint и
таймаута `DataServiceQueryTimeout`.

## ESSearchProvider

Файл: `Autogenerated/Src/ESSearchProvider.GlobalSearch.cs`.

`ESSearchProvider` реализует `ISearchProvider` поверх
`IElasticSearchRepository`:

1. Получает `EntitySchema` для `SearchQuery.SchemaName`.
2. Создает `GlobalSearchFilterHelper`.
3. Строит `PageableElasticQuery` через `IESQueryBuilder`.
4. Выполняет `IElasticSearchRepository.GetCollection`.
5. Фильтрует результат через `GlobalSearchFilterHelper.Filter`.

При ошибке запроса к ES provider пишет в лог `GlobalSearch` и пробрасывает
исключение дальше.

## ESQueryBuilder

Файл: `Autogenerated/Src/ESQueryBuilder.GlobalSearch.cs`.

`ESQueryBuilder` строит Nest-запрос:

- один `SearchQueryRule` превращается в набор `&&` фильтров;
- несколько rules объединяются через `||`;
- значения одного filter также объединяются через `||`;
- для каждого значения создается `MatchQuery`;
- имя поля берется из `ISearchColumnNameProvider`;
- `MinimumShouldMatch` по умолчанию равен `100%`;
- `From` и `Size` берутся из `SearchQuery` либо из default значений.

## GsSearchProvider и GsClient

Файлы:

- `Autogenerated/Src/GsSearchProvider.GlobalSearch.cs`;
- `Autogenerated/Src/GsClient.GlobalSearch.cs`;
- `Autogenerated/Src/GsSearchQuery.GlobalSearch.cs`.

`GsSearchProvider` адаптирует внутренний `SearchQuery` к DTO нового GS API.
`GsClient` выполняет HTTP-вызов к `_getcollection`, добавляет credentials
через `GlobalSearchConnectorHelper` и учитывает отмену запроса по
`ClientDisconnectedToken`.

`GsSearchQueryDto` содержит:

- `schemaName`;
- `columns`;
- `rules`;
- `filters`;
- `from` / `size`;
- `primaryColumnValue`.

## Credentials и connection string

Файл: `Autogenerated/Src/GlobalSearchConnectorHelper.GlobalSearch.cs`.

`GlobalSearchConnectorHelper` разбирает connection string поиска и добавляет
Basic credentials в `RestRequest`. Этот helper используется и старым proxy
service, и новым GS client.

## Имена колонок

Файл: `Autogenerated/Src/ESSearchColumnNameProvider.GlobalSearch.cs`.

`ESSearchColumnNameProvider` инкапсулирует правила именования полей в индексе:

- lookup suffix;
- special handling communication details;
- mapping client column name -> indexed field name;
- учет reference schema.
