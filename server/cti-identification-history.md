# CTI Identification And History

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: CTI, caller identification, history, VwRecentCall -->

> CTI-панель отдельно обрабатывает идентификацию абонента, ручной поиск,
> сохранение найденных связей в `Call` и историю последних коммуникаций.

## Идентификация абонента

Файл: `Autogenerated/Src/CtiPanelIdentificationUtilities.CTIBase.js`.

Миксин заполняет коллекции:

- `IdentifiedSubscriberPanelCollection`;
- `IdentifiedConsultSubscriberPanelCollection`;
- `SearchResultPanelCollection`.

Основная логика:

1. Группирует найденные коммуникации по `SubscriberId`.
2. Создает panel view models для абонентов и их коммуникаций.
3. Поддерживает advised subscriber для исходящего звонка.
4. Обновляет `Call.identificationFieldsData`.
5. Сохраняет выбранные поля в `Call` через `UpdateQuery`.
6. Генерирует событие `callIdentificationSaved`.

## CTISearchResult

Файл: `Autogenerated/Src/CTISearchResultSchema.CTIBase.cs`.

`CTISearchResult` используется как модель результата поиска в CTI. Она
помогает отделить search result для панели от основной модели контакта или
аккаунта.

Связанные UI-модели:

- `IdentifiedSubscriberItem.CTIBase.js`;
- `SubscriberSearchResultItem.CTIBase.js`;
- `SubscriberCommunicationItem.CTIBase.js`;
- `SubjectResultSchema.CTIBase.cs`.

## Сохранение идентификации в Call

`updateCallByIdentificationData` сохраняет выбранные поля только если:

- `call.needSaveIdentificationData = true`;
- звонок уже сохранен в БД;
- есть `call.databaseUId`;
- `identificationFieldsData` не пустая.

Сохранение выполняется через `BPMSoft.UpdateQuery` по root schema `Call`.
При успехе флаг `needSaveIdentificationData` сбрасывается.

## История коммуникаций

Файл: `Autogenerated/Src/CtiPanelCommunicationHistoryUtilities.CTIBase.js`.

Миксин строит ESQ по `VwRecentCall`:

- `rowCount` берется из `CtiSettings.communicationHistoryRowCount`;
- сортировка идет по `Call.CreatedOn DESC`;
- фильтр `CreatedById = CURRENT_USER_CONTACT`;
- добавляются колонки `Call.Contact`, `Call.Account`, `Direction`,
  `CallerId`, `CalledId`, `TalkTime`, `EndDate`;
- дополнительно добавляются entity connection relation columns.

## Тип звонка в истории

`getHistoryCallType` маппит runtime call в UI-тип:

| Условие | Тип |
| ------- | --- |
| `direction = OUT` | outgoing |
| входящий и inactive state | missed |
| входящий активный/завершенный разговор | incoming |

Для результата из `VwRecentCall` исходящий номер берется из `CalledId`, а
входящий — из `CallerId`.

## VwRecentCall

Файл: `Autogenerated/Src/VwRecentCallSchema.CTIBase.cs`.

`VwRecentCall` — DB view для последних звонков. Она используется CTI-панелью,
чтобы быстро построить историю без прямого чтения всех `Call` записей.
