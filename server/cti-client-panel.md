# CTI Client Panel

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: CTI, CtiPanel, CommunicationPanel, sandbox -->

> CTI UI строится вокруг `CtiPanel`, панели коммуникаций, sandbox-сообщений и
> миксинов для модели звонка, идентификации и истории.

## CtiPanel

Файл: `Autogenerated/Src/CtiPanel.CTIBase.js`.

`CtiPanel` подключает:

- `CtiProviderInitializer`;
- `CtiBaseHelper`;
- `CtiConstants`;
- `Call`;
- `CtiPanelModelUtilities`;
- `CtiPanelUtils`;
- `CtiPanelIdentificationUtilities`;
- `CtiPanelCommunicationHistoryUtilities`;
- `EntityConnectionLinksUtilities`.

Панель хранит состояние звонка через virtual attributes:

- `PhoneNumber`;
- `PhoneNumberOldValue`;
- `CallDuration`;
- `CommutationStartedOn`;
- `DurationTimerIntervalId`;
- `IdentifiedSubscribersCount`;
- `IdentifiedSubscriberKey`;
- `IdentifiedConsultSubscribersCount`;
- `IdentifiedConsultSubscriberKey`;
- коллекции идентификации, поиска и истории.

## Sandbox messages

Ключевые сообщения `CtiPanel`:

| Message | Направление | Назначение |
| ------- | ----------- | ---------- |
| `CallCustomer` | subscribe | инициировать звонок клиенту |
| `GetCallRecords` | subscribe | получить записи звонков |
| `SelectCommunicationPanelItem` | publish | выбрать элемент панели коммуникаций |
| `CallDurationChanged` | publish | обновить длительность звонка |
| `CtiPanelConnected` | publish | панель подключилась к CTI |
| `PushHistoryState` | publish | обновить navigation history |
| `AgentStateChanged` | publish | изменить состояние оператора |
| `GetHistoryState` | publish | запросить текущий history state |
| `CardModuleResponse` | subscribe | получить ответ после сохранения карточки |

## Communication panel

Репрезентативные файлы:

- `CommunicationPanel.CTIBase.js`;
- `CtiPanelModule.CTIBase.js`;
- `CTIBaseCommunicationViewModel.CTIBase.js`;
- `BaseCommunicationDetail.CTIBase.js`.

`CommunicationPanel` является контейнером, а `CtiPanelModule` подключает
view model панели в общий UI.

## Main header integration

Файлы:

- `MainHeaderSchema.CTIBase.js`;
- `MainHeaderExtensions.CTIBase.js`.

Header integration отвечает за отображение состояния оператора, иконок и
быстрых CTI-действий в верхней панели. Состояния пользователя связаны с
`SysMsgUserStateInLib` и `SysMsgLib`.

## Context call

Файл: `Autogenerated/Src/ContextCallUtilities.CTIBase.js`.

`ContextCallUtilities` добавляет сценарий "позвонить из контекста записи":

```text
phoneLinkClicked
  -> callCustomer
  -> CtiPanel message / provider call
```

В вызов передаются `entitySchemaName`, `entityId` и номер телефона. Это
важный паттерн для секций и деталей, где номер отображается как action/link.

## Call section and page

Файлы:

- `CallSectionV2.CTIBase.js`;
- `CallPageV2.CTIBase.js`;
- `CallSectionV2.WebRTCCore.js`;
- `CallPageV2.WebRTCCore.js`;
- `CallSectionGridRowViewModel.CTIBase.js`.

Эти модули отвечают за карточку и секцию звонков. WebRTC layer расширяет
базовую CTI функциональность provider-specific поведением.

## Уведомления

Файл: `Autogenerated/Src/CtiNotificationUtilities.CTIBase.js`.

Утилита работает с desktop popup notifications и типами звонков из
`CtiConstants.CallType`: incoming, outgoing, missed и другие статусы UI.
