> Входная точка в `EntitySchemaQuery` в BPMSoft: когда использовать ESQ, как устроен пакет документов и где искать практические примеры.

## Что такое ESQ

`EntitySchemaQuery` - основной объектный API для чтения данных через схему сущности.
Он строит запрос поверх `EntitySchema`, учитывает модель колонок, lookup-связи, display values, права и часть платформенных правил.

Минимальный паттерн:

```csharp
var esq = new EntitySchemaQuery(userConnection.EntitySchemaManager, "Contact");
var nameColumn = esq.AddColumn("Name");
esq.Filters.Add(esq.CreateFilterWithParameters(FilterComparisonType.Equal, "Id", contactId));
var collection = esq.GetEntityCollection(userConnection);
string name = collection[0].GetTypedColumnValue<string>(nameColumn.Name);
```

## Когда использовать ESQ

ESQ подходит, когда нужно:

- прочитать сущности с учётом схемы BPMSoft;
- пройти по lookup path (`Account.Name`, `Owner.Contact.Name`);
- получить `EntityCollection` или одну `Entity`;
- построить фильтры через платформенный API;
- сохранить читаемость для типовых CRUD/query-сценариев.

## Когда ESQ не лучший выбор

Лучше рассмотреть `Select` / `Update` / `Delete`, если:

- нужен сложный SQL с join/group/aggregate, который плохо читается в ESQ;
- выполняется массовое обновление без загрузки `Entity`;
- нужна низкоуровневая оптимизация;
- результат читается через `IDataReader`, а не через `Entity`.

Пример такого соседнего подхода виден в OCC routing: часть выборок делается через `Select`, а подсчёт открытых чатов по оператору - через ESQ.

## Базовые строительные блоки

| Блок | Для чего нужен |
| ----- | ----- |
| `EntitySchemaQuery` | корневой объект запроса |
| `AddColumn(...)` | добавить колонку или path |
| `PrimaryQueryColumn` | управление выборкой primary key |
| `Filters` | root collection фильтров |
| `CreateFilterWithParameters(...)` | простой фильтр с параметрами |
| `EntitySchemaQueryFilterCollection` | группа фильтров с `And`/`Or` |
| `GetEntity(...)` | получить одну запись |
| `GetEntityCollection(...)` | получить коллекцию |
| `GetSelectQuery(...)` | получить низкоуровневый `Select` из ESQ |

## Важные свойства ESQ

| Свойство | Практический смысл |
| ----- | ----- |
| `RowCount` | ограничить число строк |
| `IsDistinct` | убрать дубли в результирующем наборе |
| `UseAdminRights` | управлять применением прав |
| `IgnoreDisplayValues` | не тянуть display values, если они не нужны |
| `UseLocalization` | включать/выключать локализацию |
| `Cache` / `CacheItemName` | использовать кэш для повторяемых запросов |

## Основные правила

1. Добавляйте только нужные колонки.
1. Храните результат `AddColumn(...)`, если потом читаете значение через `GetTypedColumnValue(...)`.
1. Для lookup id чаще нужен путь `Lookup.Id`, а для raw FK - имя колонки `LookupId` уже на уровне `Entity`.
1. Для `OR` всегда создавайте отдельную `EntitySchemaQueryFilterCollection`.
1. Для сложных reverse-связей используйте bracket path осознанно и документируйте смысл пути.
1. Для больших выборок задавайте `RowCount`, сортировку и проверяйте права.
