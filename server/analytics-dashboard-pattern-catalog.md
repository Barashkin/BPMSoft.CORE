# Analytics Dashboard Pattern Catalog

<!-- Теги: analytics, dashboards, patterns, widgets -->

> Каталог рабочих Analytics / Dashboards / Metrics паттернов из
> `Autogenerated/Src`.

## Dashboard storage JSON

**Когда использовать:** нужно понять, почему виджет не появился.

**Паттерн:** `SysDashboard.Items` хранит item config, `ViewConfig` хранит
layout. Связка идет по `name`.

**Пример:** `AnalyticsServiceUtils.FetchDashboardItemsData`.

## Service stream response

**Когда использовать:** endpoint возвращает dashboard JSON.

**Паттерн:** WCF method возвращает `Stream`, helper выставляет
`application/json; charset=utf-8`.

**Пример:** `AnalyticsService.GetResponseStream`.

## Widget data factory

**Когда использовать:** добавить новый server widget type.

**Паттерн:** class с `[DashboardItemData("WidgetType")]` +
`IDashboardItemData`; factory ищет type в workspace assembly.

**Пример:** `DashboardItemDataFactory.Platform.cs`.

## Indicator aggregation

**Когда использовать:** нужен числовой KPI.

**Паттерн:** `IndicatorDashboardItemSelectBuilder` строит aggregation function,
читает `columnName`, `aggregationType`, `filterData`.

**Пример:** `IndicatorDashboardItemData.Platform.cs`.

## Dashboard grid widget

**Когда использовать:** нужен список записей на дашборде.

**Паттерн:** `DashboardGridModule` загружает entity schema из
`moduleConfig.entitySchemaName`, использует `DashboardListedGridViewModel`.

**Пример:** `DashboardGridModule.Platform.js`.

## Client widget registry

**Когда использовать:** добавить/найти view и designer module.

**Паттерн:** `DashboardEnums.WidgetType[code].view/design`.

**Пример:** `DashboardEnums.Platform.js`.

## Dashboard shell

**Когда использовать:** нужна страница дашбордов с вкладками.

**Паттерн:** `DashboardsModule` + `DashboardBuilder` + viewModel/viewConfig.

**Пример:** `DashboardsModule.Platform.js`.

## Favorites and dashboard ordering

**Когда использовать:** вкладки нужно искать, избрать или переупорядочить.

**Паттерн:** `FilterableList`, favorite button, drag-and-drop, edit mode,
`RightUtilities`.

**Пример:** `DashboardBuilder.Platform.js`.

## Chart drill-down

**Когда использовать:** график должен раскрывать детализацию.

**Паттерн:** `ChartDrillDownProvider` хранит history, filters, column captions,
data value types; server дает grid data.

**Пример:** `ChartDrillDownProvider.Platform.js`.

## Section dashboard binding

**Когда использовать:** виджет зависит от текущей записи секции.

**Паттерн:** `GetDashboardItemDataForSection` передает `bindingColumnValue` в
server data class.

**Пример:** `AnalyticsServiceUtils.GetDashboardItemDataForSection`.
