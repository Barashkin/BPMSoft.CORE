# Process Designer User Task Pages

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: ProcessDesigner, UserTaskPropertiesPage, ProcessUserTaskSchemaManager -->

> User task property pages связывают элемент процесса с user task schema,
> параметрами, source code edit и validation contract.

## RootUserTaskPropertiesPage

Файл: `Autogenerated/Src/RootUserTaskPropertiesPage.ProcessDesigner.js`.

Базовая страница user task наследуется от `ProcessFlowElementPropertiesPage` и
добавляет сообщения:

| Message | Направление | Роль |
| ------- | ----------- | ---- |
| `GetProcessElementInfo` | subscribe | вернуть process element и package UId |
| `SaveProcessElement` | publish | сохранить process element |
| `ValidateProcessElement` | publish | валидировать process element |
| `GetValue` | publish | получить значение source code edit |
| `GetSourceCodeData` | subscribe | отдать данные source code edit |
| `SourceCodeChanged` | subscribe | принять новое значение source code |

## AfterActivitySaveScript

Root page встраивает `SourceCodeEditPage` в модуль
`AfterActivitySaveScriptBody`.

Данные для редактора:

- `sourceCode` из `AfterActivitySaveScript`;
- `name` из user task;
- language `CSHARP`;
- marker `AfterActivitySaveScriptBody`.

Поле показывается только для user task schemas, где
`enableCustomEventHandlers = true`.

## Определение user task

`getIsUserTask` проверяет, что process element является:

- `BPMSoft.ProcessUserTaskSchema`;
- или `BPMSoft.ProcessSchemaUserTask`.

`getUserTaskSchema` получает schema:

- из `userTask.schema`, если она уже загружена;
- через `BPMSoft.ProcessUserTaskSchemaManager.forceGetInstanceByUId`;
- возвращает `null`, если `schemaUId` пустой.

## UserTaskPropertiesPage

Файл: `Autogenerated/Src/UserTaskPropertiesPage.ProcessDesigner.js`.

Страница специализирует выбор user task schema:

- `schemaManagerName = "ProcessUserTaskSchemaManager"`;
- `initSchema` загружает schema по `schemaUId`;
- `getSchemaListFilter` исключает схемы из `SysProcessUserTask`;
- `getSchemaList` вызывает `ProcessModuleUtilities.getSchemasByFilter`;
- `onOpenSchemaDesignerButtonClick` открывает user task designer;
- `getSubProcessSchemaInstance` использует `ProcessUserTaskSchemaManager`.

## Фильтр SysProcessUserTask

`getSchemaListFilter` строит ESQ по `SysProcessUserTask` и добавляет
`SysUserTaskSchemaUId` в `ExcludedSchemas`.

Так дизайнер скрывает user task schemas, которые не должны выбираться через
обычный список subprocess schemas.

## Synchronize parameters

При смене user task schema вызывается `synchronizeParameters`, затем обновляется
видимость `AfterActivitySaveScript`.

Если после выбора схемы параметры не обновились, проверяйте:

- загрузилась ли schema по `schemaUId`;
- корректно ли отработал `ProcessUserTaskSchemaManager`;
- не пустой ли mapping target после синхронизации.
