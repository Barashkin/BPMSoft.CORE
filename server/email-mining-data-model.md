# Email Mining Data Model

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: Activity, EnrchEmailData, EnrchTextEntity, EmailMining -->

> Модель Email Mining связывает email `Activity` с общим mined payload
> `EnrchEmailData` и детализацией extracted text entities.

## Activity extension

Файл: `Autogenerated/Src/ActivitySchema.EmailMining.cs`.

EmailMining расширяет `Activity` колонками:

| Колонка | Назначение |
| ------- | ---------- |
| `EnrchEmailData` | lookup на распарсенные данные письма |
| `EnrichStatus` | статус enrich/mining |

Также добавлен индекс `IX_Activity_SendDate`, важный для выборки актуальных
писем в `EmailMiningJob`.

## EnrchEmailData

Файл: `Autogenerated/Src/EnrchEmailDataSchema.EmailMining.cs`.

`EnrchEmailData` хранит общий результат разбора письма.

| Колонка | Назначение |
| ------- | ---------- |
| `JsonData` | полный response enrichment service |
| `Hash` | hash mined data, primary display column |
| `Status` | processing status, default `Mined` |

Практически важно: несколько email activities могут ссылаться на один
`EnrchEmailData`, если hash extracted data совпал.

## EnrchTextEntity

Файл: `Autogenerated/Src/EnrchTextEntitySchema.EmailMining.cs`.

`EnrchTextEntity` хранит отдельные найденные сущности из текста.

| Колонка | Назначение |
| ------- | ---------- |
| `EnrchEmailData` | родительский result |
| `Type` | тип extracted entity |
| `JsonData` | сырой JSON entity |
| `Hash` | hash entity |
| `Parent` | parent text entity |
| `DuplicationStatus` | статус дублирования |
| `Source` | источник |
| `HashVersion` | версия алгоритма hash |

`EnrchTextEntity.EnrchEmailData` cascade-linked к `EnrchEmailData`.

## Found / rejected / processed data

Дополнительные схемы пакета:

| Схема | Роль |
| ----- | ---- |
| `EnrchFoundContact` | найденные контакты для mined entity |
| `EnrchFoundAccount` | найденные аккаунты для mined entity |
| `EnrchRejectedData` | отклоненные элементы |
| `EnrchProcessedData` | обработанные элементы |
| `EnrchTypeMapping` | mapping типов |

Эти сущности отделяют raw extracted data от решений пользователя/системы:
что найдено, что сохранено, что отклонено.

## Hash as reuse boundary

В Email Mining есть два разных уровня hash:

- `Activity.MailHash` — дедупликация email sync;
- `EnrchEmailData.Hash` и `EnrchTextEntity.Hash` — reuse mined payload и
  идентификация extracted entities.

Не используйте `MailHash` для диагностики enriched data: он отвечает за другой
уровень pipeline.

## Status fields

`Activity.EnrichStatus` показывает статус обработки конкретного письма.
`EnrchEmailData.Status` показывает состояние общего mined payload.

Типичный сценарий:

1. `Activity.EnrichStatus = Inactive`.
2. `EmailMiningJob` выбирает письмо.
3. `EmailMiner` получает/создает `EnrchEmailData`.
4. `Activity.EnrichStatus = Done` или `Error`.
5. UI читает `Activity.EnrchEmailData.Status`.
