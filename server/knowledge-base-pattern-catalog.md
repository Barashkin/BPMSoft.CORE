# Knowledge Base Pattern Catalog

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: KnowledgeBase, SSP, CaseTerm, patterns -->

> Каталог рабочих Knowledge Base / Case Terms паттернов из `Autogenerated/Src`.

## Entity с HTML и plain text

Источник: `KnowledgeBaseSchema.Base.cs`.

`KnowledgeBase` хранит HTML в `Notes`, а текстовую версию в `NotHtmlNote`.
Это удобно для разных UI/search сценариев: карточка может показывать HTML, а
поиск и preview — plain text.

## Tags v2 override

Источники:

- `KnowledgeBasePageV2.UIv2.js`;
- `KnowledgeBaseSectionV2.UIv2.js`.

Карточка и секция переопределяют `initTags` и используют
`KnowledgeBaseTagV2` / `KnowledgeBaseInTagV2` вместо старых tag schemas.

## Like через Delete + Insert

Источник: `KnowledgeBasePageV2.UIv2.js`.

`setLikeIt` сначала удаляет текущий like пользователя по статье, затем при
необходимости создает новую запись. Это упрощает toggle и предотвращает
дублирование likes.

## Read-only portal page

Источник: `PortalKnowledgeBasePage.SspKnowledgeBase.js`.

Портальная карточка делает поля недоступными, удаляет лишние tabs/actions и
заменяет обычный file detail на `FileDetailReadOnly`.

## Portal search через Storage.Filters

Источник: `KnowledgeBaseSearchModule.SspKnowledgeBase.js`.

Поиск по порталу кладет `CustomFilters` в
`BPMSoft.configuration.Storage.Filters[sectionSchema]`, затем меняет route
через `PushHistoryState`.

## Popular articles как DashboardGridModule

Источник: `PopularKnowledgeBaseArticlesListModule.SspKnowledgeBase.js`.

Список популярных статей переиспользует dashboard grid infrastructure,
сортируется по aggregation `[Like:KnowledgeBase].Id` и ведет в KB section.

## Global Search row specialization

Источники:

- `KnowledgeBaseSearchRowSchema.GlobalSearch.js`;
- `CaseSearchRowSchema.GlobalSearch.js`.

Строки результата не выполняют поиск, а только подстраивают layout и набор
отображаемых колонок для сущности.

## DCM case settings через sandbox

Источник: `SectionWizardCasesSettings.DcmLibrary.js`.

Wizard step координирует DCM settings и DCM library через PTP/BROADCAST
messages, а reload получает из `BPMSoft.ServerChannel`.

## Strategy chain для сроков

Источники:

- `CaseTermStrategyQueue.Calendar.cs`;
- `CaseTermIntervalSelector.Calendar.cs`.

Стратегии сроков задаются в `DeadlineCalcSchemas`: default strategy начинает
цепочку, `AlternativeRule` задает fallback, selector идет по очереди до
полного набора response/resolve сроков.

## Диагностика сроков через store

Источники:

- `TermCalculationLogStore.Calendar.cs`;
- `TermCalculationLogger.Calendar.cs`.

Лог расчета хранит strategy rules, выбранную strategy, active intervals,
calendar data, calculation intervals и timezone conversions. Это основной
источник объяснения результата расчета срока.
