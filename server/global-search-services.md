# Global Search Services

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: GlobalSearchService, GlobalSearchHelper, DTO, rights -->

> `GlobalSearchService` — WCF proxy для глобального поиска. Он принимает
> клиентский запрос, фильтрует доступные типы по правам, вызывает внешний
> поиск и собирает `BpmSearchResponse`.

## GlobalSearchService

Файл: `Autogenerated/Src/GlobalSearchService.GlobalSearch.cs`.

Ключевые признаки:

- `[ServiceContract]`, `[DefaultServiceRoute]`, `[SspServiceRoute]`.
- Наследование от `BaseService` и `IReadOnlySessionState`.
- Метод `Search` опубликован через `WebInvoke(Method = "POST")`.
- URL берется из `SysSettings.GlobalSearchUrl`.
- Таймаут берется из `SysSettings.DataServiceQueryTimeout`.
- Для HTTP используется `RestSharp`.
- Логгер: `LogManager.GetLogger("GlobalSearch")`.
- Метрики ошибок пишутся через `IMetricReporter`.

Перед вызовом внешнего поиска сервис фильтрует параметр `type` через
`UserConnection.DBSecurityEngine.GetIsEntitySchemaReadingAllowed`. Поэтому
наличие сущности в клиентском фильтре не гарантирует, что она попадет в
запрос к поисковому сервису.

## GlobalSearchHelper

Файл: `Autogenerated/Src/GlobalSearchHelper.GlobalSearch.cs`.

`GlobalSearchHelper` отвечает за серверную обработку результата:

- формирует `SearchRequestQuery` для старого ES endpoint;
- проверяет record-level права;
- отдельно обрабатывает file entities и master record rights;
- собирает `BpmSearchResponseEntity`;
- добавляет значения колонок, lookup values и highlights;
- при `UseLocalizableGlobalSearchResult` дочитывает локализованные значения
  через `EntitySchemaQuery`;
- использует `GlobalSearchColumnUtils` для alias и имен колонок.

Основные настройки:

| SysSetting | Назначение |
| ---------- | ---------- |
| `GlobalSearchShouldMatchPercent` | процент совпадения для запроса |
| `UseInexactGlobalSearch` | включает неточный поиск |
| `UseLocalizableGlobalSearchResult` | дочитывает локализованные значения |
| `GlobalSearchDefaultPrimaryColumnWeight` | вес primary display column |
| `GlobalSearchDefaultEntityWeight` | вес сущности |

## DTO ответа

Файл: `Autogenerated/Src/GlobalSearchDto.GlobalSearch.cs`.

Типовой ответ содержит:

- `Success` и `ErrorInfo`;
- `Took`, `Total`, `NextFrom`;
- `Data` со списком `BpmSearchResponseEntity`;
- `EntityName`, `Id`, `ColumnValues`, `FoundColumns`;
- aggregation данные для фильтров.

`NextFrom` рассчитывается по фактической позиции последней доступной записи в
ответе ES. Это важно, потому что часть найденных записей может быть отброшена
после проверки прав.

## Ошибки сервиса

`GlobalSearchService` возвращает сериализованный `BaseResponse` с
`Success = false`, если:

- `RestSharp` вернул статус не `Completed`;
- внешний ответ не содержит search result;
- запрос был отменен клиентом;
- истек таймаут;
- внешний ES/GS сервис вернул ошибку.

Для диагностики см. [global-search-troubleshooting.md](global-search-troubleshooting.md).
