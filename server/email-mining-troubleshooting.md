# Email Mining Troubleshooting

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: EmailMining, troubleshooting, mailbox, enrichment -->

> Диагностика Email Mining должна идти по pipeline: mailbox sync, Activity
> selection, job, cloud proxy, persistence, UI.

## Письма не попадают в mining

Проверьте:

- `EnableEmailMining = true`;
- `EmailMiningPeriodMin` больше 0;
- `EmailMiningPackageSize` больше 0;
- письмо имеет `Activity.Type = email`;
- `Activity.Body` непустой;
- `Activity.EnrichStatus = Inactive`;
- `Activity.SendDate` попадает в `EmailMiningIdentificationActualPeriod`;
- есть `EmailMessageData` с role `To`, `Cc` или `Bcc`.

Файл для проверки условий: `Autogenerated/Src/EmailMiningJob.EmailMining.cs`.

## Job не запускается повторно

Проверьте:

- `EmailMiningAppListener` отработал на `OnAppStart`;
- группа job `EmailMiningJob` есть в scheduler;
- `EmailMiningPeriodMin != 0`;
- в логах нет `IncorrectConfigurationException`;
- при configuration error следующий запуск сдвигается на 60 минут.

Связанные документы:

- [Scheduler Quartz](scheduler-quartz.md)
- [Quartz Troubleshooting](quartz-troubleshooting.md)

## Ошибка конфигурации cloud parsing

Проверьте SysSettings:

- URL text parsing service из `EmailMiningConsts.TextParsingServiceSysSetting`;
- `CloudServicesAPIKey`;
- отсутствие пробелов в API key.

Типичные признаки:

- HTTP `Unauthorized` — неверный API key;
- HTTP `NotFound` — неверный service URL;
- `UriFormatException` — некорректный URL;
- malformed batch response — несовместимая версия Text Mining API;
- request/response count mismatch — поврежденный batch response.

Файл: `Autogenerated/Src/EmailMiningServiceProxy.EmailMining.cs`.

## Activity остается в Error

Проверьте:

- логи `EmailMining`;
- exception в `EmailMiner.ProcessEntities`;
- доступность `EnrchEmailData` и `EnrchTextEntity`;
- валидность JSON response;
- нет ли ошибок в `ContactSearcher`, `AccountSearcher`, `Deduplicator`.

`EmailMiner` обновляет `Activity.EnrichStatus` и может записать `Error`, если
cloud response failure.

## Cloud icon не отображается в email item

Проверьте:

- `Activity.EnrchEmailData` заполнен;
- `EnrchEmailData.Status != ENRICHED`;
- `EnrichContactService.GetCloudStateForEmail` возвращает data;
- `EmailItemSchema` вызвал `getContactsFromServer`;
- `EmailEnrichmentMixin.setCloudVisibility` получил непустой result.

Файлы:

- `Autogenerated/Src/EmailItemSchema.EmailMining.js`;
- `Autogenerated/Src/EmailEnrichmentMixin.EmailMining.js`;
- `Autogenerated/Src/EnrichContactService.EmailMining.cs`.

## Contact не находится

Проверьте:

- extracted emails в `EnrchTextEntity.JsonData`;
- `ContactCommunication.Number`;
- имя в `Contact.Name`;
- `GivenName` и `Surname`;
- phone normalization через `IPhoneNumberComparer`.

Файл: `Autogenerated/Src/ContactSearcher.EmailMining.cs`.

## Account не находится

Проверьте:

- extracted organization name;
- `Account.Name`;
- `Account.AlternativeName`;
- домены в `AccountCommunication.Number`;
- что домен длиннее 2 символов;
- что нужный вариант URL входит в поддерживаемые формы.

Файл: `Autogenerated/Src/AccountSearcher.EmailMining.cs`.

## Данные предлагаются как новые, хотя уже есть

Проверьте dedup:

- найден ли contact однозначно;
- корректны ли `ContactCommunication`;
- phone type входит в `ComTypebyCommunication` с code `Phone`;
- `SearchNumber` заполнен для телефонов;
- `EnrchTextEntity.DuplicationStatus`.

Файл: `Autogenerated/Src/Deduplicator.EmailMining.cs`.

## Mailbox sync остановлен

Проверьте:

- `MailboxSyncSettings.EnableMailSynhronization`;
- `SynchronizationStopped`;
- `RetryCounter`;
- `ErrorCode` и `LastError`;
- включен ли `EnableReSynchronizationMechanism`;
- период `MailReSynchronizationFrequency`.

Файл: `Autogenerated/Src/MailSyncJob.MailSync.cs`.

Если письмо не создано как `Activity`, Email Mining его не увидит.
