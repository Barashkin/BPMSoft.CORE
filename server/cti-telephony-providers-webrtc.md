# CTI Telephony Providers And WebRTC

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: CTI, WebRTC, provider, JsSIP -->

> Provider слой отделяет CTI-панель от конкретной телефонии. В
> `Autogenerated/Src` есть WebRTC providers, виртуальный provider и
> provider-specific страницы настройки.

## CtiProviderInitializer

Файл: `Autogenerated/Src/CtiProviderInitializer.CTIBase.js`.

Инициализатор получает `ctiProviderName` из `SysMsgLib` и:

1. Проверяет `BPMSoft[ctiProviderName]`.
2. Если provider еще не загружен, вызывает `require([ctiProviderName])`.
3. Передает provider в `initCustomCtiProvider`.

`initCustomCtiProvider` является extension point: по умолчанию он просто
возвращает найденный provider, но может быть переопределен для кастомной
инициализации.

## BaseWebRTCProvider

Файл: `Autogenerated/Src/BaseWebRTCProvider.WebRTCCore.js`.

`BaseWebRTCProvider` наследуется от `BPMSoft.BaseCtiProvider` и описывает
общую WebRTC-логику:

- `activeCall` и `consultCall` как `BPMSoft.Telephony.Call`;
- `webRTC` как adapter;
- `webRTCSession` и `webRTCConsultSession`;
- `msgUtilServiceUrl = /ServiceModel/MsgUtilService.svc/`;
- login method `LogInMsgServer`;
- update method `UpdateCall`;
- feature set для hold, consult call, transfer, drop, DTMF, mute;
- блокировка обновления страницы через `PageUpdateBlocker`;
- звуки звонка из `WebRTCOutRing`, `WebRTCInRing`, `WebRTCCancelRing`.

При `init` provider вызывает `LogInMsgServer`, передает `LicInfoKeys` и
`UserUId`, получает конфигурацию подключения и создает `WebRTCAdapter`.

## WebRTC adapter stack

Файлы:

- `Autogenerated/Src/WebRTCAdapter.WebRTCCore.js`;
- `Autogenerated/Src/WebRTCSession.WebRTCCore.js`;
- `Autogenerated/Src/JsSipLibrary.WebRTCCore.js`;
- `Autogenerated/Src/WebRTCObservable.WebRTCCore.js`.

Этот слой отвечает за SIP/WebSocket сессии и локальные события WebRTC. В
документации важно не смешивать:

- `MsgUtilService.svc` — серверный сервис платформы;
- JsSIP/WebSocket — транспорт до телефонии;
- sandbox messages — клиентская шина UI.

## Provider-specific modules

Репрезентативные providers:

- `UisCtiProvider.WebRTCCore.js`;
- `InfinityCtiProvider.WebRTCCore.js`;
- `WbCtiProvider.WbCollaborations.js`;
- `VirtualCtiProvider.CTIVirtualProvider.js`.

`VirtualCtiProvider` полезен как reference для тестовой или demo-телефонии:
он эмулирует звонки без внешней АТС.

## TelephonyIntegration

Файл: `Autogenerated/Src/TelephonyIntegrationSchema.WebRTCCore.cs`.

`TelephonyIntegration` хранит административные настройки интеграции:

- `SysMsgLib`;
- `TelephonyIntegrationType`;
- `TelephonyAuthType`;
- `WebServiceURL`;
- `Login`, `Password`, `Token`;
- `TelephonyServerTimeZone`;
- `EnableCallRecordingFeature`;
- `CallRecordLinkUrlTemplate`;
- event endpoint fields: `CallCame`, `CallStarted`, `CallEnded`,
  `CallTransfered`, `CallOnHold`, `CallOnUnhold`;
- UIS time shifts для начала и окончания звонка.

UI администрирования:

- `TelephonyIntegrationSection.WebRTCCore.js`;
- `TelephonyIntegrationPageV2.WebRTCCore.js`;
- `TelephonyIntegrationPage.WebRTCCore.js`.
