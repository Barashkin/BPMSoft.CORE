# DCM Data Model And Settings

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: DCM, SysDcmSettings, SysDcmSchemaInSettings, VwSysDcmLib -->

> DCM связывает секцию, колонку стадии, фильтры и активные DCM-схемы через
> `SysDcmSettings`, `SysDcmSchemaInSettings` и views библиотеки.

## Основные сущности

| Сущность/view | Назначение |
| ------------- | ---------- |
| `SysDcmSettings` | настройки DCM для section entity |
| `SysDcmSchemaInSettings` | связь DCM-схем с настройками |
| `SysModuleDcmSettings` | module-level настройки |
| `SysDcmUserTask` | метаданные user task для DCM |
| `VwSysDcmLib` | view библиотеки DCM-схем |
| `VwSysDcmLibFolder` | папки библиотеки |
| `VwSysDcmLibInFolder` | связь схем и папок |

## SysDcmSettings

Файл: `Autogenerated/Src/SysDcmSettingsSchema.Base.cs`.

Ключевые колонки:

| Колонка | Роль |
| ------- | ---- |
| `SysModuleEntity` | секция/объект, к которому привязаны DCM-настройки |
| `StageColumnUId` | UId колонки стадии в entity schema |
| `Filters` | JSON-фильтры применимости DCM-схем |
| `DefaultSchemaUId` | схема DCM по умолчанию |

`SysModuleEntity` cascade-связан с секцией. Если секция удаляется или
переопределяется, настройки DCM нужно проверять отдельно.

## SysDcmSchemaInSettings

`SysDcmSchemaInSettings` связывает DCM-схемы с конкретным набором настроек.
Эта связь используется библиотекой и мастером секции, чтобы понять, какие DCM
case schemas доступны в выбранной секции.

## VwSysDcmLib

`DcmSchemaManager.entitySchemaName = "VwSysDcmLib"`.

Это значит, что клиентский менеджер схем работает через view библиотеки, а не
напрямую через физическую таблицу. Для диагностики списка DCM-схем проверяйте:

- данные view `VwSysDcmLib`;
- package/workspace;
- active version flags;
- связь через `SysDcmSchemaInSettings`;
- фильтры по `SysDcmSettings`.

## StageColumnUId

`StageColumnUId` указывает колонку секционной сущности, где хранится текущая
стадия записи.

Если stage column некорректная:

- DCM может открыться, но стадии не будут применяться к записи;
- фильтры могут считаться несовместимыми;
- wizard может не сохранить настройки.

## Filters

`Filters` хранит условия применимости DCM-схем. `DcmSchemaManager` валидирует,
что filter column не конфликтует со stage column и что фильтры активных схем
не дублируются.

## Settings managers

Клиентские managers:

- `SysDcmSettingsManager.DcmDesigner.js`;
- `SysDcmSchemaInSettingsManager.DcmDesigner.js`;
- `SysDcmSettingsManagerItem.DcmDesigner.js`;
- `SysDcmSchemaInSettingsManagerItem.DcmDesigner.js`.

Они загружаются в `SectionDcmSettings` и используются мастером секции.

## Практические правила

- Для поиска DCM-схем начинайте с `VwSysDcmLib`.
- Для привязки к секции проверяйте `SysDcmSettings.SysModuleEntity`.
- Для отображения стадии проверяйте `StageColumnUId`.
- Для конфликтов активных схем проверяйте `Filters` и уникальность условий.
