# ESQ Troubleshooting

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: ESQ, troubleshooting, EntitySchemaQuery, filters, performance -->

> Практический checklist для диагностики проблем с `EntitySchemaQuery`: пустые результаты, неверные alias, path-ошибки, права и производительность.

## Быстрый checklist

Если ESQ работает не так, проверьте:

1. правильное имя root schema;
1. добавлены ли все нужные колонки;
1. читаете ли значение по правильному alias;
1. не превратились ли фильтры в `AND` вместо ожидаемого `OR`;
1. корректен ли lookup/bracket path;
1. не скрыли ли результат права доступа;
1. не тянете ли лишние display values или все колонки;
1. задан ли `RowCount` для поиска/списка.

## Симптом: коллекция пустая

### Возможные причины default-значений

- root schema выбрана не та;
- фильтр стоит на display value вместо id;
- root `Filters` работает как `AND`, хотя ожидался `OR`;
- user rights скрывают записи;
- bracket path описывает не ту связь;
- `UseAdminRights` настроен не так, как требуется сценарию.

### Что проверить при пустой коллекции

- временно упростить фильтры до одного условия;
- проверить `Lookup.Id` вместо `Lookup.Name`;
- вынести `OR` в `EntitySchemaQueryFilterCollection`;
- проверить тот же сценарий через `Select` или SQL на стенде;
- посмотреть `GetSelectQuery(userConnection)`, если нужно понять итоговый SQL.

## Симптом: `GetTypedColumnValue` возвращает default

### Возможные причины

- колонка не добавлена в ESQ;
- чтение идёт по исходному path, а не по alias;
- колонка-функция или bracket path получила generated name;
- primary column скрыта через `PrimaryQueryColumn.IsVisible = false`.

### Решение

Сохраняйте результат `AddColumn(...)`:

```csharp
var column = esq.AddColumn("[BPMSoftOCCOperatorUnit:SysAdminUnit:Id].Nickname");
string value = entity.GetTypedColumnValue<string>(column.Name);
```

Для агрегатов это обязательно:

```csharp
var countColumn = esq.AddColumn(esq.CreateAggregationFunction(AggregationTypeStrict.Count, "Id"));
int count = entity.GetTypedColumnValue<int>(countColumn.Name);
```

## Симптом: OR-фильтр не работает

Root `esq.Filters` не является OR-группой по умолчанию.

Правильно:

```csharp
var filters = new EntitySchemaQueryFilterCollection(esq, LogicalOperationStrict.Or);
filters.Add(esq.CreateFilterWithParameters(FilterComparisonType.Equal, "CreatedBy", contactId));
filters.Add(esq.CreateFilterWithParameters(FilterComparisonType.Equal, "IsShared", true));
esq.Filters.Add(filters);
```

Reference: `CommandLineService.NUI.cs`.

## Симптом: path не находится или ломается

### Что проверить при ошибке path

- root schema;
- имя lookup-колонки, а не display caption;
- направление связи;
- наличие нужной reference schema;
- правильность bracket notation.

Если связь обратная, обычный `Lookup.Column` не подойдёт. Используйте:

```text
[RelatedSchema:RelatedColumn:CurrentColumn].TargetColumn
```

## Симптом: запрос слишком тяжёлый

### Проверить

- нет ли `AddAllSchemaColumns()`;
- не тянутся ли display values без необходимости;
- задан ли `RowCount`;
- нет ли сложных lookup path в цикле;
- не строится ли ESQ внутри вложенного foreach;
- не лучше ли заменить сценарий на `Select`/`Update`.

## Симптом: данные видны администратору, но не пользователю

Проверьте `UseAdminRights` и права на записи.

Примеры, где права выставляются осознанно:

- `ESNFeedModuleService.ESN.cs`
- `IndexingEntityListBuilder.GlobalSearch.cs`

Не отключайте права просто ради “починить выборку”. Для пользовательских сервисов это может стать ошибкой безопасности.

## Симптом: первая запись нестабильна

Если используется `RowCount = 1`, должна быть сортировка:

```csharp
esq.RowCount = 1;
esq.AddColumn("Position").OrderByAsc();
```

Без `OrderByAsc/Desc` результат зависит от плана БД.

## Симптом: после изменения фильтра результат не меняется

Если ESQ уже строил select query, может понадобиться:

```csharp
esq.ResetSelectQuery();
```

Reference: `AccountSearcher.EmailMining.cs`, где меняется left expression фильтра и повторно выполняется запрос.

## Симптом: `AddAllSchemaColumns` работает, но медленно

Это ожидаемо для больших схем. Перепишите выборку на явные колонки.

Оставлять `AddAllSchemaColumns` разумно только для generic loaders, где документированно нужна вся сущность.

## Когда смотреть SQL

Используйте:

```csharp
Select select = esq.GetSelectQuery(userConnection);
```

Это полезно для:

- сложных path;
- прав;
- производительности;
- проверки того, какие join'ы реально построены.
