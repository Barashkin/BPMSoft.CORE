> Краткий каталог reusable patterns из Email Mining / Enrichment.

## App listener registers class job

Источник: `Autogenerated/Src/EmailMiningAppListener.EmailMining.cs`.

Паттерн:

1. `OnAppStart`.
2. `DeleteOldJobs(group)`.
3. `ScheduleImmediateJob<TJob>(group, workspace, userName, parameters, true)`.

Используется, когда job должна стартовать сразу после поднятия приложения.

## Self-rescheduling job

Источник: `Autogenerated/Src/EmailMiningJob.EmailMining.cs`.

Job после выполнения сам планирует следующий запуск через
`SchedulerUtils.ScheduleNextRun`. Период берется из SysSettings, а некорректная
конфигурация может добавить delay.

Такой паттерн удобен, когда интервал должен изменяться без перезапуска сервиса.

## Batch cloud proxy

Источник: `Autogenerated/Src/EmailMiningServiceProxy.EmailMining.cs`.

Паттерн:

- проверить URL и API key;
- собрать request array;
- вызвать RestSharp endpoint;
- проверить HTTP status;
- десериализовать response;
- проверить соответствие числа request/response;
- залогировать summary batch.

## Hash-based reuse

Источник: `Autogenerated/Src/EmailMiner.EmailMining.cs`.

Mined response получает hash. Если `EnrchEmailData` с таким hash уже есть,
новое письмо переиспользует существующий payload.

Плюсы:

- меньше дублирования данных;
- повторные письма быстрее связываются с known enriched data.

Риск:

- при изменении алгоритма hash нужен `HashVersion`/hash updater контур.

## ESQ searcher services

Источники:

- `Autogenerated/Src/ContactSearcher.EmailMining.cs`;
- `Autogenerated/Src/AccountSearcher.EmailMining.cs`.

Searchers изолируют ESQ-фильтры от orchestration. Это упрощает тестирование и
не засоряет `EmailMiner`.

## Dedup via domain rules

Источник: `Autogenerated/Src/Deduplicator.EmailMining.cs`.

Dedup не удаляет mined data. Он помечает `EnrchTextEntity.DuplicationStatus`,
чтобы UI мог скрыть или ослабить already known values.

## WCF facade over helper

Источник: `Autogenerated/Src/EnrichContactService.EmailMining.cs`.

Сервис тонкий:

- принимает REST/WCF request;
- делегирует в `EnrichContactHelper`;
- возвращает DTO для UI.

Такой фасад держит transport concerns отдельно от enrichment logic.

## Client mixin for action injection

Источник: `Autogenerated/Src/EmailEnrichmentMixin.EmailMining.js`.

Mixin добавляет actions в существующий email item без копирования всей схемы.
Он расширяется через virtual methods:

- `getAdditionalWindowConfig`;
- `getAdditionalActionsRequestConfig`;
- `getEnrichmentButtonMenuItem`.

## Settings as operational control

SysSettings управляют runtime behavior:

| Настройка | Что контролирует |
| --------- | ---------------- |
| `EnableEmailMining` | включенность mining |
| `EmailMiningPeriodMin` | период job |
| `EmailMiningPackageSize` | размер batch |
| `EmailMiningIdentificationActualPeriod` | окно актуальности |
| `CloudServicesAPIKey` | доступ к cloud services |
| `MailReSynchronizationFrequency` | resync stopped mailboxes |
| `EnableReSynchronizationMechanism` | включенность resync job |
