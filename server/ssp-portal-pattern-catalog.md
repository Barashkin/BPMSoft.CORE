# SSP Portal Pattern Catalog

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: SSP, Portal, patterns, access rights -->

> Каталог рабочих SSP / Portal паттернов из `Autogenerated/Src`.

## Portal user service

**Когда использовать:** нужно создать или пригласить пользователей портала.

**Паттерн:** `SspUserManagementService` + `CheckCanManageSspUsers` +
`SspUserManagementServiceHelper`.

**Пример:** `SspUserManagementService.SSP.cs`.

## Admin portal user CRUD

**Когда использовать:** администратор меняет portal user из UI.

**Паттерн:** partial `AdministrationService`, `SystemUserConnection`,
проверка `ConnectionType = SSP`, operation `CanAdministratePortalUsers`.

**Пример:** `AdministrationServicePortalUsers.SSP.cs`.

## Portal schema ACL

**Когда использовать:** объект должен быть доступен portal users.

**Паттерн:** запись в `SysSSPEntitySchemaAccessList` /
`PortalSchemaAccessList` по schema UId.

**Пример:** `SysSSPEntitySchemaAccessListSchema.Base.cs`.

## Portal column ACL

**Когда использовать:** портал должен видеть только часть полей.

**Паттерн:** `PortalColumnAccessList` с `PortalSchemaList`, `ColumnUId`,
`ColumnName`.

**Пример:** `PortalColumnAccessListSchema.NUI.cs`.

## SSP Global Search filter

**Когда использовать:** Global Search должен уважать portal ACL.

**Паттерн:** `GlobalSearchSSPHelper` фильтрует entities, aggregations,
columns и кеширует allowed columns.

**Пример:** `GlobalSearchSSPHelper.SSP.cs`.

## Portal section binding

**Когда использовать:** секцию нужно опубликовать в портал.

**Паттерн:** `PortalSectionStructureBuilder` добавляет `JOIN
SysModuleEntityInPortal`, general builder исключает portal entities.

**Пример:** `PortalSectionHelper.SSP.cs`.

## Portal main page builder

**Когда использовать:** нужна главная страница портала.

**Паттерн:** `PortalMainPageBuilder` наследует dashboard builder, скрывает
settings для SSP и проверяет admin operations для не-SSP.

**Пример:** `PortalMainPageBuilder.SSP.js`.

## Self-registration

**Когда использовать:** пользователь регистрируется сам.

**Паттерн:** `UserManagementService.Register`, `SystemUserConnection`,
`ShowPortalSelfRegistrationLink`, contact creation.

**Пример:** `UserManagementService.SSP.cs`.

## External access log

**Когда использовать:** нужно отследить внешний/изолированный доступ.

**Паттерн:** `ExternalAccessRequestLog` с requester, URL, customer id,
external access id.

**Пример:** `ExternalAccessRequestLogSchema.ExternalAccess.cs`.
