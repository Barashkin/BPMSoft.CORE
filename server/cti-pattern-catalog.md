> Каталог рабочих CTI / Telephony паттернов из `Autogenerated/Src`.

## Инициализация CTI через SysValue

Источник: `CtiBaseHelper.CTIBase.js`.

Паттерн:

1. Проверить `BPMSoft.SysValue.CTI`.
2. Если уже инициализировано, вернуть cached settings.
3. Иначе прочитать `SysSettings`.
4. Дочитать `SysMsgLib` и `SysMsgUserSettings`.
5. Проверить лицензии.
6. Сохранить результат обратно в `BPMSoft.SysValue.CTI`.

## Динамическая загрузка provider

Источник: `CtiProviderInitializer.CTIBase.js`.

`ctiProviderName` из `SysMsgLib` используется как AMD module name. Если
provider еще не загружен, вызывается `require([ctiProviderName])`.

## Лицензии из SysMsgLib

Источники:

- `SysMsgLibSchema.Base.cs`;
- `CtiBaseHelper.CTIBase.js`;
- `CtiRightsService.CTIBase.cs`.

`LicOperations` хранится строкой с разделителем `;`. Клиент кэширует результат
проверки в `BPMSoft.configuration.Storage.UserOperationLicense`.

## WebRTC provider поверх BaseCtiProvider

Источник: `BaseWebRTCProvider.WebRTCCore.js`.

Provider наследует базовый CTI-контракт, но инкапсулирует WebRTC specifics:
adapter, sessions, sounds, MsgUtil login, call feature set и page blocker.

## Provider-specific extension

Источники:

- `UisCtiProvider.WebRTCCore.js`;
- `InfinityCtiProvider.WebRTCCore.js`;
- `WbCtiProvider.WbCollaborations.js`;
- `VirtualCtiProvider.CTIVirtualProvider.js`.

Общая логика остается в base provider, а vendor specifics выносятся в
отдельный provider module.

## Active call vs persisted call

Источники:

- `SysActiveCallSchema.Base.cs`;
- `CallSchema.Base.cs`.

`SysActiveCall` хранит текущую линию и runtime state, `Call` — постоянную
историю звонка. Эта граница помогает не смешивать текущую сессию с
завершенной записью.

## Идентификация через UpdateQuery

Источник: `CtiPanelIdentificationUtilities.CTIBase.js`.

После выбора абонента CTI-панель сохраняет связанные поля в `Call` через
`UpdateQuery`, а не через полную перезагрузку карточки.

## История через VwRecentCall

Источник: `CtiPanelCommunicationHistoryUtilities.CTIBase.js`.

Панель читает историю из view `VwRecentCall`, ограничивает `rowCount` настройкой
`CommunicationHistoryRowCount` и фильтрует по текущему contact пользователя.

## Context call из секции

Источник: `ContextCallUtilities.CTIBase.js`.

Номер телефона в карточке/секции может инициировать звонок через message в
CTI-панель с передачей `entitySchemaName`, `entityId` и номера.

## Записи разговоров через provider handler

Источники:

- `TelephonyCallRecordServiceHelper.WebRTCCore.cs`;
- `UisCallRecordServiceHandler.WebRTCCore.cs`.

Общий helper работает с `TelephonyIntegration`, а provider-specific handler
реализует HTTP детали внешней АТС.

## Page update blocker

Источник: `PageUpdateBlocker.WebRTCCore.js`.

На время WebRTC сессии обновление страницы блокируется, чтобы случайный reload
не оборвал звонок.
