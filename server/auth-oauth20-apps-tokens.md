# Auth OAuth20 Apps And Tokens

<!-- Версия: 1.3 | Обновлено: 2026-04-28 | Платформа: BPMSoft 1.9 -->
<!-- Теги: auth, OAuth20, OAuthClientApp, tokens, IntegrationTools -->

> OAuth 2.0 в Base представлен двумя близкими контурами: platform client apps
> (`OAuthClientApp`, `OAuthConfigService`) и integration apps
> (`OAuthApplications`, `OAuth20AppPage`).
> OAuth20 как auth type в Web Service V2 относится к Integration Tools; см.
> [webservice-v2-overview.md](webservice-v2-overview.md).

Для подробного dive по OAuth20Integration см.
[auth-oauth20-integration-overview.md](auth-oauth20-integration-overview.md):
там отдельно описаны data model, platform clients, integration apps,
`ServiceOAuthAuthenticatorEndpoint`, границы и troubleshooting.

## OAuthConfigService

Файл: `Autogenerated/Src/OAuthConfigService.OAuth20.cs`.

`OAuthConfigService` — WCF-сервис для управления OAuth client/resource.

Ключевые методы:

| Метод | Назначение |
| ----- | ---------- |
| `AddDefaultResource` | создает default `OAuthResource` |
| `AddClient` | создает `OAuthClientApp` |
| `UpdateClient` | обновляет client app |
| `DeleteClient` | удаляет client app |

Все методы проверяют:

```text
GlobalAppSettings.FeatureEnableOAuth20Integration
```

Если feature выключена, сервис возвращает error response с
`OAuthDisabledMessage`.

## OAuthClientApp

Файл: `Autogenerated/Src/OAuthClientAppSchema.OAuth20.cs`.

Ключевые колонки:

| Колонка | Назначение |
| ------- | ---------- |
| `Name` | имя приложения |
| `ClientId` | OAuth client id |
| `ClientSecret` | secure text client secret |
| `RedirectUrl` | redirect URI |
| `ApplicationUrl` | URL приложения |
| `IsActive` | активность клиента |
| `SystemUser` | системный пользователь |
| `Description` | описание |

`OAuthConfigService.AddClient` дополнительно проверяет, что `SystemUserId`
указывает на `SysAdminUnit` типа system user (`SysAdminUnitTypeValue = 4`).

## OAuthClientAppListener

Файл: `Autogenerated/Src/OAuthClientAppListener.OAuth20.cs`.

`OAuthClientAppListener` слушает `OAuthClientApp`:

- `OnInserting` — проверяет `CanManageSolution`, находит default resource,
  вызывает `IdentityServiceWrapper.AddClient`, записывает `ClientId` и
  `ClientSecret`;
- `OnInserted` — создает `OAuthResourceInClient`;
- `OnUpdating` — вызывает `IdentityServiceWrapper.UpdateClient`;
- `OnDeleting` — вызывает `IdentityServiceWrapper.RemoveClient`.

Ошибки API оборачиваются через `BaseOAuthApiCaller` и localizable messages.

## Resources and scopes

Связанные схемы:

- `OAuthResourceSchema.OAuth20.cs`;
- `OAuthResourceInClientSchema.OAuth20.cs`;
- `OAuthAppScopeSchema.OAuth20Integration.cs`;
- `OAuthResourceListener.OAuth20.cs`;
- `OAuthResourceInClientListener.OAuth20.cs`.

`OAuthResourceInClient` связывает OAuth client с resource/scope.

## Token storage

Файл: `Autogenerated/Src/OAuthTokenStorageSchema.Base.cs`.

`OAuthTokenStorage` хранит пользовательские токены:

| Колонка | Назначение |
| ------- | ---------- |
| `SysUser` | пользователь |
| `OAuthApp` | OAuth application |
| `UserAppLogin` | внешний логин |
| `AccessToken` | access token |
| `ExpiresOn` | expiration |
| `RefreshToken` | refresh token |

`OAuthApp` cascade, поэтому токены удаляются вместе с приложением.

## Integration app UI

Файл: `Autogenerated/Src/OAuth20AppPage.OAuth20Integration.js`.

Страница `OAuth20AppPage` работает с `OAuthApplications` и требует operation:

```text
SecurityOperationName = "CanManageSolution"
```

Важные поля:

- `Name`;
- `SecretKey`;
- `AuthorizeUrl`;
- `TokenUrl`;
- `UseSharedUser`;
- `CredentialsLocationInRequest`;
- `AccessType`;
- `OAuthAppScopeDetail`;
- `OAuthAppUserDetail`.

Метод `_loginUser` получает authorization grant URL через
`ServiceOAuthAuthenticatorEndpointHelper.getAuthorizationGrantUrl` и открывает
его в новом окне.

## Граница с Integration Tools

`ServiceDesigner` может показывать auth type `OAuth20` для Web Service V2, если
включён feature flag `WebServiceOAuth20Auth`. Это настройка credentials
исходящего service schema call, а не самостоятельная OAuth application
registration.

Подробности: [Web Service V2 Overview](webservice-v2-overview.md) и
[Integration Tools Boundaries](integration-tools-boundaries.md).

## Граница с Office 365 mailbox OAuth

`Office365OAuthAuthenticator` относится к почтовому Exchange-контуру: он
получает Microsoft token, создаёт `MailboxSyncSettings` и связанные настройки
контактов/календаря. Это не общий OAuthConfigService flow.

Подробности: [Mailbox Exchange OAuth Office365](mailbox-exchange-oauth-office365.md).

## OAuthAuthenticationModule

Файл: `Autogenerated/Src/OAuthAuthenticationModule.NUI.js`.

Модуль читает параметры из URL hash, берет `socialNetworkName` и вызывает:

```text
../rest/SocialNetworksUtilitiesService/GetOAuthTokens
```

При успехе делает redirect на значение response header `location`.

Это social/external account сценарий, не общий login pipeline платформы.
