> Каталог рабочих паттернов из Notifications / Reminders.

## App listener as composition root

Источник: `Autogenerated/Src/NotificationEventListener.NUI.cs`.

Паттерн:

1. На `OnAppStart` зарегистрировать DI bindings.
2. Запланировать periodic jobs.
3. Использовать `SystemUserConnection`.

Так notification subsystem получает единое место сборки зависимостей:
repository, sender, handlers, counters.

## Multiple handlers behind one sender

Источник: `Autogenerated/Src/NotificationSender.NUI.cs`.

`NotificationSender` получает все `INotificationHandler` через
`ClassFactory.GetAll` и вызывает каждый handler.

Плюсы:

- websocket и push не связаны друг с другом;
- новый канал можно добавить через binding;
- ошибка одного handler логируется и не ломает остальные.

## Due-window repository

Источник: `Autogenerated/Src/RemindingRepository.NUI.cs`.

Repository выбирает reminders в окне:

```text
now - calculated interval <= RemindTime <= now
```

Окно учитывает и `IntervalInMinutesForFindReminding`, и
`NotificatonJobInterval`, чтобы не терять reminders между запусками job.

## Contact to SysAdminUnit mapping

Источник: `Autogenerated/Src/RemindingRepository.NUI.cs`.

`Reminding` адресован `Contact`, а realtime channel найден по `SysAdminUnit`.
Repository делает join:

```text
Reminding.ContactId = SysAdminUnit.ContactId
```

Это важный bridge между CRM contact и application user.

## Mark-as-sent flag

Источник: `Autogenerated/Src/RemindingRepository.NUI.cs`.

После отправки обновляется:

```text
IsNeedToSend = false
```

Это не то же самое, что `IsRead`. `IsNeedToSend` защищает transport pipeline от
повторной отправки, `IsRead` описывает UI-состояние пользователя.

## WebSocket to sandbox bridge

Источник: `Autogenerated/Src/ClientMessageBridge.UIv2.js`.

Server sender name мапится в одноименное sandbox message:

- `NotificationInfo`;
- `UpdateReminding`;
- `GetRemindingCounters`;
- `ReloadApplication`.

UI-модули работают с sandbox, а не с низкоуровневым websocket.

## Push idempotency with history

Источник: `Autogenerated/Src/PushNotificationSender.NUI.cs`.

Перед отправкой push handler проверяет `PushNotificationHistory` по:

- `MessageId`;
- `SysAdminUnitId`;
- `RemindTime`.

Если запись уже есть, push не отправляется повторно.

## Cached notification icons

Источник: `Autogenerated/Src/NotificationSettingsRepository.NUI.cs`.

Icon для notification ищется в `NotificationsSettings` по schema UId и
notification type. Результат кешируется в `WorkspaceCache`.

## Feature-gated pipeline

Источники:

- `Autogenerated/Src/RemindingJob.NUI.cs`;
- `Autogenerated/Src/PushNotificationSender.NUI.cs`;
- `Autogenerated/Src/RemindingService.NUI.cs`.

Feature flags:

| Feature | Роль |
| ------- | ---- |
| `NotificationV2` | включает новый job pipeline |
| `UseMobilePushNotifications` | включает push service |
| `SendPushByNotifications` | включает push handler в notification pipeline |
| `UseDateForNotificationsQuery` | добавляет `LastRemindTime` |
| `NotificationsOnOneClassJob` | legacy/provider behavior switch |

## Client specialization by notification group

Источник: `Autogenerated/Src/ReminderNotificationsSchema.NUI.js`.

Базовая UI-схема расширяется group-specific behavior:

- group/type;
- image;
- columns;
- actions menu;
- subject formatting.

Так можно добавлять новые группы notifications без копирования всего списка.
