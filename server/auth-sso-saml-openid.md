# Auth SSO SAML OpenID

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: auth, SSO, SAML, OpenID, OIDC -->

> В Base/Autogenerated SAML и OpenID представлены как схемы, настройки и
> контракты расширения. Полный SSO pipeline (redirect, assertion validation,
> cookies/session issue) в этом срезе не виден.

## SAML field mapping

Файл: `Autogenerated/Src/SAMLFieldNameConverterSchema.Base.cs`.

`SAMLFieldNameConverter` сопоставляет атрибуты SAML с колонками `Contact`.

Ключевые колонки:

| Колонка | Назначение |
| ------- | ---------- |
| `SAMLFieldName` | имя атрибута в SAML response |
| `ContactFieldName` | имя колонки контакта |
| `ColumnDefaultValue` | значение по умолчанию |

UI:

- `SAMLFieldNameConverterPage.UIv2.js`;
- `SAMLFieldNameConverterSection.UIv2.js`.

Этот слой отвечает за mapping, а не за проверку подписи SAML assertion.

## SamlResponse

Файл: `Autogenerated/Src/SamlResponseSchema.Base.cs`.

`SamlResponse` хранит:

| Колонка | Назначение |
| ------- | ---------- |
| `Response` | SAML response payload |
| `Contact` | связанный контакт |

`Contact` cascade, поэтому response удаляется вместе с контактом.

## OpenID extension contracts

Файлы:

- `IOpenIdUserChangeValidator.Base.cs`;
- `IOpenIdRoleChangeValidator.Base.cs`.

Это контракты валидаторов для сценариев изменения пользователя/ролей при
OpenID-интеграции. Они показывают extension points, но не описывают полный
OIDC login flow.

## Keycloak for reports

Файл:

- `InsightReports_KeycloakService.InsightReport.cs`.

Это отдельный контур получения токена для Insight Reports через настройки
`OpenIDIdentityServer*`. Его не стоит смешивать с пользовательским SSO входом
в приложение.

## Что не документировать как готовый pipeline

В `Autogenerated/Src` не найден полноценный SAML/OIDC pipeline:

- metadata endpoint;
- redirect to IdP;
- ACS endpoint;
- validation assertion/signature;
- issue auth cookie;
- низкоуровневый session/cookie management.

При диагностике этих частей нужно смотреть host/runtime конфигурацию платформы,
а не только Base package.

## Практическое правило

Если задача про:

- mapping SAML атрибутов в Contact — смотрите `SAMLFieldNameConverter`;
- сохраненный SAML payload — смотрите `SamlResponse`;
- OAuth application/client/tokens — смотрите
  [auth-oauth20-apps-tokens.md](auth-oauth20-apps-tokens.md);
- user/session/password/admin endpoints — смотрите
  [auth-sessions-admin.md](auth-sessions-admin.md);
- полный IdP login flow — выходите за границы `Autogenerated/Src`.
