# Notifications Reminders Server Pipeline

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: NotificationEventListener, RemindingJob, NotificationSender, Quartz -->

> Server pipeline выбирает due reminders, отправляет их через набор handlers и
> снимает флаг `IsNeedToSend`.

## App start

Файл: `Autogenerated/Src/NotificationEventListener.NUI.cs`.

На `OnAppStart` listener делает три операции:

1. Регистрирует DI bindings.
2. Планирует `NotificationCleanerJob`.
3. Планирует `RemindingJob`.

DI bindings:

| Contract | Implementation |
| -------- | -------------- |
| `INotificationSender` | `NotificationSender` |
| `INotificationCounterFactory` | `NotificationCounterFactory` |
| `INotificationCountHandler` | `NotificationCountHandler` |
| `IRemindingRepository` | `RemindingRepository` |
| `INotificationHandler` | `PushNotificationSender` |
| `INotificationHandler` | `WebSocketNotificationSender` |
| `INotificationCounter` | `RemindingCounter`, `ESNNotificationCounter`, `VisaNotificationCounter` |

## Jobs

`NotificationEventListener` планирует:

| Job | Группа | Расписание |
| --- | ------ | ---------- |
| `RemindingJob` | `RemindingGroup` | minutely, `NotificatonJobInterval` |
| `NotificationCleanerJob` | `NotificationCleanerGroup` | cron `0 0 2 * * ?` |

`NotificatonJobInterval` читается из SysSettings, default `1` minute.

## RemindingJob

Файл: `Autogenerated/Src/RemindingJob.NUI.cs`.

Job выполняется только если включен feature:

```text
NotificationV2
```

Поток:

1. Получить `IRemindingRepository`.
2. Вызвать `GetRemindings`.
3. Передать результат в `INotificationSender.Send`.
4. Взять `MessageId` всех отправленных reminders.
5. Вызвать `SetRemindingsIsSent`.

Если due reminders нет, job ничего не отправляет.

## RemindingRepository

Файл: `Autogenerated/Src/RemindingRepository.NUI.cs`.

Repository выбирает `Reminding`, где:

```text
RemindTime between (now - interval) and now
NotificationTypeId = RemindingConsts.NotificationTypeRemindingId
IsNeedToSend = true
```

Интервал поиска:

- `IntervalInMinutesForFindReminding`, default `5`;
- `NotificatonJobInterval`, default `1`;
- итоговый интервал не меньше job interval + base interval.

Repository join-ит `SysAdminUnit` по `Reminding.ContactId = SysAdminUnit.ContactId`.
Это превращает reminder contact в realtime recipient.

`GetNotificationInfo` формирует:

- `MessageId`;
- `Title`;
- `Body`;
- `EntityId`;
- `RemindTime`;
- `SysAdminUnit`;
- `EntitySchemaName`;
- `LoaderName`;
- `ImageId`;
- `GroupName = Reminding`.

После отправки repository выставляет:

```text
IsNeedToSend = false
```

## NotificationSender

Файл: `Autogenerated/Src/NotificationSender.NUI.cs`.

`NotificationSender` — фасад над всеми `INotificationHandler`.

Паттерн:

1. Получить handlers через `ClassFactory.GetAll<INotificationHandler>`.
2. Для каждого handler вызвать `Handle`.
3. Ошибка одного handler логируется в `Notifications`, но не останавливает другие.

Так websocket и push могут работать независимо: падение push не должно ломать
in-app websocket доставку.

## Legacy counters service

Файл: `Autogenerated/Src/RemindingService.NUI.cs`.

`RemindingService.UpdateRemindingsCountersStart` — legacy путь для process job
`GetRemindingCounters`. Метод помечен obsolete и отключается при feature:

- `TestNotificationOnWork`;
- `NotificationV2`.

Для новых сценариев ориентируйтесь на `NotificationV2` pipeline.
