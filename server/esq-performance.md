# ESQ Performance

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: ESQ, performance, RowCount, AddAllSchemaColumns, UseAdminRights, cache -->

> Практические рекомендации по производительности `EntitySchemaQuery`: объём выборки, колонки, права, display values, кэш и граница с SQL builders.

## Главный принцип

ESQ удобен, но он не бесплатный. Каждый лишний path, display value и lookup может превратиться в дополнительные join'ы и нагрузку на БД.

Минимальный production-подход:

```csharp
var esq = new EntitySchemaQuery(userConnection.EntitySchemaManager, "Contact") {
    RowCount = 50,
    IgnoreDisplayValues = true
};
var nameColumn = esq.AddColumn("Name");
esq.Filters.Add(esq.CreateFilterWithParameters(FilterComparisonType.StartWith, "Name", searchText));
```

## Добавляйте только нужные колонки

Плохо:

```csharp
esq.AddAllSchemaColumns();
```

Лучше:

```csharp
esq.AddColumn("Name");
esq.AddColumn("Email");
```

`AddAllSchemaColumns()` встречается в legacy/generic-коде, например в `BSDeliverySource.BPMSoftSender.cs` и `DiagramSchema.ProcessLibrary.cs`, но для новых сервисных сценариев это стоит считать исключением.

## Ограничивайте число строк

Используйте `RowCount`, если не нужен весь набор данных:

```csharp
var esq = new EntitySchemaQuery(entitySchema) {
    RowCount = 1
};
```

Примеры:

- `BaseEntityWithPositionSchema.Base.cs` берёт одну строку для позиции;
- `AccountSearcher.EmailMining.cs` ограничивает поиск аккаунтов;
- `ESNFeedModuleService.ESN.cs` задаёт page size для mention contacts.

## Сортировка должна быть явной

Если берётся первая запись, нужен порядок:

```csharp
esq.RowCount = 1;
esq.AddColumn("Position").OrderByAsc();
```

Без `OrderBy...` первая строка не имеет стабильного бизнес-смысла.

## `PrimaryQueryColumn`

Не тяните primary key без необходимости, но и не теряйте его там, где он нужен.

```csharp
esq.PrimaryQueryColumn.IsVisible = false;
esq.PrimaryQueryColumn.IsAlwaysSelect = true;
```

Практика:

- для проекций и distinct-поиска primary column можно скрывать;
- для последующего `Save`, `PrimaryColumnValue` или mapping лучше гарантировать выборку primary key.

## `IgnoreDisplayValues`

Если display values не нужны, отключайте:

```csharp
var esq = new EntitySchemaQuery(schema) {
    IgnoreDisplayValues = true
};
```

Это особенно полезно для:

- фоновой обработки;
- кэшей;
- routing/request pipeline;
- индексации.

## `UseAdminRights`

`UseAdminRights` влияет на применение прав.

```csharp
var esq = new EntitySchemaQuery(entitySchema) {
    RowCount = GetDefEsqRowCount(),
    UseAdminRights = false
};
```

Не выставляйте это свойство механически.

- `false` может быть нужен для системной индексации/внутренней обработки.
- `true` или дефолтное поведение может быть важно для пользовательских сервисов.

Reference: `IndexingEntityListBuilder.GlobalSearch.cs`, `ESNFeedModuleService.ESN.cs`.

## Кэширование

ESQ поддерживает кэш:

```csharp
esq.Cache = userConnection.SessionCache.WithLocalCaching("VwCommandActionCache");
esq.CacheItemName = "CommandActionCacheItem_Command";
```

Reference: `CommandLineService.NUI.cs`.

Кэш уместен, если:

- запрос часто повторяется;
- данные редко меняются;
- ключ кэша уникален для набора параметров.

## Агрегаты вместо полной выборки

Если нужен count/max, не загружайте все строки:

```csharp
var maxColumn = esq.AddColumn(esq.CreateAggregationFunction(AggregationTypeStrict.Max, "Position"));
var countColumn = esq.AddColumn(esq.CreateAggregationFunction(AggregationTypeStrict.Count, "Id"));
```

Reference: `BaseEntityWithPositionSchema.Base.cs`.

## ESQ или SQL builder

Используйте `Update`, `Delete`, `Select`, если:

- нужно массовое обновление;
- результат не требует `Entity`;
- нужна сложная группировка;
- запрос проще выразить SQL builder'ом.

Пример из Sender:

- entity читается через ESQ;
- массовое обновление recipient status выполняется через `Update`.

## Anti-patterns

### 1. `AddAllSchemaColumns()` в сервисном методе

Особенно плохо для крупных схем вроде `Contact`, `Account`, delivery/chat сущностей.

### 2. Нет `RowCount` в поисковом endpoint

Поиск без ограничения может вернуть слишком много данных.

### 3. Чтение display values в фоновой задаче

Для фоновой обработки чаще нужны raw значения и ids.

### 4. Сложный bracket path без комментария

Такой path тяжело сопровождать и легко сломать при изменении схем.
