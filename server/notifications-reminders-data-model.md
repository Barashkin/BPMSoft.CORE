# Notifications Reminders Data Model

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: Reminding, NotificationsSettings, NotificationProvider, counters -->

> Модель notification center строится вокруг `Reminding`: запись хранит адресата,
> время, subject entity, текст, тип уведомления, loader и флаг отправки.

## Reminding

Файлы:

- `Autogenerated/Src/RemindingSchema.Base.cs`;
- `Autogenerated/Src/RemindingSchema.NUI.cs`.

Базовая схема `Reminding_Base_BPMSoft` содержит основные поля:

| Колонка | Назначение |
| ------- | ---------- |
| `Author` | автор reminder |
| `Contact` | получатель на уровне contact |
| `Source` | источник reminder |
| `RemindTime` | UTC-время срабатывания |
| `Description` | описание |
| `SubjectId` | идентификатор связанной записи |
| `SysEntitySchema` | схема связанной записи |
| `SubjectCaption` | отображаемый текст subject |
| `TypeCaption` | подпись типа |
| `PopupTitle` | заголовок popup |
| `Hash` | hash для поиска/дедупликации |
| `IsRead` | прочитано |
| `NotificationType` | тип уведомления |
| `Loader` | client loader schema |
| `SenderId` | отправитель в notification pipeline |
| `IsNeedToSend` | нужно отправить через job |
| `AnniversaryDate` | дата годовщины |
| `Config` | дополнительная конфигурация |

NUI-слой расширяет базовую схему до `Reminding` и задает default
`NotificationType` для reminder.

## NotificationsSettings

Файл: `Autogenerated/Src/NotificationsSettingsSchema.Base.cs`.

`NotificationsSettings` задает icon для связки:

- `SysEntitySchemaUId`;
- `NotificationType`;
- `SysImage`.

`NotificationSettingsRepository` читает эти настройки и кеширует image id в
`WorkspaceCache` по ключу schema UId + notification type.

## NotificationProvider

Файл: `Autogenerated/Src/NotificationProviderSchema.NUI.cs`.

`NotificationProvider` хранит:

| Колонка | Назначение |
| ------- | ---------- |
| `ClassName` | class provider |
| `Type` | тип provider |

Legacy counters используют provider classes для получения counts и popup config.
При включенном `NotificationsOnOneClassJob` часть provider-логики обходится
через implemented provider helper.

## Counters

В notification center считаются несколько групп:

| Группа | Источник |
| ------ | -------- |
| `Reminding` | `RemindingCounter` |
| `Visa` | `VisaNotificationCounter` |
| `ESNNotification` | `ESNNotificationCounter` |
| `Notification` | system notifications |
| `Anniversary` | anniversary notifications |

Клиентский request `NotificationCountersServiceRequest` вызывает
`NotificationInfoService.GetCounters` и возвращает normalized object:

```text
remindingsCount
visaCount
esnNotificationsCount
notificationsCount
anniversariesCount
resetCounters
```

## Views

Представления и счетчики:

- `VwRemindingsSchema.Mobile.cs`;
- `VwRemindingsCountSchema.NUI.cs`;
- `VwUserRemindingsCountSchema.NUI.cs`.

Они нужны для mobile/панелей и не заменяют основную таблицу `Reminding`.

## Практические правила

- Для факта отправки проверяйте `IsNeedToSend`, а не только `IsRead`.
- Для UI-icon проверяйте `NotificationsSettings`, `SysEntitySchemaUId` и
  `NotificationType`.
- Для открытия карточки проверяйте `SubjectId`, `SysEntitySchema` и `Loader`.
- Для realtime доставки нужен `Contact -> SysAdminUnit` mapping.
