> Входная точка в Security/Rights Dive: операции, права на схемы и записи, лицензии, портальный доступ и клиентские проверки.

## Общая модель

В BPMSoft безопасность складывается из нескольких уровней:

| Уровень | Пример | Где проверяется |
| ------- | ------ | --------------- |
| Operation rights | `CanManageUsers`, `CanAdministratePortalUsers` | `DBSecurityEngine` |
| Schema rights | read/append/edit/delete для схемы | `RightsHelper`, `RightUtilities` |
| Record rights | read/edit/delete конкретной записи | `DBSecurityEngine`, `RightsService` |
| Column rights | доступность колонок | security engine, SSP helpers |
| License rights | лицензия на операцию или схему | `LicHelper`, `LicManager` |
| Portal/SSP access | доступ портального пользователя | SSP helpers, portal schema access |

## Базовый принцип

Клиентские проверки управляют UX, но не являются защитой. Все изменения, которые влияют на данные или конфигурацию, должны проверяться на сервере.
