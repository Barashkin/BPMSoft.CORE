# Entity Schema Inheritance

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: EntitySchema, inheritance, ExtendParent, BaseEntitySchema, BaseLookupSchema -->

> Наследование схем, package layering и `ExtendParent` в autogenerated entity schema коде.

## Базовое наследование

Обычная прикладная схема наследует один из базовых типов.

```csharp
public class MetricSchema : BaseEntitySchema
{
}

public class InsightSlugSchema : BaseLookupSchema
{
}
```

`BaseEntitySchema` даёт системные колонки, `BaseLookupSchema` добавляет `Name`, `Description` и `PrimaryDisplayColumn`.

## ExtendParent

`ExtendParent = true` означает, что текущий package layer расширяет родительскую схему.

```csharp
protected override void InitializeProperties() {
    base.InitializeProperties();
    RealUId = new Guid("...");
    Name = "KnowledgeBase_BPMSoftOCC_BPMSoft";
    ParentSchemaUId = new Guid("...");
    ExtendParent = true;
}
```

Такой файл не следует воспринимать как полностью новую независимую сущность. Это слой расширения существующей схемы.

## RealUId и GetParentRealUIds

Для расширений важна цепочка real UId.

```csharp
public override void GetParentRealUIds(Collection<Guid> realUIds) {
    base.GetParentRealUIds(realUIds);
    realUIds.Add(new Guid("..."));
}
```

Эта цепочка помогает платформе собрать metadata из нескольких пакетов.

## Generated entity class

Schema class обычно сопровождается generated entity class.

```csharp
public class Metric : BaseEntity
{
    public Guid TypeId {
        get { return GetTypedColumnValue<Guid>("TypeId"); }
        set { SetColumnValue("TypeId", value); }
    }
}
```

Для lookup-колонок generated class часто содержит:

- `{Lookup}Id`;
- `{Lookup}Name`;
- private cache field;
- navigation property.

## EventsProcess в наследовании

Каждый schema layer может вернуть свой `EventsProcess`.

```csharp
public override EmbeddedProcess CreateEventsProcess(UserConnection userConnection) {
    return new KnowledgeBase_BPMSoftOCCEventsProcess(userConnection);
}
```

При диагностике событий смотрите не только базовый `EventsProcess`, но и package-specific extensions.

## Как отличать разные сущности от разных слоёв

| Признак | Отдельная сущность | Расширение |
| --- | --- | --- |
| `ExtendParent` | `false` | `true` |
| `ParentSchemaUId` | базовая схема или domain parent | конкретная расширяемая схема |
| `Name` | имя новой схемы | часто содержит package suffix |
| Колонки | полный набор собственных колонок | только добавленные/изменённые колонки |

## Практические правила

- Не копируйте GUID из autogenerated schema вручную.
- При поиске колонки учитывайте все package layers.
- Если поведение не найдено в base schema, ищите `*.Package.cs` расширение.
- Для замещения/расширения следуйте существующей цепочке имени и родителя.
