# DCM Elements, User Tasks And Mapping

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: DCM, DcmSchemaElement, DcmUserTask, mapping -->

> DCM element хранит DCM-обвязку и вложенный `processFlowElement`.
> User task semantics берутся из DCM managers и property pages, а mapping
> использует DCM-specific modules.

## DcmSchemaElement

Файл: `Autogenerated/Src/DcmSchemaElement.DcmDesigner.js`.

Класс:

- наследуется от `BPMSoft.BaseProcessSchemaElement`;
- использует mixin `BPMSoft.ParametrizedProcessSchemaElement`;
- имеет `typeName = "BPMSoft.Core.DcmProcess.DcmSchemaElement"`;
- хранит `processFlowElement`;
- сериализует `requiredType` и `conditions`;
- делегирует edit page вложенному process flow element.

При инициализации `processFlowElement` создаётся через:

```text
BPMSoft.DcmElementSchemaManager.createInstance(managerItemUId, metaData)
```

## DcmElementSchemaManager

Файл: `Autogenerated/Src/DcmElementSchemaManager.DcmDesigner.js`.

Менеджер:

- наследуется от `BaseProcessFlowElementSchemaManager`;
- `managerName = "DcmElementSchemaManager"`;
- `userTaskSchemaManagerName = "DcmUserTaskSchemaManager"`;
- feature для недоступных элементов: `DcmNotImplementedElements`;
- core element: `BPMSoft.DcmProcessSubprocessSchema`.

Это DCM-адаптер над общей моделью process flow elements.

## User task managers

Ключевые файлы:

- `DcmUserTaskSchemaManager.DcmDesigner.js`;
- `DcmUserTaskSchemaManagerItem.DcmDesigner.js`;
- `DcmUserTaskSchema.DcmDesigner.js`;

Они задают DCM-специфику user tasks и subprocess внутри DCM stages.

- `UseFormulaConditions`.

## Transitions on property page

При смене `Transition`:

1. Создаётся transition через `DcmSchemaElementTransitionFactory`.
2. Он записывается в `DcmElement`.
3. `ExecuteAfterElement` обновляется для non-default transition.

При смене `ExecuteAfterElement` обновляется `sourceElementUId` текущего
transition.

## Mapping modules

Файл: `Autogenerated/Src/DcmMappingModule.DcmDesigner.js`.

`DcmMappingModule` наследуется от `BaseParametersMappingModule` и задаёт:

- `mappingPageName = "DcmMappingPage"`;
- `mappingSelectionPageName = "DcmParameterSelectionPage"`;
- container id `ProcessMappingPage` + suffix.

Связанные файлы:

- `DcmMappingPage.DcmDesigner.js`;
- `DcmParameterSelectionPage.DcmDesigner.js`;
- `DcmElementParametersMappingModule.DcmDesigner.js`;
- `DcmElementParametersMappingPage.DcmDesigner.js`.

## Default values for DCM

`DcmSchemaElement.applyParametersDefValues` применяет `defValueForDcm` из
`parameter.sourceValue`.

Это отличается от обычного BPM element default value: значение может быть
специально подготовлено для DCM-контекста.

## Практические правила

- Для DCM element смотрите и `DcmSchemaElement`, и вложенный `processFlowElement`.
- Для user task списка проверяйте `DcmUserTaskSchemaManager`.
- Для transitions проверяйте `DcmSchemaElementTransitionFactory`.
- Для mapping используйте DCM modules, не BPM `ProcessParametersMappingModule`.
