# Notifications Reminders Activity Process

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: Activity reminders, anniversary, process reminders, Reminding -->

> Activity и process modules часто являются источниками reminders, но не равны
> транспорту notification center. Источник создает/обновляет `Reminding`, а
> доставка идет через общий pipeline.

## Activity reminder text

Файл: `Autogenerated/Src/ActivityRemindingTextFormer.NUI.cs`.

`ActivityRemindingTextFormer` реализует `IRemindingTextFormer`.

Body формируется из:

- `Title`;
- `AccountName`;
- `ContactName`.

Если customer title есть, body выглядит как:

```text
{AccountName, ContactName}: {Title}
```

Title берется из localizable string `ActivityRemindingTextFormer.Title`.

## Activity notification provider

Файл: `Autogenerated/Src/ActivityNotificationProvider.NUI.cs`.

Это legacy provider для group `Reminding`. Его стоит рассматривать как
исторический контур provider-based counters, а не как новый способ доставки.

Для новых сценариев ориентируйтесь на `Reminding`, `NotificationV2` и общий
pipeline из [notifications-reminders-server-pipeline.md](notifications-reminders-server-pipeline.md).

## Activity fields vs Reminding

Не смешивайте:

- поля activity, которые определяют напоминание;
- записи `Reminding`, которые попадают в notification center;
- доставку через websocket/push.

Диагностика должна идти по цепочке:

```text
Activity/process source
  -> Reminding row
  -> RemindingJob selection
  -> NotificationSender handlers
  -> UI/push delivery
```

Если на Activity выставлено время напоминания, но нет строки `Reminding`, ищите
проблему в source logic. Если строка есть, но popup не пришел, проверяйте
server pipeline и delivery channels.
