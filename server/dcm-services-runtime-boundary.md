# DCM Services And Runtime Boundary

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: DCM, DcmService, RunDcmProcess, ProcessEngine -->

> DCM service слой в Base небольшой: он запускает DCM-процесс и возвращает
> разрешённые стадии. Подробный runtime `Process` остаётся в Process Dive.

## DcmService

Файл: `Autogenerated/Src/DcmService.DcmDesigner.cs`.

WCF service:

```text
ServiceModel/DcmService.svc
```

Методы:

| Method | Назначение |
| ------ | ---------- |
| `RunDcmProcess` | запустить DCM для записи |
| `GetAllowedStagesForCurrentUser` | вернуть стадии, доступные текущему пользователю |

## RunDcmProcess

Контракт:

```text
POST RunDcmProcess
dcmSchemaUId: Guid
entityRecordId: Guid
```

Поток:

1. `UserConnection.DcmSchemaManager`.
2. `GetInstanceByUId(dcmSchemaUId)`.
3. `UserConnection.ProcessEngine`.
4. `processEngine.RunDcmProcess(entityRecordId, dcmSchema)`.

Это граница между DCM metadata и runtime engine.

## GetAllowedStagesForCurrentUser

Контракт:

```text
POST GetAllowedStagesForCurrentUser
dcmSchemaUId: Guid
```

Поток:

1. `ClassFactory.Get<DcmPermissions>()`.
2. `GetAllowedStagesForCurrentUser(dcmSchemaUId)`.
3. Возврат `List<Guid>` allowed stage identifiers.

## Runtime boundary

DCM dive описывает:

- DCM schema metadata;
- stages;
- transitions;
- settings;
- запуск через `DcmService`.

Не описывает:

- общий `Process` lifecycle;
- `ProcessSchema` generated wrappers;
- `ProcessUserTask.InternalExecute`;
- `SysProcessLog`;
- Quartz запуск.

См. [process-overview.md](process-overview.md) и
[process-runtime-schema.md](process-runtime-schema.md).

## Права стадий

Стадии могут использовать permissions:

- в `DcmSchemaStage` есть `usePermissions`;
- `permissions` сериализуется при feature `DcmStagesPermissions`;
- server-side проверка доступных стадий делается через `DcmPermissions`.

Если пользователь не видит переход на стадию, проверяйте и metadata стадии,
и результат `GetAllowedStagesForCurrentUser`.

## Практические правила

- Для запуска DCM используйте `DcmService.RunDcmProcess`, а не прямой запуск
  обычного процесса.
- Для stage rights диагностируйте `DcmPermissions`, не только client state.
- Runtime ошибку после запуска ищите в Process docs, но причину выбора стадии
  и схемы ищите в DCM settings.
