# Diagnostics Deployment Logging Overview

<!-- Версия: 1.0 | Обновлено: 2026-04-28 | Платформа: BPMSoft 1.9 -->
<!-- Теги: diagnostics, deployment, logging, Common.Logging, Quartz, Services -->

> Capstone dive по runtime-диагностике: где искать технические логи, чем они
> отличаются от process/audit logs, как связать ошибки старта, services, Quartz
> и deployment.

## Источники диагностики

| Источник | Что содержит | Где описано |
| -------- | ------------ | ----------- |
| Application log (`Common.log`) | runtime exceptions, startup, integrations, jobs | `platform-overview.md`, этот документ |
| `SysProcessLog` / `SysProcessElementLog` | выполнение бизнес-процессов | `process-listeners-and-logs.md` |
| Quartz logs | scheduler/job execution, triggers, misfires | `quartz-troubleshooting.md` |
| `SysOperationAudit` | аудит операций пользователя | `nui-audit-operation-log.md` |
| Domain tables | статусы sync/import/mining/etc. | соответствующий domain troubleshooting |

Не смешивайте технический application log и бизнес-аудит: они отвечают на разные
вопросы.

## Common.Logging pattern

Типовой паттерн в `Autogenerated/Src`:

```text
private static readonly ILog _log = LogManager.GetLogger("LoggerName");
```

Группы логгеров встречаются в:

- app lifecycle and warmup;
- Global Search;
- Process maintenance;
- File Import;
- Email Mining;
- Exchange/mail sync;
- OCC/Sender;
- ML jobs;
- reports and integrations.

Логируйте причину, идентификатор записи/job/process и внешний endpoint, но не
секреты, access tokens, passwords и raw personal data.

## Deployment loop

Базовый цикл:

```text
Изменение пакета
  -> build changed configuration / compilation
  -> restart application or pool if needed
  -> cache/Redis invalidation when required
  -> smoke check services/pages/jobs
  -> domain troubleshooting if symptom persists
```

Base docs не содержат универсального health-check endpoint. Если нужен health
probe, ищите deployment-specific конфигурацию вне `Autogenerated/Src` или
добавляйте отдельный runtime endpoint в проектном слое.

## App lifecycle and jobs

`IAppEventListener`/`AppEventListenerBase` подходят для регистрации jobs,
warmup и подписок при старте приложения. Долгую работу нельзя выполнять прямо в
`OnAppStart`; её нужно переносить в Quartz/background job.

Связанные зоны:

- `event-listeners-app-lifecycle.md`;
- `event-listeners-async-and-jobs.md`;
- `scheduler-quartz.md`;
- `quartz-troubleshooting.md`.

## Services diagnostics

Для WCF/REST проверяйте:

- route and method name;
- request body shape and `BodyStyle`;
- current `UserConnection`;
- operation/license checks;
- server exception in application log;
- client response handling.

Для исходящих REST calls логируйте request id/correlation id, endpoint, status,
duration and sanitized error body.

## Boundaries

| Область | Граница |
| ------- | ------- |
| Deployment | build/restart/cache/config; не доменная бизнес-ошибка |
| Quartz | job scheduling/execution; не HTTP request lifecycle |
| Services | request/response and auth; не background retry unless job is used |
| Process logs | process execution; не Common.Logging |
| Audit | user operation trail; не technical stack trace |
| Admin packages | package metadata/build; не runtime observability |

## Troubleshooting router

| Симптом | Первый документ |
| ------- | --------------- |
| REST 404/500 | `services-troubleshooting.md` |
| Job не стартует | `quartz-troubleshooting.md` |
| Process упал | `process-listeners-and-logs.md`, `process-troubleshooting.md` |
| После сборки не видны изменения | `admin-packages-build-overview.md` |
| Не применились права | `security-troubleshooting.md`, `security-licensing-operations-matrix.md` |
| Integration timeout | `services-outgoing-rest.md` или domain troubleshooting |
| Mailbox/OCC/ML/FileImport ошибка | соответствующий `*-troubleshooting.md` |
