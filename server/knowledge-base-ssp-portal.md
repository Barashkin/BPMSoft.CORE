# Knowledge Base SSP Portal

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: KnowledgeBase, SSP, Portal, search -->

> Portal Knowledge Base — read-oriented контур: секция запрещает создание и
> удаление, карточка отключает редактирование полей, файлы показываются через
> read-only detail, а поиск работает через отдельный модуль.

## Portal section

Файл: `Autogenerated/Src/PortalKnowledgeBaseSection.SspKnowledgeBase.js`.

Секция:

- использует `entitySchemaName = "KnowledgeBase"`;
- удаляет delete/copy actions;
- скрывает `CombinedModeActionButtonsCardRightContainer`;
- отключает context help;
- запрещает add record button;
- оставляет только grid data view;
- фильтрует action menu, убирая экспорт и удаление.

Это делает KB на портале потребительской секцией, а не редактором статей.

## Portal page

Файл: `Autogenerated/Src/PortalKnowledgeBasePage.SspKnowledgeBase.js`.

Карточка:

- использует `entitySchemaName = "KnowledgeBase"`;
- удаляет `RelationsTab`;
- делает `Name`, `Type`, `ModifiedBy`, `ModifiedOn`, `Notes` read-only;
- убирает tag container;
- заменяет обычный `Files` detail на `FileDetailReadOnly`;
- скрывает actions button.

Read-only files:

```text
FileDetailReadOnly
  -> entitySchemaName: KnowledgeBaseFile
  -> masterColumn: Id
  -> detailColumn: KnowledgeBase
```

## Portal search module

Файл: `Autogenerated/Src/KnowledgeBaseSearchModule.SspKnowledgeBase.js`.

Модуль строит `BPMSoft.CommandLine`:

- `minSearchCharsCount = 3`;
- `searchDelay = 350`;
- marker `knowledgeBase-search-line`;
- placeholder из ресурсов;
- search value сохраняется в `BPMSoft.configuration.Storage.Filters`;
- фильтр кладется как `CustomFilters`;
- навигация выполняется через sandbox `PushHistoryState`.

Поиск строит wildcard-значение:

```text
%{input.trim()}%
```

И переводит пользователя в section module для `KnowledgeBase`.

## Popular articles module

Файл: `Autogenerated/Src/PopularKnowledgeBaseArticlesListModule.SspKnowledgeBase.js`.

Модуль наследуется от `DashboardGridModule` и показывает популярные статьи на
портале:

- `entitySchemaName = "KnowledgeBase"`;
- `rowCount = 10` в config;
- view model ставит `rowCount = 6` при `initGridData`;
- сортировка идет по aggregation `[Like:KnowledgeBase].Id`;
- есть кнопка перехода в секцию KB через `PushHistoryState`;
- используется стиль `widget-blue`.

Этот модуль переиспользует dashboard grid infrastructure для портальной
главной страницы.

## Portal search row

Файл: `Autogenerated/Src/PortalKnowledgeBaseSearchRowSchema.SspKnowledgeBase.js`.

Строка результата поиска на портале является отдельной схемой отображения,
чтобы не использовать полную desktop Global Search строку.

## Доступ и безопасность

Для видимости KB на портале важны общие SSP механизмы:

- `SysModuleEntityInPortal`;
- `SysSSPEntitySchemaAccessList`;
- `PortalColumnAccessList`;
- `PortalSectionHelper`;
- `GlobalSearchSSPHelper`.

См. также [ssp-portal-access-rights.md](ssp-portal-access-rights.md) и
[global-search-settings-ssp.md](global-search-settings-ssp.md).
