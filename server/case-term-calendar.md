# Case Term Calendar

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: CaseTerm, Calendar, deadline, SLA, troubleshooting -->

> `CaseTerm*` в Base — календарный механизм расчёта response/resolve сроков.
> Это не полный SLA-модуль, но важная инфраструктура для сроков обращений.

## CaseTermInterval

Файл: `Autogenerated/Src/CaseTermInterval.Calendar.cs`.

`CaseTermInterval` содержит:

- `ResponseTerm`;
- `ResolveTerm`.

Маска `CaseTermStates` показывает, какие сроки заполнены:

| Flag | Значение |
| ---- | -------- |
| `ContainsResponse` | есть response term |
| `ContainsResolve` | есть resolve term |

Срок считается заполненным, если `TimeTerm.Type` не default и `Value > 0`.

## Strategy queue

Файл: `Autogenerated/Src/CaseTermStrategyQueue.Calendar.cs`.

`CaseTermStrategyQueue` строит очередь стратегий из схемы `DeadlineCalcSchemas`:

- читает `Name`;
- читает `Handler`;
- читает `AlternativeRule.Name`;
- находит default strategy;
- последовательно идет по alternative strategy chain;
- добавляет стратегии в `CalculationStrategyRules` log store.

Метод `Dequeue` возвращает class name следующей стратегии.

## Strategy manager

Файл: `Autogenerated/Src/CaseTermStrategyManager.Calendar.cs`.

`CaseTermStrategyManager.GetItem` получает class name, вызывает
`Type.GetType`, затем создает strategy через `ClassFactory.ForceGet` с
аргументами:

- `userConnection`;
- `args`.

Стратегия должна реализовывать `BaseTermStrategy<CaseTermInterval,
CaseTermStates>`.

## Interval selector

Файл: `Autogenerated/Src/CaseTermIntervalSelector.Calendar.cs`.

`CaseTermIntervalSelector`:

1. Создает пустой `CaseTermInterval`.
2. Получает `CaseTermStrategyQueue`.
3. Получает `CaseTermStrategyManager`.
4. Перебирает стратегии, пока не найдены и response, и resolve сроки.
5. Каждую strategy вызывает с текущей mask.
6. Если strategy выбросила исключение, selector пропускает ее и идет дальше.
7. Заполняет selected strategy и `TimeTerm` в `TermCalculationLogStore`.

## TermCalculationLogStore

Файл: `Autogenerated/Src/TermCalculationLogStore.Calendar.cs`.

Store сериализует данные диагностики:

- `CalculationTermKind`;
- `CalculationParameters`;
- `CalculationStrategyRules`;
- `ActiveTimeIntervals`;
- `CalculationIntervals`;
- `CalculatedTerm`;
- `CurrentDateTime`;
- `TermTimeRemains`;
- `UsedTimeTermIntervals`;
- `TimeTerm`;
- `RegistrationDate`;
- `SelectedStrategy`;
- `CalendarData`.

Ключи cache:

- `CalculateTermsStore`;
- `IsCalculateTermsLogEnabled`.

## TermCalculationLogger

Файл: `Autogenerated/Src/TermCalculationLogger.Calendar.cs`.

Logger заполняет:

- использованные активные интервалы;
- расчетные интервалы;
- рабочие/нерабочие дни;
- остаток срока;
- локализованный день недели;
- timezone conversion между user timezone и calendar timezone.

Он также определяет, какой срок сейчас рассчитывается: response или resolve.

## Где искать настройки стратегий

Репрезентативные схемы и UI:

- `DeadlineCalcSchemasSchema.SSP.cs`;
- `CaseDeadlineCalcSchemasLookupSection.Calendar.js`;
- `CaseDeadlineCalculationSchemasEditPage.Calendar.js`.

`DeadlineCalcSchemas` задает правила и handler classes, из которых строится
очередь стратегий.

## Практический смысл

Для задач "почему срок посчитался так" сначала проверяйте:

- выбранную strategy chain;
- `SelectedStrategy`;
- `CalendarData`;
- `ActiveTimeIntervals`;
- timezone conversion;
- `UsedTimeTermIntervals`;
- `CalculationIntervals`;
- `TermTimeRemains`.
