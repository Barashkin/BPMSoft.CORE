# NUI Configuration Workplace

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: NUI, ConfigurationDataService, WorkplaceService, SysModule -->

> NUI server endpoints для загрузки configuration data, секций, карточек и
> рабочих мест. Документ дополняет Administration dive и не дублирует модель
> `SysWorkplace` / `SysModule`.

## ConfigurationDataService

Опорный файл: `Autogenerated/Src/ConfigurationDataService.NUI.cs`.

Сервис отдаёт структурированные данные о конфигурации:

| Метод | BodyStyle | Назначение |
| ----- | --------- | ---------- |
| `GetData()` | `WrappedRequest` | полная configuration data для UI |
| `GetEntityData(entityNames)` | `Bare` | configuration data по набору сущностей |

Ответ:

```text
ConfigurationDataResponse : ConfigurationServiceResponse
  -> data: ConfigurationDataDto
```

`ConfigurationDataDto` содержит module/page структуры, entity schema names,
section schema names, card modules, mini pages, `VisaSchemaUId` и другие поля,
которые нужны клиентскому shell/section layer.

## ConfigurationSectionHelper

`ConfigurationDataService` не строит всё вручную. Он делегирует в
`ConfigurationSectionHelper`, который:

- читает `SysModule`;
- связывает секции с рабочими местами;
- добавляет `SysModuleEdit`, `SysModuleEntity`, `SysModuleVisa`;
- формирует startup scripts;
- использует cache для workplace и login local cache.

Подробности по модели секций см. в
[admin-workplaces-sections.md](admin-workplaces-sections.md).

## WorkplaceService

Опорный файл: `Autogenerated/Src/WorkplaceService.NUI.cs`.

Сервис управляет кэшем и изменениями рабочих мест:

| Метод | Назначение |
| ----- | ---------- |
| `ResetScriptCache()` | очистить script/config cache |
| `ResetWorkplaceCache()` | сбросить выбранное рабочее место |
| `SetWorkplaceCache(workplaceId)` | зафиксировать workplace в cache |
| `RefreshWorkplace(applicationClientTypeId)` | вернуть workplaces script |
| `GetWorkplacesByType(type)` | список рабочих мест по типу |
| `CreateWorkplace(request)` | создать рабочее место |
| `ChangeWorkplaceName(...)` | переименовать |
| `ChangeWorkplacePosition(...)` | изменить позицию |
| `DeleteWorkplace(...)` | удалить |
| `AddSectionToWorkplace(...)` | добавить секцию |
| `RemoveSectionFromWorkplace(...)` | убрать секцию |

Для изменений сервис использует `IWorkplaceManager`, полученный через
`ClassFactory`.

## Граница с Admin dive

Admin dive описывает:

- сущности `SysWorkplace`, `SysModule`, `SysModuleInWorkplace`;
- UI настройки рабочих мест;
- package/configuration concepts.

NUI configuration/workplace описывает:

- server endpoints, которые эти данные отдают или меняют;
- DTO для client shell;
- cache reset/refresh сценарии.

## Смежные сервисы

В навигационном контуре также важны:

- `SectionService.Workplace.cs`;
- `NavigationService.Workplace.cs`;
- `PageEntityService.Workplace.cs`;
- `SectionServiceMixin.NUI.js` на клиенте.

Они находятся рядом по сценарию, даже если не все имеют суффикс `.NUI.cs`.

## Диагностика

| Симптом | Где смотреть |
| ------- | ------------ |
| Секция не появляется | `SysModuleInWorkplace`, `SysAdminUnitInWorkplace`, cache |
| Карточка открывается не той схемой | `SysModuleEdit`, `CardSchema`, `CardModule` |
| После изменения workplace UI старый | `ResetScriptCache`, `ResetWorkplaceCache` |
| В UI нет visa данных секции | `SysModuleVisa`, `VisaSchemaUId` |
