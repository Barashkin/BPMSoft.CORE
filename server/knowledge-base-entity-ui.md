# Knowledge Base Entity And UI

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->

> `KnowledgeBase` в Base-пакете — сущность статьи базы знаний с UIv2
> карточкой/секцией, файлами, тегами и механизмом likes.

## Entity schema

Файл: `Autogenerated/Src/KnowledgeBaseSchema.Base.cs`.

Базовая схема наследуется от `BaseEntitySchema`. Важные колонки:

| Колонка | Назначение |
| ------- | ---------- |
| `Name` | primary display column статьи |
| `Notes` | HTML/текст решения |
| `Code` | код статьи |
| `Keywords` | ключевые слова |
| `Type` | тип статьи, lookup `KnowledgeBaseType` |
| `ViewsCount` | счетчик просмотров |
| `NotHtmlNote` | текстовая версия без HTML |

Вокруг сущности есть стандартные helper schemas:

- `KnowledgeBaseFile`;
- `KnowledgeBaseFolder`, `KnowledgeBaseInFolder`;
- `KnowledgeBaseTag`, `KnowledgeBaseTagV2`;
- `KnowledgeBaseInTagV2`;
- `KnowledgeBaseType`.

## Package layers

Репрезентативные расширения:

- `KnowledgeBaseSchema.Base.cs`;
- `KnowledgeBaseSchema.BPMSoftOCC.cs`;
- `KnowledgeBaseSchema.SspKnowledgeBase.cs`.

Итоговое имя схемы для UI — `KnowledgeBase`, а Base layer может называться
`KnowledgeBase_Base_BPMSoft`. Это важно при поиске generated class и при
сопоставлении `entitySchemaName` в клиентских модулях.

## KnowledgeBasePageV2

Файл: `Autogenerated/Src/KnowledgeBasePageV2.UIv2.js`.

Карточка:

- использует `entitySchemaName = "KnowledgeBase"`;
- включает `HtmlEditModeEnabled`;
- проверяет feature `Playbook`;
- инициализирует HTML editor collections;
- добавляет detail `Files` с `KnowledgeBaseFile`;
- управляет likes через сущность `Like`;
- переопределяет `initTags` на `KnowledgeBaseTagV2` /
  `KnowledgeBaseInTagV2`.

Механизм likes:

1. Batch-запрос считает `SUM(LikeIt)` по текущей статье.
2. Отдельный select проверяет, поставил ли текущий contact like.
3. `setLikeIt` удаляет существующую запись и при необходимости создает новую.

## KnowledgeBaseSectionV2

Файл: `Autogenerated/Src/KnowledgeBaseSectionV2.UIv2.js`.

Секция задает:

- `entitySchemaName = "KnowledgeBase"`;
- `ContextHelpId = 1006`;
- tag schemas `KnowledgeBaseTagV2` и `KnowledgeBaseInTagV2`.

Основная логика секции минимальна: поведение опирается на базовые section
паттерны UIv2.

## Files and HTML editor

В карточке используется `FileDetailV2`:

```text
KnowledgeBase.Id -> KnowledgeBaseFile.KnowledgeBase
```

Для HTML-режима изображения сначала читаются через `readAsDataURL`, затем
попадают в `knowBaseImagesCollection`.
