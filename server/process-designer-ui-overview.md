# Process Designer UI Overview

<!-- Версия: 1.1 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: ProcessDesigner, ProcessLibrary, property pages, SchemaDesigner, DCM -->

> Входная точка в Process Designer UI Dive: как открывается дизайнер процессов,
> как устроены property pages, user task pages, mapping modules, sandbox messages
> и где проходит граница между UI-дизайнером и runtime Process Dive.

## Границы

Этот dive описывает UI и metadata-editing контур:

- открытие дизайнера из Process Library;
- `ProcessModuleUtilities.showProcessSchemaDesigner`;
- модули property pages;
- параметры и mapping pages;
- user task properties;
- validation на property pages;
- sandbox messages;
- DCM как соседний наследник process designer base.

Не дублируется runtime:

- `ProcessSchema`, `Process`, `ProcessModel` и execution flow;
- `ProcessUserTask.InternalExecute`;
- `SysProcessLog` и runtime troubleshooting;
- Quartz запуск процессов.

Runtime покрыт в [Process Overview](process-overview.md).

DCM как самостоятельный stage-based контур покрыт в
[DCM Dynamic Case Overview](dcm-dynamic-case-overview.md).

## Главная граница Base

В `Autogenerated/Src` видны вызовы и расширения дизайнера, но не весь canvas
engine:

- `SchemaDesigner` открывается через URL;
- `BPMSoft.Designers.BaseProcessSchemaDesignerViewModel` используется
  наследниками, например DCM;
- property pages и modules представлены полно.

Поэтому в Base корректно документировать UI-обвязку, property pages и contracts,
а не внутреннюю реализацию canvas.
