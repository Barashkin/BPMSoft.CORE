# Global Search Client UI

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: GlobalSearchResultPage, GlobalSearchModule, SearchRowSchema -->

> Клиентская часть Global Search состоит из shell-модуля, страницы
> результатов, local storage, генератора view config и схем строк результата.

## GlobalSearchModule

Файл: `Autogenerated/Src/GlobalSearchModule.GlobalSearch.js`.

`GlobalSearchModule` наследуется от `BPMSoft.BaseSchemaModule`. После
инициализации он вызывает `viewModel.search()`, поэтому модуль используется
как entry point для запуска поиска при открытии страницы результатов.

## GlobalSearchResultPage

Файл: `Autogenerated/Src/GlobalSearchResultPage.GlobalSearch.js`.

Страница результатов:

- подписывается на sandbox message `GlobalSearch`;
- публикует `ChangeHeaderCaption`, `InitDataViews`, `PushHistoryState`;
- слушает `HistoryStateChanged` и отменяет предыдущий запрос;
- хранит `ResultCollection`;
- ведет paging через `RecordCount`, `From`, `LastLoadedRecordsCount`;
- хранит `SearchRequestId`;
- использует `ConfigurationServiceProvider`;
- отправляет performance/GTM события;
- показывает help links через `AcademyUtilities`.

Базовый размер порции результатов — `RecordCount = 15`.

## GlobalSearchStorage

Файл: `Autogenerated/Src/GlobalSearchStorage.GlobalSearch.js`.

`GlobalSearchStorage` — singleton для client-side состояния поиска. Он
использует local store с ключами `GlobalSearch` и `SearchParams`. Через него
страница результатов восстанавливает параметры поиска при навигации.

## View generator

Файл: `Autogenerated/Src/GlobalSearchViewGenerator.GlobalSearch.js`.

`GlobalSearchViewGenerator` генерирует view config для результатов поиска.
Он отделяет структуру UI от конкретных row schemas.

## BaseSearchRowSchema

Файл: `Autogenerated/Src/BaseSearchRowSchema.GlobalSearch.js`.

Базовая схема строки результата:

- хранит `ViewConfig`;
- содержит `FoundColumnsCollection`;
- рассчитывает `EntitySchemaCaption`;
- строит `PrimaryColumnURL`;
- подгружает typed column value через `NetworkUtilities`;
- инициализирует edit pages, multilookup и атрибуты;
- строит блоки дополнительных найденных колонок;
- применяет highlight text для найденных значений;
- использует `MiniPageUtilities`.

Если найденная колонка не входит в bind map строки и не является primary
display column, она попадает в дополнительные found columns.

## Специализированные строки

Для часто используемых сущностей есть отдельные row schemas:

- `AccountSearchRowSchema.GlobalSearch.js`;
- `ContactSearchRowSchema.GlobalSearch.js`;
- `ActivitySearchRowSchema.GlobalSearch.js`;
- `LeadSearchRowSchema.GlobalSearch.js`;
- `OpportunitySearchRowSchema.GlobalSearch.js`;
- `CaseSearchRowSchema.GlobalSearch.js`;
- `KnowledgeBaseSearchRowSchema.GlobalSearch.js`;
- `OrderSearchRowSchema.GlobalSearch.js`;
- `InvoiceSearchRowSchema.GlobalSearch.js`;
- `FileSearchRowSchema.GlobalSearch.js`.

Специализированная схема обычно переопределяет отображение, набор колонок или
поведение перехода в карточку.

## Клиентский фильтр

Файл: `Autogenerated/Src/GlobalSearchFilter.GlobalSearch.js`.

`GlobalSearchFilter` отвечает за клиентское представление фильтров поиска:
типы сущностей, группировки и параметры, которые затем влияют на service call.
