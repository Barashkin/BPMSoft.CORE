# EventListeners Overview

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->

> Входная точка в EventListener Dive: какие listener'ы есть в BPMSoft, когда их использовать и куда идти дальше.

## Две основные категории

| Тип | Назначение | Базовый API |
| --- | ---------- | ----------- |
| Entity EventListener | Реакция на CRUD-события конкретной схемы | `BaseEntityEventListener`, `[EntityEventListener]` |
| App EventListener | Реакция на жизненный цикл приложения и сессии | `IAppEventListener`, `AppEventListenerBase` |

Отдельно существует `EventsProcess`: embedded process, который платформа генерирует для событий сущностей. Он описан в [event-listeners-eventsprocess.md](event-listeners-eventsprocess.md).

## Когда выбирать Entity EventListener

Используйте Entity EventListener, если логика должна срабатывать при изменении конкретной записи:

- проверить уникальность или обязательные условия перед сохранением;
- отменить удаление или изменение системной записи;
- очистить cache после изменения настроек;
- синхронизировать связанные записи;
- поставить async operation после успешного save.

## Когда выбирать App EventListener

Используйте App EventListener, если логика относится к старту или остановке приложения:

- зарегистрировать Quartz jobs;
- выполнить warmup;
- настроить `ClassFactory.Bind`;
- инициализировать notifier/listener инфраструктуру;
- очистить application cache на `OnAppEnd`.

## Когда лучше не использовать listener

Не стоит помещать в listener:

- долгие внешние API-вызовы без async/job-обёртки;
- сложную UI-specific логику;
- массовую обработку данных при каждом `Save`;
- повторные `Save()` той же сущности без защиты от рекурсии;
- код, который нужен только одному сервисному методу.
