> Client UI объединяет counters, списки уведомлений и realtime-сообщения из
> `ClientMessageBridge`.

## Notification counters request

Файл: `Autogenerated/Src/NotificationCountersServiceRequest.NUI.js`.

Request вызывает:

```text
serviceName = NotificationInfoService
contractName = GetCounters
```

Нормализованные поля:

- `remindingsCount`;
- `visaCount`;
- `esnNotificationsCount`;
- `notificationsCount`;
- `anniversariesCount`;
- `resetCounters`.

## BaseNotificationsSchema

Файл: `Autogenerated/Src/BaseNotificationsSchema.NUI.js`.

`BaseNotificationsSchema` — базовая клиентская схема для списков notification
items. Она работает с entity schema `Reminding`.

Ключевые sandbox messages:

| Message | Роль |
| ------- | ---- |
| `MarkNewNotificationsAsRead` | отметить новые как прочитанные |
| `UpdateCounters` | обновить counters |
| `UpdateCounterValues` | обновить конкретные counter values |
| `ReloadCard` | перезагрузить карточку |
| `GetNotificationInfo` | запрос notification info |
| `UpdateNotifications` | обновить список |
| `ProcessExecDataChanged` | передать параметры процесса |
| `RemoveNotificationCounters` | удалить counters |
| `GetLastRemindTime` | получить дату для query |
| `GetNeedReadOnOpen` | read-on-open behavior |

При `init` схема:

1. Проверяет feature `NotificationV2`.
2. Загружает notifications.
3. Подписывается на events.
4. Публикует counters через `UpdateCounterValues` или legacy `UpdateCounters`.

## ReminderNotificationsSchema

Файл: `Autogenerated/Src/ReminderNotificationsSchema.NUI.js`.

Специализация для reminder group:

```text
NotificationType = ConfigurationConstants.Reminding.Reminding
NotificationGroup = ConfigurationConstants.NotificationGroup.Reminding
```

Добавляет:

- меню delay на 5/10/30/60/120/1440 минут;
- activity-specific image по category;
- дополнительные columns из linked `Activity`;
- подписи account/contact/status;
- truncate `SubjectCaption` до 90 символов.

## RemindingsModule

Файл: `Autogenerated/Src/RemindingsModule.NUI.js`.

Legacy/panel module объединяет три коллекции:

- `remindingsGridData`;
- `emailsGridData`;
- `approvalsGridData`.

Модуль:

- вызывает `RemindingsUtilities.getRemindingsCounters`;
- грузит текущую grid по selected type;
- поддерживает lazy load;
- добавляет links на связанные записи;
- умеет delay reminder;
- переключает counters/buttons для `Email`, `Reminding`, `Visa`.

## CenterNotificationModule

Файл: `Autogenerated/Src/CenterNotificationModule.UIv2.js`.

Модуль наследуется от `BaseSchemaModule` и открывает schema:

```text
CenterNotificationSchema
```

`useHistoryState = false`, так как notification center работает как панель/центр,
а не как полноценный route.

## Realtime updates

Realtime updates приходят через `ClientMessageBridge`:

- `NotificationInfo` — новые notifications;
- `UpdateReminding` — изменить счетчик;
- `GetRemindingCounters` — получить counters;
- `ReloadApplication` — перезагрузка приложения.

UI-модули не должны напрямую читать websocket transport. Они работают через
sandbox messages.

## Практические правила

- Для новых notification list UI наследуйтесь от `BaseNotificationsSchema`.
- Для reminder-specific UI используйте `ReminderNotificationsSchema`.
- Для счетчиков используйте `NotificationCountersServiceRequest`.
- Для realtime обновлений подписывайтесь на sandbox messages, а не на transport.
- Не смешивайте ESN notification UI с NUI `Reminding` UI.
