# CTI Telephony Settings And Initialization

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: CTI, SysMsgLib, SysMsgUserSettings, license -->

> Инициализация CTI начинается на клиенте: `CtiBaseHelper` читает системные
> настройки, данные выбранной библиотеки сообщений, пользовательские параметры
> подключения и лицензии.

## CtiBaseHelper

Файл: `Autogenerated/Src/CtiBaseHelper.CTIBase.js`.

`CtiBaseHelper` хранит состояние в `BPMSoft.SysValue.CTI`. Если настройки уже
инициализированы, повторный вызов `queryCtiSettings` возвращает готовый
объект.

Цепочка инициализации:

1. `querySysSettings`.
2. Проверка `SysMsgLib`.
3. `initializeMsgLibSettings`.
4. Batch-запрос к `SysMsgLib` и `SysMsgUserSettings`.
5. Разбор `ConnectionParams`.
6. `queryCtiLicenses`.
7. Заполнение `canUseCti`.

## SysSettings

`CtiBaseHelper` читает настройки:

| SysSetting | Назначение |
| ---------- | ---------- |
| `SysMsgLib` | выбранная библиотека CTI/сообщений |
| `InternalNumberLength` | длина внутреннего номера |
| `SearchNumberLength` | минимальная длина номера для поиска |
| `CommunicationHistoryRowCount` | размер истории звонков |
| `DefaultContactCommunicationType` | тип коммуникации контакта по умолчанию |
| `DefaultAccountCommunicationType` | тип коммуникации аккаунта по умолчанию |

Для WebRTC дополнительно используются звуки:

- `WebRTCOutRing`;
- `WebRTCInRing`;
- `WebRTCCancelRing`.

## SysMsgLib

Файл: `Autogenerated/Src/SysMsgLibSchema.Base.cs`.

`SysMsgLib` описывает библиотеку телефонии:

| Колонка | Назначение |
| ------- | ---------- |
| `FullClassName` | серверный класс библиотеки |
| `SetupPageSchemaName` | клиентская страница настройки |
| `LicOperations` | operation licenses через `;` |
| `CtiProviderName` | AMD/provider name для загрузки |

`CtiBaseHelper` использует `CtiProviderName` для последующей загрузки
provider через `CtiProviderInitializer`.

## SysMsgUserSettings

`SysMsgUserSettings.ConnectionParams` содержит JSON параметров подключения
конкретного пользователя к выбранной библиотеке `SysMsgLib`.

Если запись отсутствует и у библиотеки есть `SetupPageSchemaName`, helper
логирует сообщение о пустой конфигурации. Если JSON не разбирается, helper
логирует ошибку некорректной конфигурации.

## Проверка лицензий

Файлы:

- `Autogenerated/Src/CtiBaseHelper.CTIBase.js`;
- `Autogenerated/Src/CtiRightsService.CTIBase.cs`.

`LicOperations` из `SysMsgLib` разбиваются по `;` и передаются в
`CtiRightsService.GetUserHasOperationLicense`.

Сервис проверяет операции через:

```text
UserConnection.LicHelper.GetHasOperationLicense(operation)
```

Если `isAnyOperation = false`, пользователь должен иметь все операции. Если
`isAnyOperation = true`, достаточно одной операции.

## Признак включенной телефонии

`GetIsTelephonyEnabled` возвращает `true`, только если:

- заполнен `sysMsgLibId`;
- заполнен `ctiProviderName`;
- есть `connectionParams`;
- `canUseCti = true`;
- `connectionParams.disableCallCentre` не включен.

Это основной быстрый чек для UI-компонентов, которые должны решить,
показывать ли CTI-функции пользователю.
