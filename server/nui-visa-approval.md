# NUI Visa And Approval

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: NUI, Visa, Approval, SSP, согласование -->

> Серверный контур согласований в NUI: `ApprovalService`,
> `VisaDataService`, `BaseVisa`, `VwVisa`, статусы и история виз.

## Область

Visa/approval в NUI состоит из двух близких, но разных частей:

| Часть | Назначение |
| ----- | ---------- |
| `ApprovalService` | approve/reject действия над конкретной визой |
| `VisaDataService` | выборка списка виз, карточки визы и истории согласований |

Оба сервиса доступны как из основного приложения, так и из SSP:

```csharp
[DefaultServiceRoute]
[SspServiceRoute]
```

## ApprovalService

Опорный файл: `Autogenerated/Src/ApprovalService.NUI.cs`.

Сервис публикует четыре метода:

| Метод | Ответ | Назначение |
| ----- | ----- | ---------- |
| `Approve(request)` | `bool` | принять визу |
| `Reject(request)` | `bool` | отклонить визу |
| `ApproveWithResponse(request)` | `BaseResponse` | принять с error model |
| `RejectWithResponse(request)` | `BaseResponse` | отклонить с error model |

`ApprovalRequest` содержит:

- `id` — идентификатор визы;
- `schemaName` — schema согласования;
- `additionalColumnValues` — значения колонок;
- `additionalValues` — дополнительные key/value параметры.

Сервис не меняет запись напрямую. Он получает `IApprovalAction` через
`ClassFactory`, добавляет `StatusId`, затем вызывает `ChangeApproval(...)` или
`ChangeApprovalWithLocationException(...)`.

## Статусы

Статусы задаются через `ApprovalConstants`:

| Статус | Использование |
| ------ | ------------- |
| `Accepted` | approve |
| `Rejected` | reject |
| `Canceled` | исключается из истории в `VisaDataService` |

При повторной обработке финальной визы сервис возвращает понятную ошибку через
`VisaFinalStatusException`: запрос уже принят или отклонён.

## VisaDataService

Опорный файл: `Autogenerated/Src/VisaDataService.NUI.cs`.

Ключевые методы:

| Метод | Назначение |
| ----- | ---------- |
| `GetVisaEntities(sysAdminUnitId, requestOptions)` | список виз для пользователя/роли |
| `GetVisaEntity(sysAdminUnitId, visaId)` | одна виза |
| `GetVisaHistory(parentSchemaName, parentRecordId)` | история согласований по записи |

`VisaDataService` использует `VisaHelper`, который строит объединённое
представление поверх `BaseVisa`, расширяет схему виртуальными колонками
(`Title`, `VisaSchemaName`, `VisaObjectId`, `SchemaName`) и возвращает
`SelectQueryResponse`, сериализованный через `ServiceStackTextHelper`.

## История согласований

Для истории сервис:

1. Находит настройку `SysModuleVisa` через `SysModule.Code`.
2. Получает `VisaSchemaUId` и `MasterColumnUId`.
3. Строит ESQ по конкретной visa-схеме.
4. Фильтрует записи по master column и `IsCanceled = false`.
5. Исключает `ApprovalConstants.Canceled`.
6. Возвращает rows и row config в формате NUI select response.

## Уведомления и counters

Visa тесно связана с Notifications/Reminders, но это разные слои:

- NUI visa API меняет и читает согласования;
- notification providers/counters доставляют уведомления и счётчики.

Ключевые смежные файлы:

- `BaseVisaNotificationProvider.NUI.cs`;
- `SysModuleVisaNotificationProvider.NUI.cs`;
- `VisaCountProvider.NUI.cs`;
- `VisaNotificationCounter.NUI.cs`.

Подробный notification pipeline см. в
[notifications-reminders-overview.md](notifications-reminders-overview.md).

## Типовые ошибки

| Симптом | Где смотреть |
| ------- | ------------ |
| Approve/reject вернул `false` | `IApprovalAction.ChangeApproval(...)` |
| `Visa not found` | schema/id в `ApprovalRequest` |
| "already approved or denied" | финальный статус визы |
| История пустая | `SysModuleVisa`, `MasterColumnUId`, `IsCanceled` |
| SSP approve не работает | `[SspServiceRoute]`, права SSP пользователя |
