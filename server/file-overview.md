> Входная точка в File Storage Dive: модель файлов, загрузка, скачивание, UI-детали, версии, удаление и ограничения.

## Общая модель

Файловая подсистема BPMSoft состоит из нескольких слоёв:

| Слой | Компоненты | Назначение |
| ---- | ---------- | ---------- |
| Metadata entity | `File`, `{Entity}File`, `SysFileStorage` | запись файла, связь с master record, storage metadata |
| Content API | `FileRepository`, `IFile`, `EntityFileLocator` | чтение, запись и удаление контента |
| Service layer | `FileApiService`, legacy `FileService` | upload/download endpoints |
| Client layer | `FileDetailV2`, `ConfigurationFileApi`, `FileDownloader` | UI, drag-and-drop, chunk upload |
| Lifecycle | `File_BaseEventsProcess`, `BaseEntityFileDeleteListener` | версия, size, sanitize, каскадное удаление |
| Security and limits | record rights, `MaxFileSize`, file security schemas | доступ и ограничения |

## Базовый принцип

Файл в BPMSoft — это не только binary column. Это запись metadata, связанная с master entity, плюс контент в выбранном storage. Поэтому upload/delete надо рассматривать как согласованное изменение metadata и content.
