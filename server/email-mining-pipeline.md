# Email Mining Pipeline

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: EmailMiningJob, EmailMiner, EmailMiningServiceProxy, Quartz -->

> Pipeline отвечает за выбор писем, вызов cloud text parsing service,
> сохранение mined data и уведомление клиентов через message channel.

## App listener

Файл: `Autogenerated/Src/EmailMiningAppListener.EmailMining.cs`.

`EmailMiningAppListener` на старте приложения:

1. Удаляет старые jobs группы `EmailMiningJob`.
2. Планирует immediate job `EmailMiningJob`.
3. Использует `SystemUserConnection`.

Группа job:

```text
EmailMiningJob
```

## EmailMiningJob

Файл: `Autogenerated/Src/EmailMiningJob.EmailMining.cs`.

SysSettings:

| SysSetting | Назначение |
| ---------- | ---------- |
| `EnableEmailMining` | включает или выключает mining |
| `EmailMiningPeriodMin` | период self-reschedule |
| `EmailMiningPackageSize` | размер пакета писем |
| `EmailMiningIdentificationActualPeriod` | окно актуальности поиска/идентификации |

Выборка писем:

- `Activity.Type = email`;
- `Activity.EnrichStatus = Inactive`;
- `Body` непустой;
- `SendDate` в актуальном периоде;
- есть `EmailMessageData` с role `To`, `Cc` или `Bcc`.

После обработки job публикует `SimpleMessage`:

```text
Sender = EmailMining
BodyTypeName = EmailEnriched
Body = JSON MiningResult[]
```

Адресаты берутся из `EmailMessageData.Owner` через `SysAdminUnit.Contact`.

## Self-reschedule

`EmailMiningJob.Execute` в `finally` планирует следующий запуск через
`SchedulerUtils.ScheduleNextRun`.

Если конфигурация cloud service некорректна, следующий запуск сдвигается на
дополнительные 60 минут.

Если `EmailMiningPeriodMin = 0`, job не перепланируется.

## EmailMiner

Файл: `Autogenerated/Src/EmailMiner.EmailMining.cs`.

`EmailMiner` получает зависимости через `ClassFactory`:

- `IEmailMiningServiceProxy`;
- `IContactSearcher`;
- `IDeduplicator`;
- `EnrichEntitySearchHelper`;
- `IMetricReporter`.

Поток обработки:

1. Отправляет batch писем в `IEmailMiningServiceProxy`.
2. Для каждого ответа строит hash extracted entities.
3. Ищет существующий `EnrchEmailData` по hash.
4. Создает `EnrchEmailData` и `EnrchTextEntity`, если данных еще нет.
5. Идентифицирует контакты и аккаунты.
6. Добавляет email отправителя, если он есть.
7. Помечает дубликаты mined data.
8. Обновляет `Activity.EnrichStatus` и `Activity.EnrchEmailData`.
9. Публикует метрику `email_mining_latency_seconds`.

## Cloud proxy

Файл: `Autogenerated/Src/EmailMiningServiceProxy.EmailMining.cs`.

`EmailMiningServiceProxy` вызывает endpoint:

```text
getEmailEntitiesBatch
```

Обязательные настройки:

| SysSetting | Назначение |
| ---------- | ---------- |
| `TextParsingServiceSysSetting` из `EmailMiningConsts` | URL text parsing service |
| `CloudServicesAPIKey` | API key |

Для каждого письма отправляются:

- `Body`;
- `MsgType = html/plain`;
- `ApiKey`;
- `SenderEmail`.

Обрабатываются ошибки:

- пустой service URL;
- пустой API key;
- `Unauthorized`;
- `NotFound`;
- malformed JSON;
- mismatch числа request/response;
- несовместимая версия Text Mining API.

Метрика доступности:

```text
email_mining_connect_error_count
```

## Важное разделение

Email Mining в Base — это client/proxy и persistence контур для cloud parsing.
Сама модель извлечения сущностей и cloud API находятся вне `Autogenerated/Src`.
