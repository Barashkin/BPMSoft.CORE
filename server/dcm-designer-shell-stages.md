# DCM Designer Shell And Stages

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: DcmDesigner, DcmSchemaDesignerViewModel, DcmSchemaStage -->

> DCM designer наследует общий `SchemaDesigner`, но редактирует не BPM flow,
> а DCM-схему со стадиями, элементами, переходами и stage connections.

## Entry shell

Файл: `Autogenerated/Src/DcmDesigner.DcmDesigner.js`.

`DcmDesigner` наследуется от `BPMSoft.SchemaDesigner` и переопределяет:

- `parseHash`;
- `initDesigner`.

Hash разбирается как:

```text
{designer}/{schemaId}/{dcmSettingsId}/{packageUId}
```

`schemaId = add` превращается в пустой id, чтобы открыть создание новой схемы.
После parse создаётся `BPMSoft.DcmSchemaDesigner` с:

- `sandbox`;
- `schemaUId`;
- `dcmSettingsId`;
- `packageUId`.

## DcmSchemaDesignerViewModel

Файл: `Autogenerated/Src/DcmSchemaDesignerViewModel.DcmDesigner.js`.

Класс:

- наследуется от `BPMSoft.BaseProcessSchemaDesignerViewModel`;
- использует `BPMSoft.DcmSchemaManager`;
- задаёт `contextHelpCode = "DcmSchemaDesigner"`;
- хранит DCM-specific storage keys;
- управляет selection, stages, property pages и validation state.

DCM-specific storage keys:

- `DcmSchemaLocationHref`;
- `DcmSchemaMetaData`;
- `DcmSchemaResources`;
- `DcmSchemaLoadedPropertiesItemName`;
- `DcmSchemaLoadedPropertiesStageColumnUId`.

## DcmSchema

Файл: `Autogenerated/Src/DcmSchema.DcmDesigner.js`.

`DcmSchema` наследуется от `BaseProcessSchema`, но имеет DCM contract:

- `contractName = "ContractDcmSchema"`;
- `managerName = "DcmSchemaManager"`;
- `typeName = "BPMSoft.Core.DcmProcess.DcmSchema"`;
- `color = "#F9763D"`.

Основные collections:

- `stages`;
- `stageConnections`;
- `transitions`;
- `methods`;
- `usings`;
- `localizableStrings`;
- `systemElements`.

## Stages

Файл: `Autogenerated/Src/DcmSchemaStage.DcmDesigner.js`.

`DcmSchemaStage` наследуется от `BaseProcessSchemaElement`.

Важные свойства:

- `elements`;
- `stageRecordId`;
- `parentStageUId`;
- `color`;
- `isSuccessful`;
- `transitionType`;
- `usePermissions`;
- `permissions`;
- `editPageSchemaName = "DcmSchemaStagePropertiesPage"`.

Feature `DcmStagesPermissions` добавляет в serializable properties
`permissions` и `usePermissions`.

## Elements

Файл: `Autogenerated/Src/DcmSchemaElement.DcmDesigner.js`.

DCM element оборачивает process flow element:

- `processFlowElement`;
- `requiredType`;
- `conditions`;
- `ParametrizedProcessSchemaElement` mixin;
- serialization вложенного `processFlowElement`;
- default values через `defValueForDcm`.

Edit page фактически берётся у вложенного `processFlowElement` через
`getEditPageSchemaName`.

## Transitions

`DcmSchema` хранит `transitions`, а `DcmSchemaElement` умеет:

- получить transition через `getTransition`;
- создать default transition, если transition отсутствует;
- заменить transition через `setTransition`;
- сбросить transition через `setDefaultTransition`.

Переходы создаются через `DcmSchemaElementTransitionFactory`.

## Практические правила

- DCM designer диагностируйте через stages, а не BPM sequence flows.
- Если элемент не выбирается, проверяйте `selectedItemId` и stage view model.
- Если удаление запрещено, смотрите `schema.canRemoveElements` и validation info.
- Если stage permissions не сериализуются, проверьте feature `DcmStagesPermissions`.
