# Process Designer Property Pages

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: ProcessDesigner, property pages, validation, ProcessFlowElementPropertiesPage -->

> Property pages — это правая панель редактирования элементов процесса. Она
> работает с metadata-объектом элемента, валидирует его и сохраняет изменения
> обратно в process schema.

## Properties edit module

Файл: `Autogenerated/Src/ProcessSchemaElementPropertiesEdit.ProcessDesigner.js`.

Модуль:

- наследуется от `BPMSoft.ConfigurationModule`;
- `useHistoryState = false`;
- `isSchemaConfigInitialized = true`;
- использует suffix `-prSchElPropCt`;
- показывает mask при рендере;
- подписан на PTP message `ReRenderPropertiesPage`.

Это контейнер для конкретной property page schema.

## Иерархия property pages

Файл: `Autogenerated/Src/BaseProcessSchemaElementPropertiesPage.ProcessDesigner.js`.

В начале файла есть важная class tree. База:

```text
BaseProcessSchemaElementPropertiesPage
  -> ProcessFlowElementPropertiesPage
    -> FormulaTaskPropertiesPage
    -> ProcessSchemaPropertiesPage
    -> ProcessBaseEventPropertiesPage
    -> RootUserTaskPropertiesPage
      -> BaseUserTaskPropertiesPage
      -> SubProcessPropertiesPage
      -> SendEmailUserTaskPropertiesPage
```

Для диагностики property page начинайте с ближайшего leaf-файла, но держите в
голове цепочку наследования.

## Базовые атрибуты

`BaseProcessSchemaElementPropertiesPage` хранит:

| Attribute | Роль |
| --------- | ---- |
| `uId` | UId элемента |
| `name` | code name элемента |
| `caption` | отображаемое имя |
| `description` | описание |
| `ProcessElement` | редактируемый metadata object |
| `ProcessSchema` | родительская схема |
| `ActiveTabName` | активная вкладка |
| `TabsCollection` | вкладки property page |
| `ToolsButtonMenu` | меню действий |
| `IsExtendedMode` | расширенный режим |
| `AllowCompileMode` | доступность compile mode |

## Mixins

Базовая страница использует:

- `BPMSoft.ProcessSchemaElementEditable`;
- `BPMSoft.ObservableItem`;
- `MappingEditMixin`.

`ProcessFlowElementPropertiesPage` добавляет:

- `BPMSoft.MappingEditMixin`;
- `BPMSoft.ProcessSchemaParametersEditMixin`.

## Validation

Base page добавляет validators:

- `nameValidator`;
- `duplicateNameValidator`;
- `customValidator`;
- mapping validator для колонок типа `MAPPING`.

Поток validation:

1. `internalValidate` на process element.
2. Column validation.
3. Custom validation.
4. Результаты складываются в `processElement.validationResults`.
5. UI state обновляется через `setValidationState`.

`nameValidator` требует code name по regex:

```text
^[a-zA-Z]{1}[a-zA-Z0-9_]*$
```

## Process schema properties

Файл: `Autogenerated/Src/ProcessSchemaPropertiesPage.ProcessDesigner.js`.

Страница свойств процесса редактирует schema-level metadata:

- `useForceCompile`;
- `tag`;
- process methods body;
- compiled methods body;
- usings;
- nested parameters;
- notification caption.

Она также встраивает `SourceCodeEditPage` для `methodsBody` и
`compiledMethodsBody`.

## Практические правила

- Не сохраняйте изменения напрямую в runtime-класс процесса из UI.
- Валидацию element metadata кладите в `validationResults`.
- Для новых flow-element страниц наследуйтесь от ближайшей существующей page.
- Для параметров и mappings используйте существующие mixins/modules.
