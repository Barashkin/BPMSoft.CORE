# DCM Dynamic Case Pattern Catalog

> Каталог практических DCM-паттернов из `Autogenerated/Src`.

## SchemaDesigner adapter

Источник: `DcmDesigner.DcmDesigner.js`.

Паттерн:

- наследовать общий `SchemaDesigner`;
- распарсить hash в DCM-specific параметры;
- создать конкретный designer с `schemaUId`, `dcmSettingsId`, `packageUId`.

## Manager over view entity

Источник: `DcmSchemaManager.DcmDesigner.js`.

Паттерн:

- `entitySchemaName = "VwSysDcmLib"`;
- `schemaManagerServiceName = "ServiceModel/DcmSchemaManagerService.svc"`;
- manager item class отдельно;
- `useNotification = true`;
- outdated event `DcmSchemaManagerOutdated`.

## Default stage on new schema

Источник: `DcmSchemaManager.DcmDesigner.js`.

Паттерн: при создании новой schema manager добавляет default stage, помечает
её invalid и связывает с текущей schema UId.

## Stage-based schema model

Источник: `DcmSchema.DcmDesigner.js`.

Паттерн:

- отдельная collection `stages`;
- `flowElements` собирается из elements всех стадий;
- stage connections живут отдельно;
- transitions сериализуются отдельно от элементов.

## DCM element wrapping process flow element

Источник: `DcmSchemaElement.DcmDesigner.js`.

Паттерн:

- DCM element хранит `processFlowElement`;
- edit page и images делегируются вложенному flow element;
- serialization включает вложенный `processFlowElement`;
- DCM-only поля: `requiredType`, `conditions`, transition.

## DCM-specific user task manager

Источник: `DcmElementSchemaManager.DcmDesigner.js`.

Паттерн:

- наследование от `BaseProcessFlowElementSchemaManager`;
- `userTaskSchemaManagerName = "DcmUserTaskSchemaManager"`;
- feature `DcmNotImplementedElements`;
- DCM core element `DcmProcessSubprocessSchema`.

## Section settings as manager composition

Источник: `SectionDcmSettings.DcmLibrary.js`.

Паттерн:

- settings module управляет тремя managers:
  `DcmSchemaManager`, `SysDcmSettingsManager`,
  `SysDcmSchemaInSettingsManager`;
- reload переинициализирует managers;
- состояние хранится в view model attributes, не в DOM.

## Wizard module composition

Источник: `SectionWizardCasesSettings.DcmLibrary.js`.

Паттерн:

- wizard step встраивает settings module и library section module;
- initialized state зависит от двух child modules;
- save/validate идут через PTP messages;
- reload идёт через broadcast и ServerChannel.

## Filter uniqueness validation

Источник: `DcmSchemaManager.DcmDesigner.js`.

Паттерн:

- filter column не должен совпадать со stage column;
- filter column должен быть совместим с settings filter column;
- активные схемы не должны иметь одинаковые filters.

## Stage permissions feature

Источник: `DcmSchemaStage.DcmDesigner.js`.

Паттерн:

- base stage содержит `permissions` и `usePermissions`;
- сериализация включается feature `DcmStagesPermissions`;
- server service возвращает allowed stages через `DcmPermissions`.

## DCM mapping module specialization

Источник: `DcmMappingModule.DcmDesigner.js`.

Паттерн:

- конкретный module наследует `BaseParametersMappingModule`;
- задаёт DCM-specific pages:
  `DcmMappingPage`, `DcmParameterSelectionPage`.

## DCM service as runtime gateway

Источник: `DcmService.DcmDesigner.cs`.

Паттерн:

- service не реализует runtime сам;
- получает DCM schema через `UserConnection.DcmSchemaManager`;
- делегирует запуск в `UserConnection.ProcessEngine.RunDcmProcess`.
