# Notifications Reminders Delivery Channels

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: WebSocketNotificationSender, PushNotificationSender, ServerChannel, mobile push -->

> Delivery layer отвечает за доставку одного набора `INotificationInfo` в
> разные каналы: websocket для web UI и push для mobile.

## WebSocket delivery

Файл: `Autogenerated/Src/WebSocketNotificationSender.NUI.cs`.

`WebSocketNotificationSender` реализует `INotificationHandler`.

Поток:

1. Группирует notifications по `SysAdminUnit`.
2. Формирует `NotificationInfoResponse`.
3. Находит `IMsgChannel` по `SysAdminUnit`.
4. Публикует `SimpleMessage`.

Message header:

```text
Sender = NotificationInfo
BodyTypeName = ""
```

Body содержит:

- `Counters`;
- `Notifications`;
- `LastRemindTime`, если включен feature `UseDateForNotificationsQuery`.

Если канал пользователя не найден, handler молча пропускает отправку.

## Client bridge

Файл: `Autogenerated/Src/ClientMessageBridge.UIv2.js`.

Bridge регистрирует websocket senders:

- `NotificationInfo`;
- `UpdateReminding`;
- `GetRemindingCounters`;
- `ReloadApplication`.

Каждый sender публикуется как sandbox message с тем же именем. Это связывает
server channel с client modules без прямой зависимости UI от транспорта.

## Push delivery

Файл: `Autogenerated/Src/PushNotificationSender.NUI.cs`.

`PushNotificationSender` также реализует `INotificationHandler`.

Feature flags:

| Feature | Назначение |
| ------- | ---------- |
| `UseMobilePushNotifications` | включить push service |
| `SendPushByNotifications` | отправлять push из notification pipeline |

Перед отправкой handler проверяет `PushNotificationHistory`, чтобы не отправить
тот же message повторно для того же `SysAdminUnit` и `RemindTime`.

Push additional data:

| Key | Значение |
| --- | -------- |
| `entityName` | `EntitySchemaName` |
| `recordId` | `EntityId` |
| `messageId` | `MessageId` |
| `remindTime` | formatted remind time |

После успешной отправки записывается история в `PushNotificationHistory`.

## Server push model

Связанные схемы:

- `PushNotificationTokenSchema.NUI.cs`;
- `PushNotificationServiceSchema.NUI.cs`;
- `PushNotificationHistorySchema.NUI.cs`.

Они относятся к server-side push infrastructure. Клиентская mobile обработка
описана отдельно в [Mobile services and push](mobile-services-push.md).

## Delivery failure rules

- Ошибка одного handler не останавливает остальные handlers.
- Websocket требует активный `MsgChannelManager` channel.
- Push требует включенные features и токены пользователя.
- Повторный push блокируется `PushNotificationHistory`.
- `RemindingJob` выставляет `IsNeedToSend = false` после передачи набора в
  `NotificationSender`; поэтому диагностику delivery нужно смотреть по логам и
  history, а не только по `Reminding`.
