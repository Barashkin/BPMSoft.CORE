# Auth LDAP Settings And Sync

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: auth, LDAP, ActiveDirectory, sync -->

> LDAP-контур состоит из системных настроек, админ-UI, connector/retry слоя,
> staging-таблиц и синхронизации в `SysAdminUnit` / `SysUserInRole`.

## Settings provider

Файл: `Autogenerated/Src/LdapSettingsProvider.LDAP.cs`.

`LdapSettingsProvider` читает LDAP-настройки из `SysSettings`.

Ключевые настройки:

| SysSetting | Назначение |
| ---------- | ---------- |
| `LDAPServer` | адрес LDAP-сервера |
| `LDAPServerLogin` | логин сервисного пользователя |
| `LDAPServerPassword` | пароль сервисного пользователя |
| `LDAPUsersEntry` | DN контейнера пользователей |
| `LDAPGroupsEntry` | DN контейнера групп |
| `LDAPUsersFilter` | фильтр пользователей |
| `LDAPGroupsFilter` | фильтр групп |
| `LDAPUsersInGroupFilter` | фильтр состава группы |
| `LDAPUserLoginAttribute` | LDAP-атрибут логина |
| `LDAPUserIdentityAttribute` | уникальный атрибут пользователя |
| `LDAPEntryModifiedOnAttribute` | атрибут изменения записи |
| `LDAPGroupNameAttribute` | имя группы |
| `LDAPGroupIdentityAttribute` | уникальный атрибут группы |
| `LDAPAuthType` | тип аутентификации |
| `LDAPUseSecureConnection` | LDAPS |
| `LDAPSynchInterval` | интервал синхронизации |
| `LdapSyncRetryDelay` | задержка повтора |
| `LdapSyncRetryCountOnFailure` | число повторов |

Если `LDAPUseSecureConnection` включен, используется порт `636`; иначе `389`.
Feature `LDAPUseSecureConnection` управляет видимостью/доступностью secure
режима.

## LDAP connector

Файл: `Autogenerated/Src/LdapConnector.LDAP.cs`.

`LdapConnector` потокобезопасен и работает через:

- `ILdapSettingsProvider`;
- `ILdapConnectionCreator`;
- `ILdapRetryPolicy`;
- `UserConnection`;
- logger `LDAP`.

Для `SearchRequest` запросы выполняются под lock и retry policy. При временной
потере авторизации в Active Directory connector делает rebind и повторяет
поисковый запрос. Для неидемпотентных запросов изменения такой recovery не
используется.

## Auth types and account controls

Файлы:

- `LDAPAuthTypeSchema.Base.cs`;
- `ActiveDirectoryAccountControl.LDAP.cs`;
- `FreeIpaAccountControl.LDAP.cs`;
- `IAccountControl.LDAP.cs`.

`LDAPAuthType` хранит доступные типы LDAP-аутентификации. На Linux сервис
ограничивает поддержанные типы: в `LDAPSysSettingsService` явно оставлен Basic
auth type.

## LDAP admin service

Файл: `Autogenerated/Src/LDAPSysSettingsService.LDAP.cs`.

`LDAPSysSettingsService` наследуется от `BaseService` и работает от текущего
`UserConnection`. Перед изменением LDAP-данных вызывает:

```text
DBSecurityEngine.CheckCanExecuteOperation("CanManageAdministration")
```

Сервис создает `ILdapSettingsProvider`, `ILdapRetryPolicy`,
`ILdapConnectionCreator`, затем через `LdapUtilities` загружает группы и
пользователей.

## Admin UI

Файл: `Autogenerated/Src/LDAPServerSettings.LDAP.js`.

UI содержит virtual columns для LDAP-настроек и operation:

```text
SecurityOperationName = "CanManageAdministration"
```

Важные поля:

- `LDAPServer`;
- `LDAPUsersEntry`;
- `LDAPGroupsEntry`;
- `LDAPAuthType`;
- `LDAPServerLogin`;
- `LDAPServerPassword`;
- `LDAPEntryModifiedOnAttribute`;
- `LDAPUserLoginAttribute`;
- `LDAPUserIdentityAttribute`;
- `LDAPGroupNameAttribute`;
- `LDAPGroupIdentityAttribute`;
- `LDAPUsersFilter`;
- `LDAPGroupsFilter`;
- `LDAPUsersInGroupFilter`;
- `LDAPAllowLicenseDistributionDuringLDAPSync`;
- `LDAPUseSecureConnection`.

## Staging schemas

Файлы:

- `SysLDAPSynchUserSchema.LDAP.cs`;
- `SysLDAPSynchGroupSchema.LDAP.cs`;
- `LDAPUserInLDAPGroupSchema.LDAP.cs`;
- `LDAPElementSchema.LDAP.cs`;
- `VwSysAdminUnitSchema.LDAP.cs`.

`SysLDAPSynchUser` хранит промежуточные LDAP-пользовательские атрибуты:
`Id`, `Name`, `ModifiedOn`, `Company`, `Email`, `Phone`, `JobTitle`,
`IsActive`, `FullName`, `Dn`, `LdapSyncId`.

## Sync helper

Файл: `Autogenerated/Src/SyncWithLDAPProcessHelper.LDAP.cs`.

Ключевые паттерны:

- статические `SemaphoreSlim` для критических участков sync;
- `OrgStructureManager` для оргструктуры;
- staging table `SysLDAPSynchUser`;
- удаление/добавление связей `SysUserInRole`;
- связь `LDAPElement` и `LDAPUserInLDAPGroup`;
- учет лицензий через `LDAPLicenseInfo`;
- logger `LDAP`.

Процессы:

- `RunLDAPImport.LDAP.cs`;
- `RunLDAPSync.LDAP.cs`;
- `SyncWithLDAPProcess.LDAP.cs`;
- `SysAdminsNotificationProcess.LDAP.cs`.
