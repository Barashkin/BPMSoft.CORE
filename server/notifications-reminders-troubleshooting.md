# Notifications Reminders Troubleshooting

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: Notifications, Reminding, troubleshooting, WebSocket, Push -->

> Диагностика reminders идет по цепочке: запись `Reminding`, выборка job,
> sender handlers, websocket/push delivery, client counters/UI.

## Reminder не отправляется

Проверьте запись `Reminding`:

- `RemindTime` попадает в due window;
- `NotificationTypeId = RemindingConsts.NotificationTypeRemindingId`;
- `IsNeedToSend = true`;
- `ContactId` заполнен;
- у contact есть связанный `SysAdminUnit`;
- `SubjectId` и `SysEntitySchemaId` корректны.

Проверьте settings:

- feature `NotificationV2` включен;
- `NotificatonJobInterval` больше 0;
- `IntervalInMinutesForFindReminding` не слишком мал.

Файлы:

- `Autogenerated/Src/RemindingJob.NUI.cs`;
- `Autogenerated/Src/RemindingRepository.NUI.cs`.

## Job не запускается

Проверьте:

- `NotificationEventListener.OnAppStart`;
- job group `RemindingGroup`;
- job name `RemindingJob`;
- `NotificatonJobInterval`;
- состояние Quartz/AppScheduler.

Связанные документы:

- [Scheduler Quartz](scheduler-quartz.md);
- [Quartz Troubleshooting](quartz-troubleshooting.md).

## Reminder отправился, но popup не пришел

Проверьте websocket delivery:

- `NotificationSender` получил notifications;
- `WebSocketNotificationSender` зарегистрирован как `INotificationHandler`;
- пользователь имеет активный `MsgChannelManager` channel;
- `SysAdminUnit` в `NotificationInfo` совпадает с активным пользователем;
- `ClientMessageBridge` зарегистрировал sender `NotificationInfo`;
- UI подписан на sandbox messages.

Файлы:

- `Autogenerated/Src/NotificationSender.NUI.cs`;
- `Autogenerated/Src/WebSocketNotificationSender.NUI.cs`;
- `Autogenerated/Src/ClientMessageBridge.UIv2.js`.

## Push не приходит

Проверьте:

- feature `UseMobilePushNotifications`;
- feature `SendPushByNotifications`;
- push token пользователя;
- `PushNotificationHistory` — не был ли push уже отправлен;
- `MessageId`, `SysAdminUnitId`, `RemindTime`;
- mobile receiver flow.

Файл:

- `Autogenerated/Src/PushNotificationSender.NUI.cs`.

Подробнее:

- [Mobile services and push](mobile-services-push.md);
- [Mobile troubleshooting](mobile-troubleshooting.md).

## Счетчик не обновляется

Проверьте:

- `NotificationCountersServiceRequest`;
- `NotificationInfoService.GetCounters`;
- counter groups: `Reminding`, `Visa`, `ESNNotification`, `Notification`,
  `Anniversary`;
- sandbox messages `UpdateCounters` и `UpdateCounterValues`;
- `ClientMessageBridge` messages `UpdateReminding` и `GetRemindingCounters`.

Файлы:

- `Autogenerated/Src/NotificationCountersServiceRequest.NUI.js`;
- `Autogenerated/Src/BaseNotificationsSchema.NUI.js`;
- `Autogenerated/Src/RemindingsModule.NUI.js`.

## UI список пустой, но счетчик есть

Проверьте:

- selected notification group;
- filters в конкретной schema (`ReminderNotificationsSchema`,
  `SystemNotificationsSchema`, `VisaNotificationsSchema`);
- `CanLoadMoreData`;
- `LastRemindTime`, если включен `UseDateForNotificationsQuery`;
- `IsRead` и read-on-open logic.

## Delay reminder не работает

Проверьте:

- меню delay в `ReminderNotificationsSchema`;
- значения 5/10/30/60/120/1440 минут;
- сервис/utility, который обновляет `RemindTime`;
- что после delay запись не остается в due window;
- что counters обновились после удаления item из collection.

## Icon отсутствует

Проверьте:

- `NotificationsSettings`;
- `SysEntitySchemaUId`;
- `NotificationType`;
- `SysImage`;
- `NotificationSettingsRepository` cache в `WorkspaceCache`.

Если settings исправлены, а icon не меняется, учитывайте cache.

## ESN notification не приходит

Не начинайте с `Reminding`, если речь про feed/comment/mention.

Проверьте ESN pipeline:

- `ESNNotification`;
- `ESNNotificationProvider`;
- `ESNNotificationCounter`;
- `ESNNotificationModule`.

Подробнее: [ESN troubleshooting](esn-troubleshooting.md).

## Activity reminder date есть, но reminder не создан

Это проблема source logic Activity, а не delivery.

Проверьте:

- Activity lifecycle;
- reminder fields;
- text former;
- user task/source logic, которая должна создать `Reminding`.

Подробнее:

- [Activity lifecycle](activity-lifecycle.md);
- [Activity troubleshooting](activity-troubleshooting.md).

## Повторные push или popup

Для push проверьте `PushNotificationHistory`.

Для popup/websocket проверьте:

- не создаются ли дубли `Reminding`;
- корректен ли `Hash`;
- `IsNeedToSend` сбрасывается после отправки;
- due window не захватывает уже обработанные записи повторно.
