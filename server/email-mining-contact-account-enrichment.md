# Email Mining Contact Account Enrichment

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: ContactSearcher, AccountSearcher, Deduplicator, EnrichmentService -->

> Этот слой превращает extracted text entities в кандидатов Contact/Account,
> помечает дубликаты и дает UI данные для ручного сохранения.

## Contact search

Файл: `Autogenerated/Src/ContactSearcher.EmailMining.cs`.

`ContactSearcher` реализует `IContactSearcher` и ищет контакты:

| Метод | Критерий |
| ----- | -------- |
| `SearchByEmails` | `ContactCommunication.Number` |
| `SearchByName` | `Contact.Name` |
| `SearchByFirstAndLastName` | `GivenName` + `Surname` |
| `SearchByPhones` | filters из `IPhoneNumberComparer` |

Пустые входные значения возвращают пустую коллекцию, а не исключение.

## Account search

Файл: `Autogenerated/Src/AccountSearcher.EmailMining.cs`.

`AccountSearcher` реализует `IAccountSearcher`:

- `SearchByName` ищет по primary display column, затем по `AlternativeName`;
- `SearchByDomain` ищет в `AccountCommunication.Number`;
- учитываются варианты `http://`, `https://`, `www.`;
- берутся не больше 3 доменов и не больше 5 результатов.

Для domain search используются `StartWith` filters и `IsDistinct`.

## EnrichEntitySearchHelper

Файл: `Autogenerated/Src/EnrichEntitySearchHelper.EmailMining.cs`.

Helper связывает результат поиска с `EnrchTextEntity`:

- добавляет найденные контакты;
- добавляет найденные аккаунты;
- фиксирует тип идентификации;
- отделяет search logic от orchestration в `EmailMiner`.

## Deduplicator

Файл: `Autogenerated/Src/Deduplicator.EmailMining.cs`.

`Deduplicator` помечает mined entities, которые уже есть в найденном контакте.

Проверяются:

- телефоны через `IPhoneNumberComparer`;
- прочие коммуникации по `ContactCommunication.Number`;
- адрес;
- должность;
- email и другие communication values.

Дубликаты обновляются в `EnrchTextEntity`:

```text
DuplicationStatus = ExistsInEntity
```

Это UI-сигнал: найденная сущность не должна предлагаться как новое значение.

## Manual enrichment service

Файл: `Autogenerated/Src/EnrichContactService.EmailMining.cs`.

Основные методы:

| Метод | Назначение |
| ----- | ---------- |
| `GetCloudStateForEmail` | actions enrichment для email activity |
| `GetCloudStateForContact` | actions enrichment для contact |
| `GetTextDataForExistContact` | text data для существующего contact |
| `GetTextDataForNewContact` | text data для нового contact из `EnrchTextEntity` |
| `SaveEnrichedData` | сохранение выбранных enriched items |
| `RunEmailMining` | ручной запуск mining |

Сервис делегирует бизнес-логику в `EnrichContactHelper`.

## Account enrichment package

Файл: `Autogenerated/Src/EnrichmentService.Enrichment.cs`.

Это отдельный, но близкий контур enrichment для аккаунтов:

- собирает `Account.Name`, web urls, emails и email primary contact;
- фильтрует social links;
- вызывает cloud endpoint `search`;
- сохраняет `AccountEnrichedData`;
- использует `CloudServicesAPIKey` и service URL из настроек.

Модель данных:

- `BaseEnrichedData`: `SearchDate`, `CategoryTag`, `Value`;
- `AccountEnrichedData`: specialization для `Account`;
- `GlobalEnrichedData`: общий enrichment result.

Не смешивайте Account Enrichment с Email Mining: первый enrichment запускается
по аккаунту, второй — по email activity и extracted text entities.
