# Email Mining Mailbox Sync

<!-- Версия: 1.1 | Обновлено: 2026-04-28 | Платформа: BPMSoft 1.9 -->
<!-- Теги: EmailMining, MailboxSyncSettings, IMAP, Exchange -->

> Email Mining начинается не с cloud parsing, а с появления email-активности.
> Входом служат `MailboxSyncSettings`, IMAP/Exchange sync процессы и связанные
> записи `Activity`.
> Подробный mailbox/Exchange runtime вынесен в
> [Mailbox Exchange Sync Overview](mailbox-exchange-sync-overview.md).

## MailboxSyncSettings

Файл: `Autogenerated/Src/MailboxSyncSettingsSchema.Base.cs`.

Ключевые поля:

| Колонка | Назначение |
| ------- | ---------- |
| `SysAdminUnit` | владелец ящика, default current user |
| `MailServer` | сервер/провайдер |
| `UserPassword` | пароль |
| `EnableMailSynhronization` | включена синхронизация |
| `AutomaticallyAddNewEmails` | добавлять новые письма |
| `CyclicallyAddNewEmails` | циклическая загрузка |
| `EmailsCyclicallyAddingInterval` | интервал загрузки |
| `LastSyncDate` | последняя синхронизация |
| `SenderEmailAddress` | email отправителя |
| `OAuthTokenStorage` | OAuth token storage |
| `RetryCounter` | число retries |
| `ErrorCode` / `LastError` | ошибка синхронизации |
| `SynchronizationStopped` | синхронизация остановлена |

Exchange/Listener слои расширяют эту модель и добавляют listener-specific
flags. Подробнее базовый контур описан в
[Activity Mailbox Sync](activity-mailbox-sync.md).

## IMAP per-user scheduler

Файл: `Autogenerated/Src/ImapSyncJobScheduler.Base.cs`.

`ImapSyncJobScheduler` создает Quartz job:

```text
JobName = SyncImap
Group = IMAP
Process = SyncImapMail
```

Имя job строится из:

- `SyncImap`;
- `SenderEmailAddress`, если передан в parameters;
- `CurrentUser.Id`.

Это позволяет иметь отдельные jobs синхронизации для пользователей/ящиков.

## MailSyncJob

Файл: `Autogenerated/Src/MailSyncJob.MailSync.cs`.

`MailSyncJob` — задача повторной синхронизации остановленных ящиков.

SysSettings:

| SysSetting | Назначение |
| ---------- | ---------- |
| `MailReSynchronizationFrequency` | частота запуска |
| `EnableReSynchronizationMechanism` | включить механизм |

Job выбирает `MailboxSyncSettings`, где:

```text
EnableMailSynhronization = true
SynchronizationStopped = true
```

Затем сбрасывает:

- `SynchronizationStopped = false`;
- `RetryCounter = 0`;
- `ErrorCodeId = null`.

## Process entry points

Сгенерированные process-схемы:

- `SyncImapMail.Base.cs`;
- `LoadImapEmailsProcess.Base.cs`;
- `LoadExchangeEmailsProcess.Exchange.cs`.

Их лучше документировать как точки входа, а не разбирать построчно: основная
логика загрузки почты уходит в runtime/mail integration assemblies.

## Activity link

После sync письмо становится `Activity` типа email. Для mining важны поля:

- `Body`;
- `IsHtmlBody`;
- `Sender`;
- `SendDate`;
- `Type = ActivityConsts.EmailTypeUId`;
- `EnrichStatus`;
- `EnrchEmailData`.

Дедуп на уровне mailbox sync не равен dedup mined data. Для sync используется
`MailHash`, а для mining — hash extracted text entities.
