# NUI Excel Grid Export

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: NUI, Excel, export, grid, SysOperationAudit -->

> Серверный pipeline экспорта NUI grid/dashboard данных в Excel. Это не
> импорт Excel и не печатные формы Word/FastReport.

## Точка входа

Опорный файл: `Autogenerated/Src/ExportToExcelService.NUI.cs`.

Главный класс находится в namespace:

```text
BPMSoft.Configuration.ExportToExcel
```

`ExportToExcelService` наследуется от `BaseService` и работает с:

- `SelectQuery` из `BPMSoft.Nui.ServiceModel.DataContract`;
- `ExcelConverter` / `ExcelConverterCore`;
- `IRepository<StorableStreamEntity>`;
- operation rights и feature flags;
- `SysOperationAudit` для логирования экспорта.

## Основной flow

```text
Serialized SelectQuery
  -> Json.Deserialize<SelectQuery>
  -> CheckCanExport(root schema)
  -> SelectQuery.BuildEsq(UserConnection)
  -> RemoveBinaryColumnsFromQuery
  -> ExcelConverter.GetExcelData
  -> StorableStreamEntity
  -> ExportToExcelResponse { key, count }
```

Полученный `key` используется для скачивания stream через `GetExcelStream(...)`.

## Проверка прав

Перед построением файла сервис вызывает `CheckCanExport(schemaName)`.

Порядок:

1. Проверяется operation right `CanExportGrid`.
2. Если включён feature flag `UseEntityOperationGrantee`, fallback идёт в
   entity operation `Export` для конкретной схемы.
3. Если права нет, исключение не скрывается.

Это связывает Excel export с Security dive, но предметная логика экспорта
остаётся в NUI.

## Batch size

Размер батча берётся из системной настройки:

```text
ExcelExportBatchSize
```

Если настройка отсутствует или не `int`, используется default `2000`.

## Binary/image columns

Перед экспортом сервис удаляет бинарные и image lookup колонки из ESQ:

- `BinaryDataValueType`;
- `ImageLookupDataValueType`.

Это защищает grid export от попытки выгрузить неподходящие значения напрямую в
Excel.

## Pivot export

Перегрузка `PrepareExport(...)` поддерживает pivot/table data. На Unix
используется `ExcelConverterCore`, иначе обычный `ExcelConverter`.

Это важно для dashboard/table сценариев, но не делает экспорт частью Reports
dive: здесь источником остаётся serialized NUI `SelectQuery`.

## Audit записи

Если включена настройка `UseAdminOperationImportExport`, экспорт логируется в
`SysOperationAudit`.

Лог включает:

- объект (`schemaName`);
- количество выгруженных записей;
- список колонок;
- фильтры;
- column filters;
- client IP.

## Границы

| Область | Отличие |
| ------- | ------- |
| FileImport | импортирует данные в систему; NUI Excel export выгружает данные из grid |
| Reports | Word/FastReport/printables; NUI export строит Excel по `SelectQuery` |
| Analytics | dashboard widgets; pivot export здесь только формат выгрузки |
| Security | описывает права; NUI export применяет `CanExportGrid` / `Export` |
