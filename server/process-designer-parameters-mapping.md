# Process Designer Parameters Mapping

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: ProcessDesigner, parameters, mapping, mini edit pages -->

> Parameters mapping в дизайнере связывает параметры процесса, элементов и
> user task schemas через отдельные modules, mixins и sandbox messages.

## Основные файлы

| Файл | Роль |
| ---- | ---- |
| `ProcessFlowElementPropertiesPage.ProcessDesigner.js` | messages и collections параметров элемента |
| `ProcessSchemaParametersEditMixin.ProcessDesigner.js` | операции редактирования parameters |
| `ProcessSchemaParameterEditModule.ProcessDesigner.js` | module редактирования параметра схемы |
| `ProcessSchemaParameterViewModel.ProcessDesigner.js` | view model параметра |
| `ProcessParametersMappingModule.ProcessDesigner.js` | module mapping page |
| `ProcessParametersMappingPage.ProcessDesigner.js` | page для mapping |
| `ProcessElementParametersMappingModule.ProcessDesigner.js` | mapping параметров элемента |
| `ProcessDateTimeMappingModule.ProcessDesigner.js` | mapping date/time |
| `ProcessMiniEditPageModule.ProcessDesigner.js` | mini edit pages в designer context |

## ProcessParametersMappingModule

Файл: `Autogenerated/Src/ProcessParametersMappingModule.ProcessDesigner.js`.

Модуль минимален и наследуется от `BaseProcessParametersEditModule`:

- `initParametersInfo` инициализирует `parametersInfo = {}`;
- `initSchemaName` задаёт `schemaName = "ProcessParametersMappingPage"`.

Это типичный паттерн: конкретный module задаёт schema name, а базовый модуль
берёт на себя lifecycle.

## Messages flow

`ProcessFlowElementPropertiesPage` регистрирует PTP messages:

| Message | Роль |
| ------- | ---- |
| `GetParametersInfo` | отдать parameter values |
| `SetParametersInfo` | принять parameter values |
| `SaveParameterInfo` | сохранить parameter metadata |
| `DiscardParameterInfoChanges` | закрыть parameter edit без сохранения |
| `GetParameterEditInfo` | отдать данные parameter edit page |
| `GetItemEditInfo` | отдать данные item mini page |
| `SaveItemInfo` | сохранить item mini page |
| `SaveParameter` | publish сохранения параметра |

## Parameters collection

Flow element page хранит:

- `Parameters` как `BPMSoft.ObjectCollection`;
- `IsEmptyParameters`;
- `ActiveParameterEditUId`;
- flags visibility/enabled для `serializeToDB`, logging, multi-instance.

При загрузке элемента page очищает параметры, затем инициализирует свойства
элемента и collection.

## Mapping validation

Base property page добавляет `mappingValidator` для колонок
`BPMSoft.DataValueType.MAPPING`.

Это важно для диагностики: ошибка mapping часто появляется не в module mapping
page, а при общей validation property page.

## Mini edit pages

Mini edit pages используются для вложенных элементов:

- parameters;
- using declarations;
- localizable strings;
- autogenerated page elements/buttons;
- mapping rows.

Коммуникация идёт через sandbox PTP messages, поэтому при поломке mini page
проверяйте module id и набор зарегистрированных сообщений.

## Практические правила

- Для нового вида mapping создавайте узкий module поверх
  `BaseProcessParametersEditModule`.
- Не храните state mapping только в DOM. Источник истины — process element /
  process schema metadata.
- Для mapping columns используйте `BPMSoft.DataValueType.MAPPING`.
- Если save не срабатывает, проверяйте `SaveParameterInfo` и validation
  `validationResults`.
