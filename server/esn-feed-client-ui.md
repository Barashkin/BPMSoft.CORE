# ESN Feed Client UI

<!-- Версия: 1.0 | Обновлено: 2026-04-27 | Платформа: BPMSoft 1.9 -->
<!-- Теги: ESN, SocialFeed, Timeline, SocialFeedUtilities -->

> Клиентский ESN состоит из отдельного feed module, схемы `SocialFeed`,
> view model сообщения, общего миксина `SocialFeedUtilities` и timeline
> provider.

## ESNFeedModule

Файл: `Autogenerated/Src/ESNFeedModule.ESN.js`.

`ESNFeedModule` наследуется от `BPMSoft.BaseSchemaModule`.

Особенности:

- `schemaName = "SocialFeed"`;
- `generateViewContainerId = false`;
- `initHistoryState` отключен;
- подписывается на sandbox-сообщение `RerenderModule`;
- прокидывает `activeSocialMessageId` в view model.

## SocialFeed

Файл: `Autogenerated/Src/SocialFeed.ESN.js`.

Основная клиентская схема ленты. Работает вместе с:

- `SocialFeedUtilities`;
- `SocialMentionUtilities`;
- `SocialMessageViewModel`;
- `ESNFeedConfig`;
- `ESNConstants`.

## SocialFeedUtilities

Файл: `Autogenerated/Src/SocialFeedUtilities.ESN.js`.

Центральный миксин ленты.

Ключевые паттерны:

| Паттерн | Деталь |
| ------- | ------ |
| `esnServiceName = "EsnService"` | клиент вызывает серверный сервис ESN |
| `EntitiesCache` | кэш связанных бизнес-записей |
| `SocialMessageColumnNames` | минимальный набор колонок сообщения |
| `UseEsnRights` | feature-gated режим локальных прав |
| `RightUtilities` | fallback на schema operation rights |
| `initCommentCount = 3` | первичная загрузка комментариев |
| `loadAnyEntity` | загрузка произвольной связанной записи |

Если feature `UseEsnRights` включена, миксин выставляет локальные
`CanDelete`, `CanAdd`, `CanEdit` в `true`. Иначе права берутся через
`RightUtilities.getSchemaOperationRightLevel("SocialMessage")`.

## SocialMessageViewModel

Файл: `Autogenerated/Src/SocialMessageViewModel.ESN.js`.

View model поста/комментария объединяет:

- данные DTO из `EsnService`;
- форматирование текста;
- likes/comments actions;
- mentions;
- мини-карточки и links.

## Timeline provider

Файл: `Autogenerated/Src/EsnTimelineDataProvider.Timeline.js`.

`EsnTimelineDataProvider` наследуется от `BaseAjaxTimelineDataProvider` и
использует `BPMSoft.EsnTimelineSearchProvider`.

Запрос строится так:

```text
entityConfig + userFilters + search filters
  -> ServiceHelper.callService
  -> EsnService.ReadEntityMessage
  -> response.EsnMessages
  -> timeline row view models
```

Ограничение: provider ожидает одну entity config. Если `config.entities.length`
не равен `1`, выбрасывается `BPMSoft.InvalidOperationException`.

## Timeline item

Файлы:

- `SocialMessageTimelineItemViewModel.Timeline.js`;
- `SocialMessageTimelineItemView.Timeline.js`.

Отвечают за плитку ESN-сообщения в универсальном timeline. Не путать с
настройками `TimelinePageSetting` и `TimelineTileSetting`: это общий timeline,
а не только ESN.
